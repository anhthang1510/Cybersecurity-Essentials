# 7.1 - Windows History

## Tổng quan

Phần này tóm tắt quá trình phát triển của Windows từ DOS/MS-DOS đến Windows NT và Windows GUI hiện đại. Nội dung cũng nhấn mạnh rằng hệ điều hành nhiều dòng code luôn có vulnerabilities, nên cần cấu hình bảo mật, cập nhật và quản lý quyền đúng cách.

## Ý chính

- Các máy tính đời đầu cần hệ điều hành để đọc/ghi dữ liệu trên storage như floppy disk và hard disk.
- DOS và MS-DOS dùng command line để quản lý file và chạy chương trình.
- Windows đời đầu là GUI chạy trên MS-DOS; Windows hiện đại dựa trên Windows NT.
- Windows NT hỗ trợ nhiều users, nhiều processes và kiểm soát phần cứng trực tiếp.
- Windows có nhiều phiên bản client/server, 32-bit/64-bit và editions khác nhau.
- GUI gồm Desktop, Taskbar, Start menu, Notification Area, Context Menu và File Explorer.
- Windows cần được hardening vì OS và software luôn có vulnerabilities.

## Ghi chú chi tiết

### DOS và MS-DOS

DOS giúp máy tính đọc/ghi file trên disk và tổ chức file system. MS-DOS dùng command line, nơi người dùng nhập lệnh để di chuyển thư mục, sao chép, xóa, đổi tên file hoặc chạy chương trình.

Một số lệnh cơ bản gợi nhớ:

- `dir`: liệt kê file/folder.
- `cd`: chuyển thư mục.
- `cd ..`: lên thư mục cha.
- `copy`: sao chép file.
- `del`: xóa file.
- `mkdir`: tạo thư mục.
- `ren`: đổi tên.
- `help`: xem trợ giúp.

Ngày nay, Command Prompt vẫn tồn tại nhưng không phải MS-DOS thật; nó là môi trường dòng lệnh chạy bên trong Windows hiện đại.

### Từ Windows GUI đến Windows NT

Windows đời đầu cung cấp GUI chạy phía trên MS-DOS. DOS vẫn kiểm soát nhiều hoạt động nền tảng. Windows hiện đại dựa trên Windows NT, trong đó OS trực tiếp quản lý hardware, memory, processes và security.

Điểm khác biệt lớn:

- MS-DOS thiên về single-user, single-process.
- Windows NT hỗ trợ multitasking, multiple users/processes, security model và file permissions mạnh hơn.

### Windows versions và 64-bit

Windows có nhiều dòng client và server. Các phiên bản server được thiết kế cho roles như domain services, file services, web services, virtualization và datacenter workloads.

Kiến trúc 64-bit quan trọng vì:

- Cho phép address space lớn hơn rất nhiều so với 32-bit.
- Hỗ trợ workload lớn như database, scientific computing, video processing.
- Thường backward-compatible với nhiều ứng dụng 32-bit, nhưng chương trình 64-bit không chạy trên hardware/OS 32-bit.

### Windows GUI

Windows GUI giúp người dùng thao tác file, folders, shortcuts và applications qua Desktop, Taskbar và File Explorer.

Các thành phần chính:

- Desktop: nơi đặt shortcut, file, folders và Recycle Bin.
- Start menu/Search: tìm ứng dụng, settings và file.
- Quick launch/pinned icons: mở nhanh ứng dụng.
- Notification Area: hiển thị trạng thái network, audio, updates, security alerts.
- Context Menu: right-click để truy cập nhanh thao tác như copy, delete, share, print.

### Operating system vulnerabilities

Windows và applications có hàng triệu dòng code nên luôn có nguy cơ vulnerabilities. Attacker có thể khai thác lỗ hổng để lấy quyền, thay đổi permissions, đánh cắp dữ liệu hoặc điều khiển máy.

Khuyến nghị bảo mật cơ bản:

- Bật malware protection như Windows Defender/Microsoft Defender.
- Kiểm tra services không rõ nguồn gốc.
- Mã hóa dữ liệu quan trọng, đặc biệt trên thiết bị di động.
- Thiết lập security policy phù hợp.
- Bật và rà soát firewall rules định kỳ.
- Cấu hình file/share permissions theo least privilege.
- Dùng strong passwords.
- Không đăng nhập thường xuyên bằng Administrator account.

## Ví dụ thực tế

Một máy Windows chạy chậm bất thường sau khi người dùng cài phần mềm lạ. Analyst kiểm tra Task Manager, startup items và services; phát hiện một service không rõ tên chạy bằng quyền cao. Nếu người dùng thường ngày dùng standard account thay vì administrator, phần mềm độc hại sẽ khó tự cài service hoặc thay đổi toàn hệ thống hơn.

## Cần ghi nhớ

- Windows hiện đại dựa trên NT, không còn phụ thuộc MS-DOS.
- GUI giúp dễ dùng nhưng command line vẫn rất quan trọng cho quản trị và điều tra.
- 64-bit Windows hỗ trợ address space lớn hơn nhiều.
- OS vulnerabilities là điều không tránh khỏi, nên cần patching và hardening.
- Least privilege là nguyên tắc nền tảng để giảm tác hại khi malware chạy.

# 7.4 - Windows Security

## Tổng quan

Phần này tập trung vào các công cụ và thực hành giúp giữ Windows an toàn: kiểm tra network connections bằng `netstat`, phân tích Event Viewer, quản lý Windows Update, cấu hình Local Security Policy, dùng Windows Defender và Windows Defender Firewall.

## Ý chính

- Malware thường mở ports hoặc tạo connections để gửi/nhận dữ liệu.
- `netstat -abno` giúp liên kết connections với process, executable và PID.
- Event Viewer lưu Application, Security, System events và hỗ trợ custom views.
- Windows Update giúp vá vulnerabilities và giảm rủi ro từ zero-day hoặc malware mới.
- Local Security Policy cấu hình password policy, account lockout, user rights, firewall rules và AppLocker.
- Windows Defender cung cấp bảo vệ malware/spyware tích hợp.
- Windows Defender Firewall lọc traffic vào/ra theo rule và nên được rà soát định kỳ.

## Ghi chú chi tiết

### netstat để phát hiện connections bất thường

Malware có thể mở listening port hoặc kết nối ra server điều khiển. `netstat` giúp xem connections hiện tại.

Lệnh hữu ích:

```cmd
netstat -abno
```

Ý nghĩa thường dùng:

- `-a`: hiển thị all connections và listening ports.
- `-b`: hiển thị executable liên quan.
- `-n`: hiển thị địa chỉ/port dạng số.
- `-o`: hiển thị PID.

Khi thấy connection lạ, analyst có thể đối chiếu PID trong Task Manager, kiểm tra đường dẫn executable, chữ ký số, hash, parent process và Event Logs.

### Event Viewer

Event Viewer lưu lịch sử events của system, application và security. Event có level như Information, Warning, Error, Critical. Các trường thường quan trọng:

- Date and time.
- Source.
- Event ID.
- Level.
- User hoặc computer liên quan.

Custom View giúp lọc theo thời gian, level hoặc nguồn. Administrative Events là view hữu ích để bắt đầu troubleshooting vì gom critical/error/warning từ nhiều administrative logs.

### Windows Update Management

Windows cần được cập nhật để vá vulnerabilities. Security updates, critical updates và service packs giúp giảm nguy cơ malware/worm khai thác lỗ hổng đã biết.

Thực hành tốt:

- Bật automatic updates hoặc quản lý update tập trung trong doanh nghiệp.
- Theo dõi update history.
- Đặt active hours để tránh restart giữa giờ làm việc.
- Ưu tiên vá lỗ hổng đang bị khai thác.
- Kiểm tra máy không cập nhật được.

### Local Security Policy

Local Security Policy dùng cho standalone computer hoặc cấu hình local. Trong domain, nhiều policy được áp qua Domain Security Policy/Group Policy.

Các chính sách quan trọng:

- Password Policy: độ dài, phức tạp, tuổi mật khẩu.
- Account Lockout Policy: khóa tài khoản sau nhiều lần đăng nhập sai để giảm brute-force.
- User Rights Assignment: quyền hành động như log on locally, backup files, shutdown system.
- Security Options: nhiều thiết lập bảo mật cục bộ.
- AppLocker: hạn chế file/app được phép chạy.
- Firewall rules: rule cho network traffic.

Policy có thể export/import để áp dụng nhất quán cho nhiều standalone computers.

### Windows Defender

Windows Defender/Microsoft Defender cung cấp bảo vệ malware, spyware và nhiều threat phổ biến. Công cụ này chạy nền và có thể scan thủ công.

Các lớp bảo vệ thường gặp:

- Antivirus protection.
- Adware protection.
- Phishing protection.
- Spyware/keylogger detection.
- Cảnh báo unsafe programs hoặc unsafe websites.

Lưu ý: không nên chạy nhiều antimalware real-time engines cùng lúc nếu chúng xung đột. Quan trọng là phải cập nhật definitions/security intelligence thường xuyên.

### Windows Defender Firewall

Firewall lọc traffic vào/ra máy Windows. Cách tiếp cận an toàn là restrictive: chỉ mở ports/programs cần thiết, traffic không được permit thì deny.

Quản trị viên có thể:

- Allow app hoặc feature qua firewall.
- Tạo inbound/outbound rules.
- Bật/tắt firewall theo profile như Domain, Private, Public.
- Rà soát rule cũ không còn dùng.

Không nên tắt firewall chỉ để “sửa nhanh” lỗi kết nối. Thay vào đó cần xác định application/port/protocol cần thiết và tạo rule tối thiểu.

## Ví dụ thực tế

Một máy Windows bắt đầu gửi traffic tới địa chỉ IP lạ. Analyst chạy `netstat -abno`, thấy một process lạ đang kết nối ra ngoài. Dựa vào PID, họ tìm process trong Task Manager, kiểm tra đường dẫn file, sau đó xem Event Viewer để tìm thời điểm process bắt đầu chạy. Nếu file là malware, analyst cô lập máy, chạy Defender scan, kiểm tra startup entries và vá hệ thống.

## Cần ghi nhớ

- `netstat -abno` rất hữu ích để liên kết network connection với process/PID.
- Event Viewer là nguồn dữ liệu quan trọng cho troubleshooting và incident analysis.
- Windows Update là lớp phòng thủ quan trọng chống lỗ hổng đã biết.
- Local Security Policy giúp giảm rủi ro password yếu, brute-force và chạy ứng dụng trái phép.
- Defender và Firewall nên được bật, cập nhật và rà soát định kỳ.

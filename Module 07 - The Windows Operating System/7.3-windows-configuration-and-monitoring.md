# 7.3 - Windows Configuration and Monitoring

## Tổng quan

Phần này giới thiệu các công cụ và khái niệm dùng để cấu hình, giám sát và quản lý Windows: Run as Administrator, local users/groups/domains, CLI, PowerShell, WMI, net commands, Task Manager, Resource Monitor, network configuration, SMB/UNC, RDP và Windows Server roles.

## Ý chính

- Không nên đăng nhập hằng ngày bằng Administrator account.
- Local users và groups giúp quản lý quyền trên từng máy; domain dùng DC để quản lý tập trung.
- CMD và PowerShell hỗ trợ quản trị bằng dòng lệnh; PowerShell mạnh hơn nhờ cmdlets, objects và scripting.
- WMI có thể quản lý remote computers nhưng cũng có thể bị attacker lạm dụng.
- `net` command hỗ trợ quản lý accounts, sessions, shares, services và network resources.
- Task Manager và Resource Monitor giúp theo dõi processes, services, CPU, memory, disk và network.
- Network and Sharing Center, `nslookup`, `netstat`, SMB/UNC và RDP là các thành phần quan trọng khi cấu hình và troubleshooting Windows network.

## Ghi chú chi tiết

### Run as Administrator

Best practice là dùng standard user cho công việc thường ngày. Khi cần cài đặt hoặc cấu hình hệ thống, dùng Run as Administrator cho tác vụ cụ thể. Cách này giảm nguy cơ malware thừa hưởng quyền admin toàn thời gian.

Có thể chạy bằng quyền admin bằng cách:

- Right-click ứng dụng và chọn Run as administrator.
- Mở Command Prompt/PowerShell với Run as administrator.

### Local users, groups và domains

Khi cài Windows hoặc khởi tạo máy mới, user tạo local account. Local account chứa profile, settings, permissions và dữ liệu riêng của user.

Các điểm bảo mật:

- Built-in Administrator và Guest thường bị disabled mặc định.
- Không nên bật Guest account nếu không có lý do rõ ràng.
- Không nên cho standard users vào Administrators group.
- Dùng groups để quản lý quyền thay vì gán quyền rời rạc cho từng user.

Domain dùng Domain Controller để quản lý tập trung users, computers, groups, policies và authentication. Trong môi trường doanh nghiệp, domain giúp áp Group Policy, quản lý tài khoản và chuẩn hóa security settings.

### CMD và PowerShell

CMD phù hợp với lệnh truyền thống, quản lý file/folder và chạy batch files. Một số điểm cần nhớ:

- Đường dẫn Windows dùng drive letter như `C:\`.
- File/path mặc định không phân biệt hoa thường.
- Switch thường dùng dấu `/`.
- Tab hỗ trợ auto-complete.
- Mũi tên lên/xuống gọi lại command history.

PowerShell mạnh hơn CMD vì dùng cmdlets, scripts và functions. PowerShell có thể tự động hóa tác vụ quản trị phức tạp và trả về objects thay vì chỉ text.

Ví dụ:

```powershell
Get-Help Get-Process -Examples
Get-Process
Get-Service
```

### WMI

WMI dùng để thu thập thông tin và quản lý local/remote computers: hardware, software, services, health và configuration. WMI hữu ích cho admin nhưng cũng có thể bị attacker lạm dụng để chạy lệnh từ xa, thay đổi Registry hoặc tránh bị phát hiện.

Khuyến nghị:

- Giới hạn quyền truy cập WMI.
- Ghi log và giám sát remote WMI activity.
- Không mở WMI rộng trong mạng nếu không cần.

### The net command

`net` là nhóm lệnh quản trị Windows quan trọng.

Ví dụ các lệnh thường gặp:

- `net accounts`: đặt yêu cầu password/logon.
- `net user`: quản lý users.
- `net localgroup`: quản lý local groups.
- `net share`: quản lý shared resources.
- `net use`: kết nối/ngắt kết nối network shares.
- `net session`: xem hoặc ngắt sessions.
- `net start`: liệt kê hoặc khởi động services.
- `net stop`: dừng service.
- `net view`: xem computers/devices trên network.

### Task Manager và Resource Monitor

Task Manager cung cấp cái nhìn nhanh về:

- Processes: ứng dụng và processes đang chạy, CPU/memory/disk/network usage.
- Performance: tổng quan CPU, memory, disk, network.
- App history: tài nguyên ứng dụng dùng theo thời gian.
- Startup: programs khởi động cùng Windows.
- Users: users đang đăng nhập và tài nguyên họ dùng.
- Details: PID, priority, affinity và thông tin chi tiết process.
- Services: trạng thái services.

Resource Monitor chi tiết hơn:

- CPU: threads, handles, modules, services.
- Memory: working set, committed, hard faults.
- Disk: read/write per process.
- Network: TCP connections, listening ports, process network activity.

Resource Monitor đặc biệt hữu ích khi nghi ngờ process lạ đang kết nối network.

### Networking tools

Network and Sharing Center dùng để xem active network, loại network, adapter settings và troubleshoot. Adapter properties cho phép cấu hình IPv4/IPv6, DHCP hoặc static IP.

Các lệnh hữu ích:

```cmd
nslookup example.com
netstat
netsh /?
```

- `nslookup`: kiểm tra DNS resolution.
- `netstat`: xem active connections và listening ports.
- `netsh`: xem/sửa network configuration từ command line.

### SMB, UNC và RDP

SMB dùng để chia sẻ files và network resources. UNC path có dạng:

```text
\\servername\sharename\file
```

Administrative shares như `C$`, `ADMIN$`, `print$` chỉ dành cho users có quyền admin.

RDP cho phép điều khiển Windows host từ xa. Vì RDP là mục tiêu phổ biến của attackers, cần:

- Hạn chế expose RDP ra internet.
- Dùng VPN/ZTNA hoặc jump host.
- Bật MFA nếu có.
- Patch hệ thống.
- Giới hạn user được phép RDP.
- Theo dõi log đăng nhập.

### Windows Server roles

Windows Server cung cấp nhiều services:

- Network services: DNS, DHCP, Terminal Services, Hyper-V Network Virtualization.
- File services: SMB, NFS, DFS.
- Web services: FTP, HTTP, HTTPS.
- Management: Group Policy, Active Directory Domain Services.

## Ví dụ thực tế

Một máy Windows có dấu hiệu gửi traffic lạ ra ngoài. Analyst mở Resource Monitor để xem process nào có TCP connection bất thường, dùng Task Manager để lấy PID, dùng `netstat -abno` trong Command Prompt admin để đối chiếu executable, sau đó kiểm tra Event Viewer và startup entries để xem process có tự chạy sau reboot hay không.

## Cần ghi nhớ

- Run as Administrator chỉ nên dùng khi cần thiết.
- Domain giúp quản lý identity và policies tập trung hơn local accounts.
- PowerShell mạnh, hữu ích cho admin nhưng cũng cần kiểm soát vì attacker có thể lạm dụng.
- WMI và RDP cần được giới hạn và giám sát.
- Task Manager, Resource Monitor, netstat và Event Viewer là bộ công cụ quan trọng khi điều tra Windows endpoint.

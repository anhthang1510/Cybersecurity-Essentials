# 7.2 - Windows Architecture and Operations

## Tổng quan

Phần này giải thích kiến trúc và cách Windows vận hành: từ Hardware Abstraction Layer, kernel, user mode/kernel mode, file systems, boot process, startup/shutdown, processes/threads/services đến memory, handles và Windows Registry. Đây là nền tảng quan trọng để hiểu cách Windows hoạt động và cách attacker có thể lạm dụng các thành phần hệ thống.

## Ý chính

- HAL giúp Windows tương tác với nhiều loại hardware khác nhau.
- Kernel kiểm soát input/output, memory, peripherals và các tài nguyên cốt lõi.
- User mode cô lập ứng dụng; kernel mode có quyền truy cập sâu hơn và rủi ro cao hơn khi lỗi.
- NTFS là file system phổ biến của Windows, hỗ trợ permissions, timestamps, recovery và encryption.
- Alternate Data Streams có thể bị lạm dụng để ẩn dữ liệu hoặc mã độc.
- Boot process gồm BIOS/UEFI, POST, MBR/EFI files, Bootmgr, BCD, Winload/Winresume, kernel, HAL, SMSS và logon.
- Registry lưu cấu hình hardware, software, users và system settings.

## Ghi chú chi tiết

### HAL, kernel và drivers

Windows phải chạy trên nhiều loại phần cứng. HAL giúp che bớt sự khác biệt giữa hardware và kernel, để OS có thể tương tác với CPU, memory, storage, bus và peripherals theo cách nhất quán hơn.

Kernel là phần lõi của Windows. Nó xử lý memory management, I/O requests, scheduling, device communication và nhiều chức năng cốt lõi khác. Drivers là cầu nối giữa OS và thiết bị. Driver có thể chạy ở user mode hoặc kernel mode tùy loại.

### User mode và kernel mode

Windows tách quyền thực thi thành hai vùng chính:

- User mode: ứng dụng chạy trong address space riêng, không trực tiếp truy cập hardware hoặc kernel memory.
- Kernel mode: OS code và kernel drivers có quyền cao, có thể truy cập hardware và memory trực tiếp.

Ý nghĩa bảo mật:

- Lỗi trong user mode thường chỉ làm crash ứng dụng.
- Lỗi trong kernel mode có thể làm crash toàn hệ thống.
- Kernel-mode malware/rootkit nguy hiểm hơn vì có thể ẩn sâu và can thiệp vào OS.

### File systems và NTFS

Windows có thể tương tác với nhiều file systems như FAT/exFAT, NTFS và đọc một số file systems khác bằng phần mềm hỗ trợ. Trong Windows hiện đại, NTFS là file system quan trọng nhất.

NTFS hỗ trợ:

- Large files và large partitions.
- File/folder ownership.
- Permissions qua security descriptors.
- Recovery features.
- Timestamps hữu ích cho forensics.
- Encryption trong một số cấu hình.

Các thành phần NTFS quan trọng:

- Partition Boot Sector: chứa thông tin boot và vị trí MFT.
- Master File Table: metadata cho files/folders.
- System Files: files ẩn chứa thông tin hệ thống file.
- File Area: vùng lưu dữ liệu chính.

Timestamps thường được nhắc trong forensics gồm Modify, Access, Create và Entry Modified. Các thời điểm này giúp dựng lại lịch sử file/folder.

### Alternate Data Streams

NTFS lưu nội dung file trong data stream. ADS cho phép gắn thêm stream khác vào cùng một file, ví dụ dạng `file.txt:streamname`. Ứng dụng hợp pháp có thể dùng ADS để lưu metadata, nhưng attacker cũng có thể ẩn dữ liệu hoặc payload trong ADS.

Ví dụ minh họa:

```cmd
echo "hidden note" > report.txt:secret
dir
dir /r
more < report.txt:secret
```

Ý nghĩa: `dir` thông thường có thể không hiển thị ADS, trong khi `dir /r` giúp thấy stream phụ.

### Windows boot process

Windows boot có thể bắt đầu từ BIOS hoặc UEFI.

Với BIOS:

- BIOS initialization.
- POST kiểm tra hardware.
- Tìm và chạy MBR.
- Boot process tiếp tục tải Windows.

Với UEFI:

- UEFI initialization.
- Tải EFI files từ EFI System Partition.
- Hỗ trợ boot hiện đại và security tốt hơn.

Sau đó, Windows dùng Bootmgr, BCD, Winload hoặc Winresume, kiểm tra driver signing, tải kernel `ntoskrnl.exe`, thiết lập HAL, chạy SMSS và chuẩn bị môi trường logon.

### Startup và shutdown

Startup items có thể nằm trong Registry hoặc được quản lý qua tools như System Configuration (`msconfig`) và Task Manager. Các vị trí startup là điểm quan trọng khi kiểm tra malware persistence.

Shutdown đúng cách giúp OS đóng applications, services và ghi thay đổi cấu hình. Tắt nguồn đột ngột có thể gây hỏng file, mất dữ liệu hoặc làm service dừng sai thứ tự.

### Processes, threads và services

Process là chương trình đang chạy. Mỗi process có ít nhất một thread. Thread là đơn vị được CPU thực thi. Windows multitasking cho phép nhiều threads/processes chạy luân phiên hoặc song song tùy CPU cores.

Services là chương trình nền có thể tự khởi động cùng Windows, chạy thủ công hoặc bị disabled. Một số services rất quan trọng; tắt sai service có thể làm ứng dụng hoặc OS hoạt động lỗi.

### Memory allocation và handles

Mỗi process có virtual address space riêng. User-mode process không được truy cập trực tiếp kernel resources. Khi cần truy cập tài nguyên hệ thống, process dùng handles.

Công cụ như RAMMap trong Sysinternals giúp xem Windows phân bổ memory cho processes, drivers, kernel và applications.

### Windows Registry

Registry là database phân cấp lưu cấu hình Windows. Cấu trúc gồm hives, keys, subkeys và values.

Các hives quan trọng:

- HKEY_CURRENT_USER (HKCU): thông tin user đang đăng nhập.
- HKEY_USERS (HKU): thông tin tất cả user accounts trên host.
- HKEY_CLASSES_ROOT (HKCR): thông tin file association, OLE và COM-related registrations.
- HKEY_LOCAL_MACHINE (HKLM): thông tin system-wide về hardware/software.
- HKEY_CURRENT_CONFIG (HKCC): hardware profile hiện tại.

Registry có giá trị lớn trong security và forensics vì có thể chứa startup entries, user activity, device history, opened documents và cấu hình applications. Tuy nhiên chỉnh sai Registry có thể làm hệ thống lỗi nghiêm trọng.

## Ví dụ thực tế

Một analyst nghi ngờ malware tự chạy sau khi reboot. Họ kiểm tra Task Manager Startup, `msconfig`, services và Registry keys liên quan đến Run/RunOnce trong HKLM/HKCU. Nếu phát hiện một executable lạ trong startup location, analyst có thể cô lập file, kiểm tra hash, xem Event Logs và xác định user/process đã tạo entry đó.

## Cần ghi nhớ

- User mode giúp cô lập ứng dụng; kernel mode có quyền cao và rủi ro lớn hơn.
- NTFS hỗ trợ permissions và forensics metadata quan trọng.
- ADS có thể hữu ích nhưng cũng là nơi attacker ẩn dữ liệu.
- Boot process là chuỗi nhiều bước, có liên quan đến firmware, boot manager, BCD, kernel và drivers.
- Registry là mục tiêu quan trọng khi kiểm tra persistence và cấu hình hệ thống.

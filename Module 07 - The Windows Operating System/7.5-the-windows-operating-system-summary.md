# 7.5 - The Windows Operating System Summary

## Tổng quan

Phần tổng kết nhắc lại các nội dung chính của Module 7: lịch sử Windows, kiến trúc và vận hành hệ điều hành, công cụ cấu hình/giám sát, và các lớp bảo mật tích hợp. Với cybersecurity, Windows cần được hiểu từ cả góc nhìn người dùng, quản trị viên và analyst điều tra sự cố.

## Ý chính

- Windows phát triển từ môi trường DOS/GUI đời đầu sang kiến trúc NT hiện đại.
- Windows NT hỗ trợ nhiều users/processes, security model và quản lý hardware trực tiếp.
- HAL, kernel, drivers, user mode và kernel mode là nền tảng kiến trúc Windows.
- NTFS hỗ trợ permissions, timestamps, recovery và ADS.
- Boot process liên quan đến BIOS/UEFI, boot manager, BCD, kernel, HAL và user logon.
- Processes, threads, services, memory, handles và Registry là các thành phần quan trọng khi phân tích hệ thống.
- Công cụ quản trị gồm CMD, PowerShell, WMI, Task Manager, Resource Monitor, Network and Sharing Center, Event Viewer.
- Windows security dựa trên least privilege, updates, Defender, firewall, local policy, logs và monitoring.

## Ghi chú chi tiết

### Windows History

Windows bắt đầu từ giai đoạn DOS/MS-DOS, nơi command line là cách chính để quản lý file và chương trình. Windows GUI giúp người dùng thao tác dễ hơn, nhưng Windows hiện đại không còn là GUI chạy trên DOS. Windows NT đem lại kiến trúc mạnh hơn với multitasking, nhiều users/processes và mô hình bảo mật phù hợp hơn cho doanh nghiệp.

### Windows Architecture and Operations

Windows dùng HAL để xử lý sự khác biệt hardware, kernel để điều phối tài nguyên và user/kernel mode để tách quyền thực thi. NTFS cung cấp file permissions và metadata hữu ích cho forensics. Registry lưu cấu hình hệ thống và là điểm quan trọng khi kiểm tra persistence. Boot process và startup entries giúp analyst hiểu malware có thể bám vào hệ thống ở đâu.

### Windows Configuration and Monitoring

Admin có thể dùng CMD, PowerShell, WMI, Task Manager, Resource Monitor, Network and Sharing Center và các lệnh như `net`, `netstat`, `nslookup`, `netsh`. Những công cụ này giúp tạo users, quản lý services, xem performance, kiểm tra network connections, truy cập shares và điều tra sự cố.

### Windows Security

Bảo mật Windows cần nhiều lớp:

- Dùng standard user cho công việc thường ngày.
- Chỉ Run as Administrator khi cần.
- Tắt hoặc hạn chế Guest/Administrator built-in accounts.
- Cấu hình password policy và account lockout.
- Bật Windows Defender và cập nhật signatures.
- Bật Windows Defender Firewall và chỉ mở rule cần thiết.
- Cài Windows Updates đều đặn.
- Theo dõi Event Viewer và network connections.
- Hạn chế RDP/WMI và giám sát truy cập từ xa.

## Ví dụ thực tế

Trong một sự cố nghi malware trên Windows endpoint, analyst có thể bắt đầu bằng Task Manager để xem process lạ, Resource Monitor để kiểm tra network activity, `netstat -abno` để lấy PID/executable, Event Viewer để xác định thời điểm xuất hiện, Registry để kiểm tra startup persistence, Defender để scan, và Windows Update để xác nhận máy đã được vá đầy đủ.

## Cần ghi nhớ

- Hiểu Windows architecture giúp điều tra sự cố nhanh và chính xác hơn.
- Registry, services, startup items và scheduled/background processes là các vị trí cần chú ý.
- Event logs có giá trị lớn cho timeline và attribution nội bộ.
- Defender/Firewall/Update chỉ hiệu quả khi được bật và quản lý đúng.
- RDP, WMI, admin privileges và weak passwords là các điểm rủi ro phổ biến.

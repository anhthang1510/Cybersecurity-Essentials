# 6.1 - Security Devices

## Tổng quan

Phần này trình bày các thiết bị bảo mật chuyên dụng trong mạng, gồm firewall, IDS/IPS và security appliances. Các thiết bị này giúp thực thi policy, kiểm soát traffic, phát hiện tấn công, ngăn traffic nguy hiểm và bảo vệ trước malware, web threats hoặc email threats.

## Ý chính

- Firewall thực thi access control policy giữa các vùng mạng.
- Kiến trúc firewall thường dựa trên inside/outside networks, DMZ hoặc zone-based policies.
- Các loại firewall khác nhau hoạt động ở các tầng khác nhau, từ packet filtering đến application gateway và next-generation firewall.
- IDS phát hiện và cảnh báo; IPS phát hiện và chặn traffic nguy hiểm.
- IDS/IPS dùng signatures để phát hiện atomic hoặc composite attack patterns.
- HIPS bảo vệ từng host; NIPS bảo vệ traffic tại các điểm chiến lược trong mạng.
- Specialized security appliances hỗ trợ malware protection, web security và email security.

## Ghi chú chi tiết

### Firewall operation

Firewall là điểm kiểm soát giữa các mạng có mức tin cậy khác nhau. Nó dùng rule hoặc policy để cho phép, chặn và ghi log traffic dựa trên thông tin như source, destination, protocol, port, state hoặc application.

Các đặc điểm chung của firewall:

- Chống chịu tốt hơn trước network attacks so với thiết bị thông thường.
- Là điểm transit giữa internal network và external network nếu thiết kế đúng.
- Thực thi access control policy.
- Có thể log traffic để hỗ trợ audit và troubleshooting.

Lợi ích:

- Giảm bề mặt tấn công.
- Chặn traffic không mong muốn.
- Tách vùng mạng có mức tin cậy khác nhau.
- Hỗ trợ kiểm soát truy cập vào dịch vụ public-facing.

Giới hạn:

- Không chặn được threat đi qua kênh được phép nếu không có inspection phù hợp.
- Không bảo vệ tốt nếu rule sai hoặc quá rộng.
- Không thay thế endpoint protection, user training, patching hoặc monitoring.
- Có thể bị bypass nếu có đường đi khác không đi qua firewall.

### Common security architectures

Private/Public design là mô hình đơn giản nhất: inside network được tin cậy hơn, outside network không tin cậy. Traffic từ inside ra outside có thể được cho phép và theo dõi; traffic từ outside vào inside thường bị chặn trừ khi có policy rõ ràng.

DMZ đặt các server cần phục vụ internet, ví dụ web, mail hoặc DNS public, vào vùng trung gian. Mục tiêu là nếu server trong DMZ bị compromise, attacker không tự động có quyền vào internal network.

Zone-based policy firewall chia mạng thành nhiều zones như inside, outside, DMZ, guest, management. Policy được áp theo cặp zone, giúp kiểm soát rõ hơn so với chỉ có inside/outside.

### Firewall types

Packet filtering firewall lọc theo Layer 3/4 như source IP, destination IP, protocol, source port, destination port hoặc TCP flags. Ưu điểm là đơn giản, nhanh; nhược điểm là không hiểu sâu ứng dụng.

Stateful firewall theo dõi trạng thái phiên. Ví dụ, nó có thể cho phép traffic phản hồi từ internet nếu đó là response cho session do host nội bộ khởi tạo.

Application gateway/proxy firewall đại diện client kết nối tới server hoặc kiểm tra traffic ở tầng ứng dụng. Nó có thể hiểu protocol/application tốt hơn nhưng thường cần nhiều tài nguyên hơn.

Next-generation firewall mở rộng firewall truyền thống với application awareness, user awareness, integrated IPS, malware protection, URL filtering và threat intelligence feed.

Host-based firewall chạy trên PC/server để lọc traffic vào/ra chính host đó. Transparent firewall lọc traffic giữa các bridged interfaces mà không cần thay đổi IP addressing. Hybrid firewall kết hợp nhiều khả năng, ví dụ stateful inspection cộng application inspection.

### IDS và IPS

IDS và IPS đều dùng sensors và signatures để phát hiện patterns trong traffic. Một signature có thể nhận diện:

- Atomic pattern: dấu hiệu trong một packet.
- Composite pattern: chuỗi dấu hiệu qua nhiều packets hoặc nhiều bước.

IDS thường đặt offline hoặc out-of-band. Nó phân tích bản sao traffic và cảnh báo. Ưu điểm là ít ảnh hưởng latency/jitter và nếu sensor lỗi thì traffic vẫn đi qua. Nhược điểm là trigger packet có thể đã tới đích trước khi admin phản ứng.

IPS thường đặt inline. Traffic đi qua IPS trước khi tới đích, nên IPS có thể drop packet, reset connection hoặc gửi log. Ưu điểm là chặn được traffic nguy hiểm nhanh hơn. Nhược điểm là sensor lỗi, overload hoặc false positive có thể ảnh hưởng traffic hợp lệ.

### HIPS và NIPS

HIPS là phần mềm trên host, giám sát OS, application và behavior. Nó có thể phát hiện registry changes, system directory changes, unauthorized process execution, buffer overflow behavior hoặc host tham gia DoS/illicit sessions. HIPS có lợi vì hiểu rõ host nhưng phải triển khai trên từng endpoint và phụ thuộc OS.

NIPS triển khai tại các điểm trong mạng, ví dụ giữa firewall và server farm, trước DMZ hoặc trong internal segmentation. NIPS quan sát nhiều traffic hơn HIPS nhưng có thể không thấy dữ liệu sau khi đã được mã hóa end-to-end nếu không có giải pháp inspection phù hợp.

### Specialized security appliances

Security appliances thường tập trung vào một nhóm threats:

- Advanced Malware Protection: phân tích file, phát hiện malware trước, trong và sau khi tấn công.
- Web Security Appliance: kiểm soát web traffic, URL filtering, malware scanning và policy enforcement.
- Email Security Appliance: lọc spam, phishing, malicious attachments và email-borne malware.

Các appliances hiện đại thường dùng threat intelligence để cập nhật dấu hiệu tấn công mới và liên kết events trên nhiều nguồn.

## Ví dụ thực tế

Một doanh nghiệp đặt web server public trong DMZ, database server trong internal network và chỉ cho web server kết nối database qua port cần thiết. Firewall chặn traffic từ internet vào database, IPS theo dõi traffic vào DMZ, WSA kiểm soát web browsing của nhân viên, ESA lọc email phishing và Syslog/SIEM tập trung log. Nếu web server bị tấn công, segmentation và firewall policy giúp giảm khả năng attacker đi sâu vào mạng nội bộ.

## Cần ghi nhớ

- Firewall cần được đặt ở đường đi bắt buộc của traffic, nếu không attacker có thể bypass.
- DMZ giúp tách public-facing servers khỏi internal network.
- IDS cảnh báo; IPS có thể chặn inline.
- IPS mạnh nhưng phải tune kỹ để tránh false positive làm gián đoạn dịch vụ.
- Security appliances bổ sung năng lực bảo vệ chuyên sâu cho malware, web và email.

# 15.1 - Secure Networks with Firewalls

## Tổng quan

Phần này giải thích cách firewall giúp bảo vệ mạng bằng cách kiểm soát traffic. Firewall có thể lọc packet theo thông tin header, theo dõi trạng thái kết nối, kiểm tra ứng dụng hoặc kết hợp nhiều kỹ thuật để giảm rủi ro xâm nhập.

## Ý chính

- Firewall nằm giữa các vùng có mức tin cậy khác nhau, ví dụ internet và internal LAN.
- Chính sách firewall thường xét source, destination, service, direction và action.
- Packet filtering phù hợp với lọc cơ bản nhưng không hiểu toàn bộ ngữ cảnh phiên.
- Stateful inspection theo dõi session để cho phép return traffic hợp lệ và chặn traffic bất thường.
- Application/proxy firewall có thể kiểm soát sâu hơn ở tầng ứng dụng.
- NGFW kết hợp nhiều tính năng như stateful inspection, application awareness, IDS/IPS, URL filtering và malware protection.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Access control rule | Quy tắc cho phép hoặc chặn traffic dựa trên điều kiện cụ thể. |
| Source | Địa chỉ hoặc vùng mạng bắt đầu traffic. |
| Destination | Địa chỉ hoặc vùng mạng nhận traffic. |
| Service | Dịch vụ hoặc port/protocol được kiểm soát, ví dụ HTTP, HTTPS, DNS. |
| Inbound traffic | Traffic đi từ bên ngoài vào mạng được bảo vệ. |
| Outbound traffic | Traffic đi từ mạng nội bộ ra bên ngoài. |
| Stateful inspection | Cơ chế kiểm tra traffic dựa trên trạng thái phiên kết nối. |
| Application firewall | Firewall có khả năng hiểu hoặc kiểm soát hành vi ở tầng ứng dụng. |
| Proxy | Thành phần đứng trung gian giữa client và server, có thể kiểm tra và thay mặt truyền traffic. |
| NGFW | Firewall thế hệ mới tích hợp nhiều năng lực bảo mật nâng cao. |

## Ghi chú chi tiết

**Firewall như một điểm thực thi policy:**

Firewall không tự biết traffic nào là hợp lệ nếu không có policy đúng. Người quản trị phải xác định vùng nào được nói chuyện với vùng nào, dùng dịch vụ nào và trong điều kiện nào. Càng mô tả rule rõ ràng, bề mặt tấn công càng nhỏ.

**Packet filtering:**

Packet filtering kiểm tra thông tin như source IP, destination IP, protocol và port. Cách này nhanh, dễ triển khai và phù hợp với những rule đơn giản. Tuy nhiên, nếu firewall không theo dõi trạng thái, nó có thể khó phân biệt packet phản hồi hợp lệ với packet giả mạo trông giống hợp lệ.

**Stateful inspection:**

Stateful firewall tạo session table để ghi nhận kết nối đã được cho phép. Khi return traffic quay lại, firewall kiểm tra xem traffic đó có thuộc session hợp lệ hay không. Nhờ vậy, chính sách có thể cho phép client nội bộ truy cập web bên ngoài mà không cần mở inbound rule quá rộng.

**Application và proxy firewall:**

Application firewall hiểu ngữ cảnh ở tầng ứng dụng tốt hơn packet filter. Proxy firewall có thể đóng vai trò trung gian, nhận request từ client, kiểm tra nội dung, rồi gửi request tới server. Điều này hữu ích khi cần kiểm soát HTTP, web application hoặc dịch vụ có rủi ro cao.

**NGFW:**

Next-Generation Firewall thường kết hợp firewall truyền thống với application identification, IPS, URL filtering, malware inspection và threat intelligence. NGFW phù hợp với môi trường cần nhận diện ứng dụng thực tế thay vì chỉ dựa trên port, vì nhiều ứng dụng hiện đại có thể dùng chung HTTP/HTTPS.

## Ví dụ thực tế

**Ví dụ 1: Cho phép user truy cập web an toàn**

Một doanh nghiệp cho phép user network truy cập internet qua HTTPS. Stateful firewall ghi nhận session khi user mở kết nối ra ngoài. Khi web server trả lời, return traffic được cho phép vì thuộc session hợp lệ, còn kết nối lạ từ internet vào user PC vẫn bị chặn.

**Ví dụ 2: Kiểm soát dịch vụ public trong DMZ**

Web server public được đặt trong DMZ. Firewall chỉ cho internet truy cập TCP 443 tới web server, chặn SSH từ internet và chỉ cho admin network quản trị server qua kênh riêng. Nếu web server bị tấn công, attacker vẫn không tự do đi vào internal database network.

**Ví dụ 3: NGFW nhận diện ứng dụng**

Hai ứng dụng đều dùng HTTPS, nhưng một ứng dụng là business SaaS hợp lệ, ứng dụng còn lại là file-sharing không được phép. NGFW có thể dùng application awareness để áp policy khác nhau thay vì chỉ nhìn port 443.

## Cần ghi nhớ

- **Firewall rule cần cụ thể**: source, destination, service và action càng rõ thì càng dễ kiểm soát.
- **Stateless filtering có giới hạn**: nó không hiểu đầy đủ trạng thái kết nối.
- **Stateful firewall theo dõi session**: return traffic hợp lệ được cho phép mà không cần mở inbound quá rộng.
- **Application-aware control quan trọng**: port không luôn phản ánh đúng ứng dụng đang chạy.
- **Firewall không thay thế endpoint security**: máy trạm và server vẫn cần hardening, patching và monitoring.

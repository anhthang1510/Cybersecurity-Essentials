# 9.3 Host-Based Intrusion Prevention

## Tổng quan

Host based intrusion prevention tập trung bảo vệ từng endpoint bằng firewall, HIDS, HIPS, policy, log và alert. Host firewall kiểm soát inbound và outbound traffic theo rule. HIDS phát hiện dấu hiệu đáng ngờ trên host. HIPS có thể chặn hành vi dựa trên policy hoặc detection rule.

Các công cụ này quan trọng vì endpoint có thể rời mạng nội bộ, kết nối Wi Fi công cộng hoặc bị tấn công từ bên trong. Bảo vệ ở mức host giúp giảm phụ thuộc vào perimeter firewall.

## Ý chính

* Host firewall kiểm soát traffic theo profile, address, protocol và port.
* Log firewall thường có timestamp, allow hoặc block action, source và destination IP, port.
* Windows Defender Firewall, iptables, nftables, TCP Wrappers và distributed firewall là các ví dụ liên quan.
* HIDS theo dõi config, application, log, integrity, policy, rootkit và alert.
* HIDS thường có agent trên endpoint và manager tập trung.
* Detection có thể dựa trên signature, anomaly hoặc policy.
* OSSEC là ví dụ HIDS open source có manager, agent, syslog, log monitoring, integrity check và rootkit detection.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| Host firewall | Firewall chạy trên endpoint để kiểm soát traffic của chính host đó |
| Inbound traffic | Traffic đi vào endpoint |
| Outbound traffic | Traffic đi ra từ endpoint |
| Firewall rule | Quy tắc cho phép hoặc chặn traffic theo điều kiện |
| HIDS | Host Intrusion Detection System, phát hiện dấu hiệu xâm nhập trên host |
| HIPS | Host Intrusion Prevention System, phát hiện và có thể chặn hành vi trên host |
| Agent | Thành phần cài trên endpoint để thu thập dữ liệu hoặc thực thi policy |
| Manager | Thành phần tập trung nhận log, quản lý rule và tạo alert |
| Signature based detection | Phát hiện dựa trên mẫu đã biết |
| Anomaly based detection | Phát hiện lệch khỏi baseline hành vi bình thường |
| Policy based detection | Phát hiện hoặc chặn dựa trên quy tắc chính sách định nghĩa trước |
| File integrity monitoring | Theo dõi thay đổi trái phép trên file quan trọng |

## Ghi chú chi tiết

### Host firewall

Host firewall có thể áp dụng profile khác nhau cho domain, private hoặc public network. Rule có thể dựa trên source address, destination address, protocol, port hoặc application. Log allow và block giúp analyst hiểu traffic nào bị chặn và traffic nào được phép.

### HIDS và HIPS

HIDS chủ yếu phát hiện và cảnh báo. HIPS tiến thêm một bước bằng cách chặn hoặc can thiệp. Trong thực tế, nhiều sản phẩm kết hợp cả hai. Khái niệm HIPDS đôi khi được dùng để chỉ hệ thống vừa phát hiện vừa ngăn chặn trên host.

### Các kiểu detection

Signature detection nhanh và rõ với threat đã biết nhưng yếu trước zero day hoặc polymorphic malware. Anomaly detection phát hiện hành vi lạ so với baseline nhưng dễ có false positive nếu baseline không tốt. Policy based detection rõ ràng với môi trường kiểm soát chặt, ví dụ không cho PowerShell chạy script từ thư mục tạm.

### OSSEC và công cụ tương tự

OSSEC dùng agent gửi dữ liệu về manager, hỗ trợ log monitoring, syslog, file integrity, rootkit detection và có thể trigger script phản ứng. Các sản phẩm thương mại hoặc nền tảng như Cisco AMP, AlienVault USM và Tripwire cũng cung cấp nhiều khả năng giám sát endpoint hoặc integrity.

## Ví dụ liên quan

Một endpoint bỗng tạo outbound connection đến IP lạ trên port hiếm. Host firewall log ghi action allow, source port, destination IP và timestamp. HIDS đồng thời phát hiện file hệ thống bị sửa. Hai nguồn dữ liệu này giúp SOC ưu tiên điều tra host đó.

## Cần ghi nhớ

Host based controls tạo lớp phòng thủ ngay trên thiết bị. Chúng không thay thế network security, nhưng giúp phát hiện và chặn hành vi khi endpoint không còn nằm sau perimeter quen thuộc.

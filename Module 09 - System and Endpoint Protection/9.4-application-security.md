# 9.4 Application Security

## Tổng quan

Application security tập trung giảm lỗ hổng và bề mặt tấn công của phần mềm. Attack surface không chỉ gồm network port, mà còn có code, dependency, user behavior, cloud service, mobile app, IoT, BYOD và quy trình con người. Khi bề mặt tấn công mở rộng, tổ chức cần kiểm soát ứng dụng được phép chạy, dùng threat intelligence và phân tích file nghi ngờ trong sandbox.

Allowlist và blocklist là hai cách kiểm soát phần mềm hoặc tài nguyên. Sandboxing giúp chạy file hoặc app nghi ngờ trong môi trường cô lập để quan sát hành vi mà không làm nguy hiểm hệ thống thật.

## Ý chính

* Vulnerability là điểm yếu có thể bị khai thác.
* Attack surface gồm network, software và human attack surface.
* IoT, BYOD, cloud và mobile mở rộng bề mặt tấn công.
* Blocklist chặn thứ đã biết là xấu, allowlist chỉ cho thứ đã phê duyệt chạy.
* Baseline và security intelligence giúp quyết định app hoặc domain nào đáng tin.
* Sandboxing hỗ trợ static analysis và dynamic analysis.
* IOC có thể gồm domain, IP, hash, URL, registry key, command hoặc YARA rule.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| Vulnerability | Điểm yếu trong hệ thống, app hoặc quy trình có thể bị khai thác |
| Attack surface | Tổng các điểm attacker có thể tương tác hoặc khai thác |
| Network attack surface | Bề mặt tấn công qua port, protocol, service và traffic |
| Software attack surface | Bề mặt tấn công từ code, dependency, API, plugin và cấu hình |
| Human attack surface | Rủi ro đến từ hành vi, nhận thức và quy trình của con người |
| Blocklist | Danh sách đối tượng bị chặn vì đã biết hoặc bị đánh giá là xấu |
| Allowlist | Danh sách đối tượng được phép, mọi thứ ngoài danh sách bị chặn |
| Threat intelligence | Thông tin về threat, IOC, TTP và hạ tầng độc hại |
| Sandbox | Môi trường cô lập để chạy hoặc phân tích đối tượng nghi ngờ |
| Static analysis | Phân tích file không cần chạy file |
| Dynamic analysis | Chạy file trong môi trường kiểm soát để quan sát hành vi |
| IOC | Indicator of Compromise, dấu hiệu gợi ý compromise |
| C2 | Command and Control, kênh điều khiển giữa malware và attacker |

## Ghi chú chi tiết

### Attack surface

Một ứng dụng web có attack surface ở form đăng nhập, API, file upload, dependency, server config và cả người vận hành. Thiết bị IoT, ứng dụng mobile, cloud storage và BYOD làm attack surface khó kiểm soát hơn vì dữ liệu và truy cập nằm ngoài máy chủ truyền thống.

### Allowlist và blocklist

Blocklist dễ triển khai cho domain, hash hoặc app đã biết là xấu, nhưng luôn chạy sau attacker một bước. Allowlist chặt hơn vì chỉ cho app, script hoặc publisher đã phê duyệt chạy. Đổi lại, allowlist cần baseline tốt và quy trình cập nhật để không cản trở công việc hợp lệ.

### Security intelligence

Cisco Talos, Spamhaus và các nguồn threat intelligence khác có thể giúp đánh giá IP, domain, spam source, malware campaign hoặc IOC. Windows Group Policy có thể hỗ trợ kiểm soát app và cấu hình trên endpoint Windows trong môi trường domain.

### Sandboxing và IOC

Sandbox có thể hiển thị Details, Relations, Behavior, process tree, PowerShell command, obfuscated script, HTTP request, domain, IP và hash. Analyst không nên mở URL độc hại trực tiếp trên máy thật. Phân tích nên thực hiện trong môi trường cô lập hoặc công cụ chuyên dụng.

## Ví dụ liên quan

Một file Office nghi ngờ chứa macro được gửi qua email. Static analysis có thể xem metadata và chuỗi đáng ngờ. Dynamic analysis trong sandbox có thể cho thấy macro gọi PowerShell, tải payload từ domain lạ và tạo persistence. Các domain, IP, hash và command thu được trở thành IOC để săn tìm trên hệ thống khác.

## Cần ghi nhớ

Application security là giảm cơ hội attacker tương tác với điểm yếu. Kiểm soát ứng dụng, giới hạn script, dùng threat intelligence và sandboxing giúp tổ chức phát hiện sớm hành vi nguy hiểm mà không đặt endpoint thật vào rủi ro.

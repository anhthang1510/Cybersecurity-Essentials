# 20.3 - Network Logs

## Tổng quan

Network logs là dữ liệu từ thiết bị và công nghệ mạng như firewall, router, switch, IDS/IPS, DNS, DHCP, VPN, proxy và NetFlow/IPFIX. Chúng giúp analyst hiểu traffic đi đâu, bị cho phép hay chặn, dùng protocol nào và có dấu hiệu bất thường không.

## Ý chính

- Firewall logs cho biết traffic allow/deny, policy hit, NAT và session.
- IDS/IPS alerts cho biết traffic khớp signature hoặc anomaly.
- DNS logs giúp phát hiện domain độc hại, tunneling hoặc beaconing.
- DHCP logs ánh xạ IP với thiết bị theo thời gian.
- VPN logs ghi nhận remote access sessions, user, source IP và duration.
- Proxy logs cung cấp URL, user, category, action và web policy.
- NetFlow/IPFIX giúp phân tích traffic volume, direction và pattern.
- Network device logs từ router/switch giúp phát hiện link changes, config changes hoặc routing issues.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Firewall event | Sự kiện allow/deny hoặc policy match trên firewall. |
| NAT log | Log ánh xạ địa chỉ trước/sau NAT. |
| IDS alert | Cảnh báo traffic nghi ngờ tấn công. |
| DNS log | Bản ghi truy vấn DNS và phản hồi. |
| DHCP log | Bản ghi cấp phát địa chỉ IP. |
| VPN log | Log kết nối remote access hoặc site-to-site VPN. |
| Proxy log | Log truy cập web qua proxy/web gateway. |
| NetFlow/IPFIX | Flow telemetry mô tả metadata kết nối. |
| Config change log | Log thay đổi cấu hình thiết bị. |

## Ghi chú chi tiết

**Firewall logs:**

Firewall logs rất hữu ích để hiểu traffic có bị chặn hay được cho phép. Tuy nhiên, một allow log không có nghĩa traffic an toàn; nó chỉ nói policy cho phép. Cần kết hợp với IDS/IPS, endpoint logs hoặc application logs để đánh giá rủi ro.

**DNS và DHCP logs:**

DNS cho biết host muốn nói chuyện với domain nào. DHCP cho biết IP đó thuộc thiết bị nào tại thời điểm đó. Hai nguồn này kết hợp rất mạnh khi điều tra máy nhiễm malware hoặc truy cập domain độc hại.

**VPN và authentication context:**

VPN logs cho biết user remote access từ đâu, lúc nào và trong bao lâu. Nếu kết hợp với identity logs, analyst có thể phát hiện impossible travel, login ngoài giờ hoặc source IP rủi ro.

**NetFlow/IPFIX:**

Flow data giúp phát hiện scanning, lateral movement, beaconing hoặc exfiltration theo pattern volume/tần suất. Vì không có payload, flow cần kết hợp với các log khác để hiểu nội dung ứng dụng.

**Router/switch logs:**

Thiết bị mạng có thể ghi nhận interface up/down, routing adjacency changes, spanning-tree events hoặc config changes. Những sự kiện này có thể liên quan tới outage, misconfiguration hoặc hoạt động trái phép.

## Ví dụ thực tế

**Ví dụ 1: Điều tra domain độc hại**

Threat intelligence báo domain X độc hại. DNS logs cho biết host nào từng query domain X. DHCP logs xác định thiết bị sở hữu IP tại thời điểm query. EDR logs kiểm tra process nào tạo kết nối.

**Ví dụ 2: Phát hiện port scan**

NetFlow cho thấy một host nội bộ kết nối tới nhiều IP và nhiều port trong thời gian ngắn. Firewall logs có nhiều deny tương ứng. Đây có thể là dấu hiệu host bị compromise và đang scan mạng.

**Ví dụ 3: VPN login bất thường**

VPN logs ghi nhận user đăng nhập từ quốc gia lạ lúc 03:00. Authentication logs cho thấy nhiều lần MFA failed trước đó. SIEM correlation tạo alert credential compromise.

## Cần ghi nhớ

- **Firewall log nói policy action**: không tự chứng minh traffic an toàn hay độc hại.
- **DNS + DHCP là cặp dữ liệu mạnh**: domain query và asset mapping hỗ trợ điều tra.
- **Flow phù hợp phát hiện pattern**: scanning, beaconing và exfiltration thường hiện qua metadata.
- **VPN logs phản ánh remote access risk**: cần kết hợp với identity context.
- **Network device logs cũng quan trọng**: config change và link/routing events có thể là dấu hiệu sự cố.

# 20.2 - End Device Logs

## Tổng quan

End device logs là dữ liệu từ máy trạm, server, mobile device hoặc workload. Đây là nguồn quan trọng vì nhiều hành vi tấn công xảy ra trực tiếp trên endpoint: process lạ, file bị sửa, privilege escalation, malware execution hoặc credential abuse.

## Ý chính

- OS logs ghi nhận đăng nhập, lỗi hệ thống, service và security events.
- Authentication logs cho biết user nào đăng nhập, thất bại hay thành công.
- Process logs giúp phát hiện command hoặc binary đáng ngờ.
- File integrity logs cho biết file quan trọng bị tạo, sửa hoặc xóa.
- EDR telemetry cung cấp process tree, network connection, registry/file activity và detection.
- Endpoint logs cần time sync và hostname/user mapping chính xác.
- Logs từ endpoint nên được gửi về hệ thống tập trung để tránh attacker xóa local evidence.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Endpoint | Thiết bị cuối như laptop, server, VM hoặc workstation. |
| OS log | Log từ hệ điều hành. |
| Security event | Event liên quan tới đăng nhập, quyền, policy hoặc hành vi bảo mật. |
| Process creation | Sự kiện tạo process mới trên endpoint. |
| Command line logging | Ghi nhận command và tham số khi process chạy. |
| EDR telemetry | Dữ liệu endpoint phục vụ phát hiện và phản ứng. |
| File integrity monitoring | Theo dõi thay đổi file quan trọng. |
| Local evidence | Bằng chứng còn nằm trên thiết bị bị điều tra. |

## Ghi chú chi tiết

**Authentication và user activity:**

Endpoint logs cho biết đăng nhập local, remote login, account lockout hoặc privilege changes. Đây là dữ liệu quan trọng khi nghi ngờ credential theft hoặc lateral movement.

**Process và command line:**

Nhiều hành vi tấn công thể hiện qua process tree. Ví dụ Office mở PowerShell, PowerShell tải file từ internet, rồi tạo scheduled task. Nếu không có process/command line logs, chuỗi hành vi này khó thấy.

**File và registry changes:**

Malware có thể tạo persistence bằng registry key, startup folder hoặc service mới. File integrity monitoring giúp phát hiện thay đổi ở file cấu hình, system binaries hoặc web directory.

**Tập trung log:**

Nếu log chỉ nằm trên endpoint bị compromise, attacker có thể xóa hoặc sửa. Gửi log về SIEM/log collector giúp bảo vệ evidence và correlation với network logs.

## Ví dụ thực tế

**Ví dụ: Phát hiện PowerShell đáng ngờ**

EDR ghi nhận `winword.exe` tạo `powershell.exe` với tham số tải script từ internet. Firewall log cũng thấy kết nối ra domain lạ. Kết hợp endpoint process logs và network logs giúp xác định phishing document đã kích hoạt script.

**Ví dụ: Lateral movement**

Một account admin đăng nhập vào nhiều server trong thời gian ngắn bằng remote management. Authentication logs và EDR process logs giúp xác định đây là hoạt động quản trị hợp lệ hay lateral movement đáng ngờ.

## Cần ghi nhớ

- **Endpoint là nơi hành vi tấn công thường xảy ra**: process, file, user và network activity đều quan trọng.
- **Command line logs rất có giá trị**: giúp hiểu process làm gì.
- **EDR bổ sung ngữ cảnh sâu**: process tree và telemetry giúp điều tra nhanh hơn.
- **Local logs dễ bị xóa**: nên gửi log về hệ thống tập trung.
- **User context quan trọng**: cùng một command có thể bình thường với admin nhưng lạ với user thường.

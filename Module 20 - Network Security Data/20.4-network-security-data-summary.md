# 20.4 - Network Security Data Summary

## Tổng quan

Module 20 tổng kết vai trò của network security data trong phát hiện và điều tra sự cố. Dữ liệu bảo mật cần được thu thập, chuẩn hóa, bảo vệ, lưu trữ đủ lâu và correlation để tạo bức tranh đầy đủ.

## Ý chính

| Chủ đề | Tóm tắt |
|---|---|
| Types of security data | Log, event, alert, flow, packet capture và telemetry có vai trò khác nhau. |
| End device logs | Cho biết user, process, file, command line và hành vi trên host. |
| Network logs | Cho biết traffic, protocol, policy action, domain, VPN, firewall và flow. |
| Correlation | Kết hợp nhiều nguồn để dựng timeline và xác định phạm vi ảnh hưởng. |
| Data quality | Timestamp, normalization, retention và integrity quyết định giá trị điều tra. |
| Incident response | Security data giúp xác định điều gì xảy ra, ở đâu, khi nào, do ai và tác động gì. |

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Timeline | Chuỗi sự kiện theo thời gian trong điều tra. |
| Normalized log | Log đã được chuẩn hóa về format/schema chung. |
| Retention policy | Chính sách thời gian lưu trữ log. |
| Evidence integrity | Khả năng chứng minh dữ liệu điều tra không bị sửa đổi. |
| Asset context | Thông tin về thiết bị, owner, criticality và vai trò hệ thống. |
| Event correlation | Liên kết nhiều event để tạo ngữ cảnh. |

## Ghi chú chi tiết

Dữ liệu bảo mật tốt giúp analyst trả lời các câu hỏi cơ bản: điều gì xảy ra, xảy ra khi nào, liên quan tới tài sản nào, user nào, traffic nào, hành động nào được thực hiện và tác động ra sao. Nếu chỉ có một nguồn log, câu trả lời thường thiếu chắc chắn. Nếu có nhiều nguồn nhưng timestamp lệch hoặc không chuẩn hóa, correlation cũng khó chính xác.

## Mối liên hệ giữa các bài

```
Security data types
    ↓
Endpoint logs
    ↓
Network logs
    ↓
Normalization và time sync
    ↓
Correlation
    ↓
Incident timeline và response decision
```

## Ví dụ thực tế

Một alert IDS báo outbound malware traffic. Analyst kiểm tra NetFlow để xem volume, DNS logs để xem domain, DHCP để xác định thiết bị, EDR để xem process, firewall để xem policy action và authentication logs để xem user. Khi kết hợp các nguồn, analyst có thể xác định thiết bị bị nhiễm, scope và bước containment.

## Cần ghi nhớ

- **Security data phải được correlation**: một log đơn lẻ thường chỉ là một mảnh ghép.
- **Endpoint logs và network logs bổ sung nhau**: host cho biết process/user, network cho biết traffic/path.
- **Time sync là bắt buộc**: timeline sai làm điều tra sai.
- **Retention cần phù hợp rủi ro**: log quá ngắn khiến không điều tra được sự cố cũ.
- **Bảo vệ log như bằng chứng**: attacker có thể sửa/xóa log để che dấu hành vi.

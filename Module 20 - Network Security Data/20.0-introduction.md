# 20.0 - Introduction

## Tổng quan

Module 20 giới thiệu vai trò của security data trong network defense. Dữ liệu từ endpoint, network devices, security tools và applications giúp analyst phát hiện bất thường, điều tra nguyên nhân và phản ứng sự cố.

## 20.0.1 Why Should I Take this Module?

Không có dữ liệu, đội bảo mật gần như “mù” trước sự cố. Attacker có thể để lại dấu vết ở nhiều nơi: endpoint process, firewall deny, DNS query, VPN login, IDS alert hoặc proxy log. Hiểu loại dữ liệu nào trả lời câu hỏi nào giúp điều tra nhanh hơn và tránh kết luận sai.

## 20.0.2 What Will I Learn in this Module?

Trong module này, bạn sẽ học:

- Các loại security data thường dùng trong network security.
- Sự khác nhau giữa log, event, alert, flow và packet capture.
- Giá trị của end device logs trong phát hiện hành vi trên host.
- Giá trị của network logs trong phân tích traffic và policy.
- Cách correlation nhiều nguồn dữ liệu để dựng bức tranh sự cố.

## Mục tiêu học tập

| Chủ đề | Mục tiêu |
|---|---|
| Types of security data | Phân biệt log, event, alert, telemetry, flow và packet capture. |
| End device logs | Hiểu dữ liệu từ endpoint, OS, authentication và EDR. |
| Network logs | Nhận biết log từ firewall, IDS/IPS, DNS, DHCP, VPN, proxy và network devices. |
| Correlation | Kết hợp nhiều nguồn để tạo timeline và ngữ cảnh điều tra. |

## Cần ghi nhớ

- **Log không giống alert**: log là bản ghi, alert là cảnh báo dựa trên logic phát hiện.
- **Endpoint và network nhìn từ hai góc khác nhau**: cần kết hợp cả hai.
- **Timestamp phải chính xác**: lệch thời gian làm timeline sai.
- **Correlation tạo ngữ cảnh**: một sự kiện đơn lẻ thường chưa đủ kết luận.
- **Retention quyết định khả năng điều tra quá khứ**: log bị xóa sớm sẽ mất bằng chứng.

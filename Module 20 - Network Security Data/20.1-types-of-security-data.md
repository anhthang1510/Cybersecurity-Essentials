# 20.1 - Types of Security Data

## Tổng quan

Phần này phân loại các dạng security data. Mỗi loại dữ liệu có mục đích riêng: log ghi nhận sự kiện, alert báo rủi ro, flow mô tả kết nối, packet capture cho chi tiết sâu và telemetry phản ánh trạng thái/hành vi.

## Ý chính

- Log là bản ghi sự kiện từ hệ thống, thiết bị hoặc ứng dụng.
- Event là hành động hoặc trạng thái xảy ra; không phải event nào cũng nghiêm trọng.
- Alert là event hoặc pattern được đánh giá là đáng chú ý.
- Flow data mô tả kết nối ở mức metadata.
- Packet capture ghi lại packet chi tiết để phân tích sâu.
- Telemetry có thể bao gồm metrics, trạng thái cấu hình, process, network behavior hoặc performance data.
- Context như asset criticality, user role và threat intelligence giúp dữ liệu có ý nghĩa hơn.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Log record | Một dòng hoặc bản ghi sự kiện cụ thể. |
| Event | Một hành động/trạng thái xảy ra trong hệ thống. |
| Alert | Cảnh báo sinh ra từ rule, signature, anomaly hoặc correlation. |
| Indicator | Dấu hiệu có thể liên quan tới threat, ví dụ IP, hash, domain. |
| Observable | Dữ liệu quan sát được, chưa chắc đã độc hại. |
| Flow record | Metadata mô tả một luồng traffic. |
| Packet capture | Dữ liệu packet chi tiết, thường dùng để điều tra sâu. |
| Context | Thông tin bổ sung giúp diễn giải dữ liệu đúng hơn. |

## Ghi chú chi tiết

**Log, event và alert:**

Log là dữ liệu thô hoặc bán cấu trúc. Event là sự kiện được ghi nhận. Alert là kết quả của logic phát hiện khi hệ thống cho rằng event hoặc chuỗi event đáng chú ý. Một firewall deny là log/event; nếu deny đó khớp pattern scan diện rộng, SIEM có thể tạo alert.

**Flow và packet:**

Flow data trả lời câu hỏi “ai nói chuyện với ai, qua port nào, bao nhiêu dữ liệu, trong bao lâu”. Packet capture trả lời sâu hơn về nội dung packet, nhưng tốn dung lượng và có thể chứa dữ liệu nhạy cảm.

**Context làm dữ liệu có ý nghĩa:**

Cùng một kết nối SSH có thể bình thường nếu từ jump server quản trị, nhưng đáng ngờ nếu từ laptop người dùng tới database server lúc nửa đêm. Asset inventory, user role và baseline giúp diễn giải đúng.

**Data quality:**

Dữ liệu bảo mật cần chính xác, đầy đủ, có timestamp đúng, định dạng dễ phân tích và được bảo vệ khỏi sửa/xóa trái phép. Dữ liệu sai hoặc thiếu có thể dẫn tới kết luận sai.

## Ví dụ thực tế

**Ví dụ: Một IP lạ truy cập server**

Firewall log cho biết IP ngoài kết nối tới server. Flow data cho biết lượng dữ liệu truyền. Server logs cho biết có đăng nhập thành công hay không. EDR logs cho biết process nào chạy sau đó. Kết hợp nhiều loại dữ liệu giúp hiểu sự kiện có nghiêm trọng không.

## Cần ghi nhớ

- **Log là dữ liệu nền tảng**: alert thường được tạo từ log/event hoặc telemetry.
- **Alert không tự bằng chứng đầy đủ**: cần kiểm tra dữ liệu gốc.
- **Flow là metadata**: không thay thế packet capture khi cần chi tiết payload.
- **Context quyết định ý nghĩa**: cùng một event có thể bình thường hoặc nguy hiểm tùy tài sản và user.
- **Data quality ảnh hưởng kết luận**: timestamp, completeness và integrity rất quan trọng.

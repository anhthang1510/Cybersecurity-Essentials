# 4.1 - IP Services

## Tổng quan

Phần này giải thích các điểm yếu trong những dịch vụ nền tảng hỗ trợ IP networking, gồm ARP, DNS và DHCP. Đây là các giao thức rất quen thuộc, nhưng nhiều cơ chế của chúng dựa trên sự tin cậy trong mạng cục bộ hoặc trong hệ thống phân giải tên, vì vậy có thể bị lạm dụng để chuyển hướng traffic, giả mạo thông tin hoặc che giấu kênh điều khiển.

## Ý chính

- ARP giúp host tìm MAC address tương ứng với IP address trong cùng LAN.
- Gratuitous ARP có thể cập nhật ARP cache mà không cần request trước, nên có thể bị lợi dụng để ARP cache poisoning.
- DNS chuyển tên miền thành IP address nhưng có thể bị tấn công qua open resolver, cache poisoning, amplification/reflection, stealth techniques, domain shadowing và tunneling.
- DNS tunneling che dữ liệu hoặc command-and-control traffic trong DNS queries/responses.
- DHCP tự động cấp IP configuration cho client, nhưng rogue DHCP server có thể cấp gateway hoặc DNS server độc hại.
- Phân tích DNS traffic và DHCP behavior giúp phát hiện dấu hiệu bất thường trong mạng.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| ARP | Giao thức ánh xạ IP address sang MAC address trong local network. |
| ARP Request | Gói broadcast hỏi “ai đang dùng IP address này?”. |
| ARP Reply | Gói trả lời chứa MAC address tương ứng với IP address được hỏi. |
| Gratuitous ARP | ARP reply tự gửi để thông báo IP/MAC mapping, không cần request trước. |
| ARP cache | Bảng tạm lưu ánh xạ IP address và MAC address trên host. |
| ARP cache poisoning | Làm sai ARP cache để traffic đi qua attacker. |
| DNS open resolver | DNS resolver trả lời truy vấn từ bên ngoài phạm vi quản trị của nó. |
| DNS cache poisoning | Đưa bản ghi DNS giả vào cache để chuyển người dùng sang địa chỉ độc hại. |
| DNS amplification | Lạm dụng DNS resolver để khuếch đại lưu lượng DoS/DDoS. |
| Fast Flux | Kỹ thuật thay đổi nhanh IP address của domain độc hại để khó bị chặn. |
| Double IP Flux | Thay đổi nhanh cả ánh xạ domain/IP và authoritative name server. |
| DGA | Domain Generation Algorithm, tạo nhiều domain ngẫu nhiên để malware tìm C&C server. |
| Domain shadowing | Chiếm quyền tài khoản domain rồi tạo subdomain ẩn phục vụ tấn công. |
| DNS tunneling | Đóng gói dữ liệu không phải DNS bên trong DNS traffic. |
| DHCP | Giao thức cấp IP configuration tự động cho client. |
| DHCP spoofing | Rogue DHCP server cấp thông tin mạng giả cho client hợp lệ. |

## Ghi chú chi tiết

### ARP và rủi ro trong LAN

ARP hoạt động bằng broadcast trong local subnet. Khi một host biết IP address đích nhưng chưa biết MAC address, nó gửi ARP Request để hỏi toàn mạng cục bộ. Host sở hữu IP đó sẽ gửi ARP Reply kèm MAC address của mình.

Vấn đề là ARP không có cơ chế xác thực mạnh. Một host có thể gửi gratuitous ARP để thông báo ánh xạ IP/MAC, và các host khác có thể cập nhật ARP cache dựa trên thông tin đó. Attacker lợi dụng điểm này để gán MAC address của mình với IP address của default gateway hoặc của nạn nhân.

Kết quả thường gặp:

- Traffic của nạn nhân đi qua máy attacker.
- Attacker có thể thực hiện MiTM attack.
- Passive attack: nghe lén dữ liệu.
- Active attack: sửa dữ liệu, chèn dữ liệu độc hại hoặc làm gián đoạn kết nối.

### ARP cache poisoning

Trong kịch bản ARP cache poisoning, attacker gửi ARP replies giả cho cả host nạn nhân và gateway. Host nạn nhân tin rằng MAC của attacker là MAC của gateway, còn gateway tin rằng MAC của attacker là MAC của nạn nhân. Như vậy attacker đứng giữa hai bên.

Biện pháp giảm thiểu thường dùng:

- Dynamic ARP Inspection (DAI) trên switch hỗ trợ.
- DHCP snooping để tạo cơ sở dữ liệu IP/MAC đáng tin cậy.
- Port security để giới hạn MAC address trên switch port.
- Segmentation để giảm phạm vi ảnh hưởng trong LAN.
- Giám sát ARP table và cảnh báo khi có thay đổi bất thường.

### DNS attacks

DNS là dịch vụ quan trọng vì hầu hết người dùng truy cập tài nguyên bằng domain name thay vì IP address. Nếu DNS bị thao túng, người dùng có thể bị chuyển tới website giả dù họ nhập đúng tên miền.

Các dạng tấn công đáng chú ý:

- DNS cache poisoning: attacker đưa bản ghi giả vào resolver cache.
- DNS amplification/reflection: attacker gửi truy vấn DNS với source IP giả là IP nạn nhân, khiến resolver gửi phản hồi lớn về nạn nhân.
- DNS resource utilization attack: làm DNS server cạn tài nguyên xử lý.
- Fast Flux: hạ tầng độc hại thay đổi IP liên tục để tránh bị blocklist.
- Double IP Flux: thay đổi cả IP mapping và authoritative name server.
- DGA: malware tạo nhiều domain ngẫu nhiên để liên lạc với C&C server.
- Domain shadowing: attacker chiếm quyền tài khoản quản trị domain rồi tạo subdomain độc hại.

### DNS tunneling

DNS tunneling dùng DNS queries/responses làm kênh truyền dữ liệu hoặc lệnh điều khiển. Dữ liệu có thể được chia nhỏ, mã hóa hoặc encode, sau đó nhét vào subdomain labels hoặc DNS records như TXT, NULL, MX, SRV, A hoặc CNAME.

Dấu hiệu cần chú ý:

- DNS query dài bất thường.
- Nhiều truy vấn tới domain lạ hoặc Dynamic DNS domain.
- Tần suất truy vấn cao, mẫu tên miền nhìn giống chuỗi encode.
- Host nội bộ liên tục truy vấn domain ít phổ biến.

Countermeasures:

- DNS filtering và DNS security gateway.
- Chỉ cho client dùng resolver nội bộ được kiểm soát.
- Log DNS queries và phân tích bất thường.
- Chặn domain độc hại hoặc suspicious newly registered domains.
- Kiểm tra outbound DNS traffic thay vì chỉ tập trung inbound traffic.

### DHCP operation và DHCP attacks

DHCP giúp client nhận cấu hình mạng tự động. Quy trình cơ bản thường gồm:

1. Client gửi DHCPDISCOVER để tìm DHCP server.
2. Server trả DHCPOFFER với IP address, subnet mask, default gateway, DNS server và lease time.
3. Client gửi DHCPREQUEST để chọn offer.
4. Server gửi DHCPACK để xác nhận lease.

Trong DHCP spoofing, rogue DHCP server phản hồi nhanh hơn server hợp lệ và cấp cấu hình sai. Attacker có thể cấp:

- Default gateway là máy attacker để tạo MiTM.
- DNS server độc hại để chuyển hướng người dùng.
- IP configuration sai để gây DoS.

Biện pháp giảm thiểu:

- DHCP snooping, chỉ cho DHCP replies đi từ trusted ports.
- IP Source Guard để ngăn source IP giả.
- Port security để hạn chế thiết bị trái phép.
- Theo dõi log DHCP và phát hiện server lạ.
- Kiểm soát physical access vào switch port.

## Ví dụ thực tế

Một nhân viên cắm nhầm thiết bị Wi-Fi router cá nhân vào mạng công ty. Thiết bị này bật DHCP server và bắt đầu cấp DNS server riêng cho máy trạm. Một số máy nhận cấu hình từ thiết bị đó và bị chuyển tới website đăng nhập giả. Nếu switch có DHCP snooping và chỉ port kết nối tới DHCP server chính thức được đánh dấu trusted, các DHCP Offer từ thiết bị lạ sẽ bị chặn.

## Cần ghi nhớ

- ARP và DHCP đều rất quan trọng trong LAN nhưng thiếu xác thực mặc định.
- DNS vừa là dịch vụ thiết yếu vừa là kênh mà attacker có thể dùng để chuyển hướng hoặc che giấu traffic.
- Không nên để client tự do dùng bất kỳ DNS resolver nào trên internet trong môi trường doanh nghiệp.
- ARP poisoning và DHCP spoofing thường dẫn tới MiTM hoặc DoS.
- DNS tunneling thường được phát hiện bằng phân tích mẫu truy vấn, độ dài domain và tần suất truy vấn.

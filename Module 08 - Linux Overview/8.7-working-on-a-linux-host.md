# 8.7 Working on a Linux Host

## Tổng quan

Làm việc trên Linux host bao gồm cài ứng dụng, cập nhật package, theo dõi process, kiểm tra network, xử lý service và phát hiện dấu hiệu malware. Dù Linux có mô hình quyền mạnh, nó vẫn có thể bị tấn công qua service lỗi thời, cấu hình sai, tài khoản yếu, script độc hại hoặc rootkit.

Package manager giúp cài, cập nhật và gỡ phần mềm theo repository. Process management giúp analyst xác định chương trình đang chạy, quan hệ parent child và hành vi tiêu thụ tài nguyên.

## Ý chính

* Package manager như pacman, dpkg, apt và apt-get giúp quản lý phần mềm.
* `update` cập nhật danh sách package, còn `upgrade` nâng cấp package đã cài.
* GUI updater có thể hỗ trợ người dùng phổ thông, nhưng CLI vẫn quan trọng cho server.
* Process có PID, có thể fork để tạo child process.
* Apache và nhiều service khác dùng nhiều process hoặc worker để phục vụ request.
* Linux vẫn có thể bị virus, Trojan, worm và rootkit.
* Port scan và service version enumeration giúp phát hiện bề mặt tấn công.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| Package | Gói phần mềm có metadata, dependency và file cài đặt |
| Package manager | Công cụ quản lý cài đặt, cập nhật và gỡ package |
| pacman | Package manager thường dùng trên Arch Linux |
| dpkg | Công cụ quản lý package định dạng `.deb` ở mức thấp |
| apt | Công cụ quản lý package cấp cao trên Debian và Ubuntu |
| apt-get | Công cụ CLI truyền thống để cài và cập nhật package |
| PID | Process ID, số định danh process đang chạy |
| Fork | Cơ chế process tạo process con |
| Rootkit | Malware ẩn sâu nhằm che giấu truy cập và duy trì backdoor |
| Backdoor | Cơ chế truy cập bí mật bỏ qua đường xác thực bình thường |
| Version enumeration | Thu thập phiên bản service để đánh giá lỗ hổng có thể tồn tại |

## Ghi chú chi tiết

### Package update và upgrade

Trên hệ Debian và Ubuntu, `apt-get update` tải metadata mới từ repository. Nó chưa nâng cấp phần mềm. `apt-get upgrade` mới cài phiên bản mới cho package đang có nếu điều kiện dependency cho phép. Nhầm hai thao tác này có thể khiến hệ thống tưởng đã vá nhưng thực tế chỉ mới cập nhật danh sách.

### Process và PID

Mỗi process đang chạy có PID. Process có thể tạo process con bằng fork. Khi điều tra, quan hệ parent child giúp analyst hiểu process nào sinh ra process lạ. Ví dụ web server có thể có master process và nhiều worker process.

### Malware trên Linux

Linux có thể bị malware, đặc biệt khi server phơi ra Internet và chạy service lỗi thời. Rootkit nguy hiểm vì có thể ẩn file, process, connection và thay đổi công cụ hệ thống. Vì vậy cần kết hợp patching, least privilege, monitoring, integrity check và audit tool.

### Kiểm tra service từ mạng

Attacker thường dùng port scan và version enumeration để tìm service lỗi thời. Ngay cả thao tác đơn giản như kết nối đến HTTP port bằng `telnet` cũng có thể cho thấy banner hoặc phản hồi gợi ý loại service.

## Lệnh / Ví dụ thực hành

```bash
sudo apt-get update
sudo apt-get upgrade
```

Cập nhật danh sách package rồi nâng cấp package. Trong môi trường thật, cần có lịch bảo trì và kiểm thử trước.

```bash
ps
kill 1234
```

Xem process, theo dõi tài nguyên và gửi tín hiệu kết thúc đến PID cụ thể. Thay `1234` bằng PID thật trong lab.

```bash
ip address
ping 8.8.8.8
```

Kiểm tra địa chỉ IP của host và kiểm tra kết nối mạng cơ bản.

```bash
telnet example.com 80
```

Kết nối thử đến port 80 để quan sát phản hồi giao thức trong lab được phép.

```bash
sudo apt-get install chkrootkit
sudo apt-get install lynis
sudo chkrootkit
sudo lynis audit system
```

Cài và chạy công cụ kiểm tra rootkit, audit cấu hình. Kết quả cần được phân tích cẩn thận vì có thể có false positive.

## Ví dụ liên quan

Một Linux web server chạy phiên bản ứng dụng cũ có lỗ hổng remote code execution. Sau khi bị khai thác, attacker cài rootkit để che process và mở backdoor. Analyst cần kiểm tra process, connection, package, log, file integrity và kết quả từ công cụ như Lynis hoặc chkrootkit.

## Cần ghi nhớ

Cập nhật package và quản lý process là việc hằng ngày của bảo mật Linux. Đừng xem Linux là miễn nhiễm malware. Hãy theo dõi service, phiên bản, process lạ và dấu hiệu che giấu.

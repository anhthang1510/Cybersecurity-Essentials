# 8.8 Linux Basics Summary

## Tổng quan

Module 8 giới thiệu Linux như một nền tảng quan trọng cho CyberOps. Bạn đã đi qua Linux basics, shell, server và client, quản trị server, file system, GUI và cách làm việc trên Linux host. Các chủ đề này liên kết với nhau: muốn bảo vệ host, bạn cần hiểu lệnh, service, permission, log, package và process.

Phần tóm tắt này giúp gom lại ý chính để ôn tập dài hạn và chuẩn bị cho lab.

## Ý chính

* Linux là open source OS linh hoạt, phổ biến trong server, cloud và công cụ bảo mật.
* Distro kết hợp kernel với utilities, package manager và cấu hình mặc định.
* Shell giúp analyst kiểm tra hệ thống, lọc dữ liệu và tự động hóa.
* Service lắng nghe trên port cần được kiểm kê, cập nhật và giám sát.
* Hardening gồm tắt thứ không cần, vá lỗi, giới hạn quyền, bảo vệ SSH và xem log.
* File system Linux bắt đầu từ `/`, dùng mount point và permission chi tiết.
* GUI hữu ích cho máy người dùng, nhưng server thường ưu tiên CLI.
* Linux host vẫn có rủi ro malware, rootkit và service lỗi thời.

## Ghi chú chi tiết

### Nhìn Linux theo góc độ SOC

SOC analyst thường không chỉ đọc màn hình cảnh báo. Họ cần kiểm tra host, đối chiếu log, hiểu service đang lắng nghe, xem process, xác định file cấu hình và đánh giá liệu hành vi có phù hợp baseline hay không. Linux cung cấp nhiều công cụ để làm những việc này bằng text và CLI.

### Nhóm kỹ năng cần luyện

Kỹ năng đầu tiên là đọc lệnh và manual page. Kỹ năng thứ hai là hiểu file system và permission. Kỹ năng thứ ba là kiểm tra service, port và process. Kỹ năng thứ tư là áp dụng hardening và giám sát log. Bốn nhóm kỹ năng này tạo nền tảng cho incident response trên Linux.

### Tư duy an toàn

Không chạy lệnh quản trị nếu không hiểu tác động. Không mở service nếu không có nhu cầu. Không tin rằng server an toàn chỉ vì nó chạy Linux. Bảo mật đến từ cấu hình đúng, cập nhật đều, quyền tối thiểu và giám sát liên tục.

## Lệnh / Ví dụ thực hành

```bash
man ls
ls -l
uname -a
whoami
id
```

Nhóm lệnh xem hướng dẫn, liệt kê file và kiểm tra danh tính người dùng.

```bash
nmap localhost
sudo netstat -tunap
sudo ps -elf
```

Nhóm lệnh xem bề mặt mạng và process liên quan.

```bash
mount
lsblk
chmod 640 report.txt
chown analyst report.txt
```

Nhóm lệnh xem file system và chỉnh quyền cơ bản.

```bash
sudo apt-get update
sudo apt-get upgrade
top
sudo lynis audit system
```

Nhóm lệnh cập nhật, theo dõi process và audit cấu hình.

## Ví dụ liên quan

Khi thấy cảnh báo kết nối outbound bất thường từ server, analyst có thể dùng `sudo netstat -tunap` để xem process tạo kết nối, `ps` để kiểm tra parent process, log để tìm thời điểm bắt đầu, package manager để xem phần mềm lỗi thời và permission check để phát hiện file đáng ngờ.

## Cần ghi nhớ

Module này không yêu cầu bạn nhớ mọi option. Điều quan trọng là hiểu cách đặt câu hỏi với Linux: service nào đang chạy, ai sở hữu file, quyền có hợp lý không, log nói gì, package có được vá không và process nào tạo hành vi mạng.

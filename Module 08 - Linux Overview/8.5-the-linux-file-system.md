# 8.5 The Linux File System

## Tổng quan

Linux tổ chức mọi thứ dưới cây thư mục bắt đầu từ root directory `/`. Thiết bị lưu trữ, partition và file system được gắn vào cây này thông qua mount point. Người học bảo mật cần hiểu file system vì permission, log, config, binary và dữ liệu người dùng đều nằm trong cấu trúc này.

Linux hỗ trợ nhiều file system như ext2, ext3, ext4, NFS, CDFS, swap, HFS+ và APFS tùy mục đích. ext3 và ext4 có journaling, giúp giảm rủi ro hỏng dữ liệu sau sự cố mất điện hoặc crash.

## Ý chính

* Root directory `/` là điểm bắt đầu của cây file system Linux.
* Partition cần được mount vào mount point trước khi dùng như một thư mục bình thường.
* MBR là cấu trúc phân vùng cũ, vẫn gặp trong nhiều hệ thống.
* Permission gồm user, group và others với quyền read, write, execute.
* `ls -l` hiển thị quyền, owner, group, kích thước và thời gian sửa đổi.
* Hard link và symbolic link giúp trỏ đến dữ liệu hoặc đường dẫn theo cách khác nhau.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| ext2 | File system Linux cũ, không có journaling |
| ext3 | File system có journaling, phát triển từ ext2 |
| ext4 | File system phổ biến, cải thiện hiệu năng và dung lượng so với ext3 |
| Journaling | Cơ chế ghi nhật ký thay đổi để phục hồi tốt hơn sau sự cố |
| NFS | Network File System, cho phép truy cập file qua mạng |
| CDFS | File system cho đĩa quang như CD |
| swap | Không gian đĩa hỗ trợ memory khi RAM thiếu |
| MBR | Master Boot Record, kiểu lưu thông tin partition truyền thống |
| Partition | Phân vùng logic trên thiết bị lưu trữ |
| Mount point | Thư mục nơi file system được gắn vào cây Linux |
| Symlink | Link trỏ đến đường dẫn khác, còn gọi là symbolic link |
| Hard link | Tên file khác trỏ đến cùng inode dữ liệu |

## Ghi chú chi tiết

### Mount và thiết bị

Thiết bị lưu trữ thường xuất hiện trong `/dev`, ví dụ `/dev/sda1` là partition đầu tiên trên disk đầu tiên theo cách đặt tên truyền thống. Khi mount `/dev/sdb1` vào `~/second_drive/`, nội dung partition đó xuất hiện trong thư mục `second_drive`.

### Permission

Linux chia quyền theo user, group và others. Mỗi nhóm quyền có thể có read `r`, write `w` và execute `x`. Với file, execute nghĩa là có thể chạy file như chương trình hoặc script. Với directory, execute cho phép đi vào thư mục và truy cập mục bên trong nếu quyền khác cho phép.

### Octal permission

Permission thường được biểu diễn bằng số octal. Read có giá trị 4, write là 2, execute là 1. Ví dụ 7 nghĩa là `rwx`, 5 nghĩa là `r-x`, 0 nghĩa là không có quyền. Dù root có thể vượt nhiều giới hạn quyền, vận hành an toàn vẫn cần đặt permission đúng.

### Hard link và symlink

Hard link tạo thêm một tên cho cùng dữ liệu trên file system. Symlink giống shortcut trỏ đến đường dẫn. Symlink có thể bị hỏng nếu đích bị xóa hoặc đổi tên.

## Lệnh / Ví dụ thực hành

```bash
mount
mount | grep sda1
lsblk
```

Xem file system đang mount và danh sách block device.

```bash
cd /
ls -l
```

Di chuyển đến root directory và xem nội dung dạng chi tiết.

```bash
mkdir second_drive
sudo mount /dev/sdb1 ~/second_drive/
sudo umount /dev/sdb1
```

Tạo mount point, mount partition và unmount khi không dùng nữa. Chỉ chạy với thiết bị đúng trong lab.

```bash
ln source.txt hardlink.txt
ln -s /var/log/messages messages-link
chmod 640 report.txt
chown analyst report.txt
chgrp security report.txt
```

Ví dụ tạo link và chỉnh quyền, owner, group.

## Ví dụ liên quan

Nếu file log nhạy cảm cho phép `others` đọc, người dùng không có nhiệm vụ vẫn có thể xem thông tin hệ thống. Một permission như `640` thường hợp lý hơn cho file chỉ owner và group được đọc.

## Cần ghi nhớ

File system là nền tảng của quản trị Linux. Hiểu mount, permission và link giúp bạn điều tra sự cố tốt hơn và tránh cấu hình làm lộ dữ liệu.

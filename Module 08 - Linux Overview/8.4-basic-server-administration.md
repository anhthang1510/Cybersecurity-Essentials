# 8.4 Basic Server Administration

## Tổng quan

Quản trị server Linux ở mức cơ bản gồm đọc và chỉnh file cấu hình, khởi động lại hoặc reload service, quản lý quyền quản trị, theo dõi log và áp dụng hardening. Nhiều service như Nginx, NTP hoặc Snort dùng file cấu hình dạng text, trong đó dòng bắt đầu bằng `#` thường là comment.

Server an toàn không chỉ dựa vào một công cụ. Nó cần cấu hình tối thiểu, cập nhật đều, hạn chế quyền, kiểm soát SSH, chính sách mật khẩu, quản lý log và giám sát liên tục.

## Ý chính

* File cấu hình service thường là text file và cần reload hoặc restart sau khi chỉnh.
* Superuser có quyền rất mạnh, nên chỉ dùng khi cần.
* Comment trong config giúp ghi chú hoặc vô hiệu hóa dòng cấu hình.
* Hardening giảm bề mặt tấn công bằng cách loại bỏ phần mềm, service và quyền không cần thiết.
* Log là nguồn dữ liệu quan trọng để điều tra lỗi, sự kiện và dấu hiệu xâm nhập.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| Configuration file | File chứa thiết lập của service hoặc OS |
| Reload | Nạp lại cấu hình mà không nhất thiết dừng hẳn service |
| Restart | Dừng rồi khởi động lại service |
| Comment | Dòng ghi chú, thường bắt đầu bằng `#` trong nhiều file cấu hình |
| Hardening | Giảm rủi ro bằng cấu hình an toàn và loại bỏ thành phần không cần |
| Daemon | Chương trình chạy nền để cung cấp service |
| Log | Bản ghi sự kiện phục vụ vận hành và điều tra |
| SSSD | System Security Services Daemon, hỗ trợ identity và authentication trong nhiều môi trường |
| `/var/log/messages` | File log hệ thống phổ biến trên một số distro |
| Least privilege | Nguyên tắc cấp quyền tối thiểu cần thiết |

## Ghi chú chi tiết

### Cấu hình service

Service như Nginx có file cấu hình quy định port, thư mục web, log path và hành vi xử lý request. NTP dùng cấu hình để đồng bộ thời gian. Snort dùng rule và config để phát hiện traffic đáng ngờ. Sau khi chỉnh, service thường cần reload hoặc restart để áp dụng thay đổi.

### Hardening server

Các bước hardening cơ bản gồm bảo vệ vật lý, gỡ package không cần thiết, tắt service không dùng, cấu hình SSH an toàn, tránh cho phép root login qua SSH, cập nhật OS, tắt tự động mount USB nếu không cần, đặt mật khẩu mạnh, đổi mật khẩu theo chính sách phù hợp, ngăn dùng lại mật khẩu cũ và phân quyền quản trị theo vai trò.

### Log và timestamp

Log có thể đến từ application, event, service, system hoặc kernel. Timestamp giúp dựng lại dòng thời gian sự cố. Khi nhiều hệ thống cùng gửi log về SIEM, đồng bộ thời gian là điều kiện quan trọng để correlation chính xác.

## Lệnh / Ví dụ thực hành

```bash
cat /var/log/messages
more /var/log/messages
```

Đọc log trực tiếp hoặc theo từng trang. Trên một số distro, file log chính có thể nằm ở đường dẫn khác.

```bash
man nginx
```

Đọc manual page để hiểu cách gọi Nginx và các option liên quan.

```bash
sudo nano /etc/hosts
```

Ví dụ chỉnh một file cấu hình hệ thống bằng quyền quản trị. Luôn tạo bản sao hoặc kiểm soát thay đổi trong môi trường thật.

## Ví dụ liên quan

Một server vẫn bật SSH cho root login và dùng mật khẩu yếu. Dù web app đã vá lỗi, attacker vẫn có thể thử brute force SSH. Hardening đúng sẽ tắt root SSH login, yêu cầu key hoặc MFA nếu có, giới hạn nguồn truy cập và theo dõi log đăng nhập.

## Cần ghi nhớ

Quản trị server an toàn là kết hợp giữa cấu hình, cập nhật, log và quyền. Đừng sửa config nếu không biết cách quay lại trạng thái cũ, và đừng để service chạy chỉ vì nó được cài mặc định.

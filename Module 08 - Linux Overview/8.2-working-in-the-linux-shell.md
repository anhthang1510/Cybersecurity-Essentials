# 8.2 Working in the Linux Shell

## Tổng quan

Shell là nơi người dùng nhập lệnh để tương tác với Linux. GUI giúp thao tác trực quan, nhưng CLI thường nhanh hơn, rõ hơn và phù hợp cho quản trị từ xa, tự động hóa và phân tích sự cố. Terminal emulator là ứng dụng mở ra môi trường dòng lệnh trong GUI.

Khi làm việc với shell, bạn cần hiểu command, option, argument, `PATH`, permission và cách đọc hướng dẫn bằng `man`. Linux xem nhiều tài nguyên như file, vì vậy thao tác file và text là kỹ năng rất quan trọng.

## Ý chính

* GUI tiện cho thao tác trực quan, CLI mạnh khi cần tốc độ, lặp lại và tự động hóa.
* Terminal emulator cung cấp cửa sổ để chạy shell trong môi trường đồ họa.
* `man` giúp đọc manual page của lệnh.
* `PATH` cho shell biết nơi tìm chương trình khi bạn nhập tên lệnh.
* Permission quyết định ai có thể đọc, ghi hoặc thực thi file.
* `sudo` cho phép chạy lệnh với quyền cao hơn khi người dùng được cấp phép.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| GUI | Graphical User Interface, giao diện đồ họa |
| CLI | Command Line Interface, giao diện dòng lệnh |
| Terminal emulator | Ứng dụng cung cấp phiên terminal trong GUI |
| Command | Lệnh yêu cầu hệ thống thực hiện một tác vụ |
| Option hoặc switch | Tham số thay đổi hành vi của lệnh, ví dụ `-l` |
| Argument | Đối tượng lệnh xử lý, ví dụ tên file hoặc thư mục |
| `PATH` | Biến môi trường chứa danh sách thư mục để tìm chương trình |
| Permission | Quyền đọc, ghi, thực thi trên file hoặc directory |
| `sudo` | Cơ chế chạy lệnh với quyền quản trị có kiểm soát |
| Text file | File dạng văn bản, thường dùng cho cấu hình và log |

## Ghi chú chi tiết

### Cấu trúc lệnh

Một lệnh Linux thường có dạng `command option argument`. Ví dụ `ls -l Documents/` dùng `ls` làm command, `-l` làm option và `Documents/` làm argument. Không phải lệnh nào cũng cần đủ ba phần.

### Manual page

`man` là nguồn hướng dẫn cục bộ rất quan trọng. Khi gặp lệnh mới, đọc manual page giúp bạn biết cú pháp, option và ví dụ sử dụng. Đây là thói quen tốt hơn việc đoán tham số.

### Text file và cấu hình

Linux dùng rất nhiều text file cho cấu hình. File `/etc/hosts` là ví dụ quen thuộc, dùng để ánh xạ tên host với địa chỉ IP ở mức cục bộ. Khi sửa file hệ thống, bạn thường cần `sudo` và phải cẩn thận vì thay đổi sai có thể ảnh hưởng mạng hoặc service.

### Nano shortcuts

`nano` là text editor đơn giản trong terminal. Một số phím tắt hữu ích gồm `Ctrl+O` để lưu, `Ctrl+W` để tìm kiếm, `Ctrl+G` để mở trợ giúp và `Ctrl+X` để thoát.

## Lệnh / Ví dụ thực hành

```bash
man ls
```

Mở manual page của lệnh `ls` để xem cách liệt kê file và directory.

```bash
ls
ls -l Documents/
ls -l Documents/ | grep OS
```

Các lệnh này liệt kê nội dung thư mục, hiển thị dạng chi tiết và lọc dòng có chứa `OS` bằng pipeline.

```bash
uname -a
```

Xem thông tin kernel, hostname, kiến trúc và phiên bản hệ thống.

```bash
sudo nano /etc/hosts
```

Mở file cấu hình host cục bộ bằng quyền quản trị. Trong lab, chỉ chỉnh sửa khi bạn hiểu mục đích thay đổi.

## Ví dụ liên quan

Nếu một analyst cần tìm file log chứa chuỗi `failed`, CLI cho phép kết hợp lệnh đọc file và lọc text rất nhanh. Cách làm này tiết kiệm thời gian hơn nhiều so với mở từng file bằng GUI.

## Cần ghi nhớ

Shell là kỹ năng nền cho Linux security. Hãy đọc manual page, hiểu quyền trước khi sửa file và dùng `sudo` như một công cụ có kiểm soát, không phải thói quen mặc định cho mọi lệnh.

# 8.1 Linux Basics

## Tổng quan

Linux bắt đầu từ năm 1991 với Linux kernel, sau đó phát triển thành một hệ sinh thái lớn gồm nhiều distribution như Ubuntu, Debian, Fedora, Kali Linux và Security Onion. Với người học cybersecurity, Linux đáng học vì nó cho phép kiểm soát hệ thống bằng CLI, hỗ trợ mạng tốt, có nhiều công cụ phân tích và minh bạch hơn nhiều nền tảng đóng.

Linux thường được dùng cho cả phòng thủ và kiểm thử. Security Onion phù hợp cho defensive monitoring, còn Kali Linux thường dùng cho penetration testing trong môi trường được phép.

## Ý chính

* Linux kernel là phần lõi, còn distro là gói hoàn chỉnh gồm kernel, công cụ và package manager.
* Root hoặc superuser có quyền cao nhất, vì vậy cần dùng cẩn thận.
* Linux phù hợp với mạng vì ổn định, hỗ trợ service tốt và dễ tự động hóa.
* SOC dùng Linux để chạy packet capture, IDS, log manager, SIEM connector và ticketing workflow.
* Security Onion thiên về giám sát phòng thủ, Kali Linux thiên về kiểm thử tấn công có kiểm soát.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| Kernel | Lõi OS giao tiếp với phần cứng và quản lý tài nguyên |
| Distribution hoặc distro | Bản Linux hoàn chỉnh gồm kernel, utilities, package và cấu hình |
| Root | Tài khoản quyền cao nhất trên Linux |
| Superuser | Người dùng có toàn quyền quản trị hệ thống |
| CLI | Command Line Interface, giao diện nhập lệnh bằng text |
| Security Onion | Distro và nền tảng dùng cho network security monitoring |
| Sguil | Công cụ hỗ trợ analyst xem và xử lý cảnh báo bảo mật mạng |
| Kali Linux | Distro phổ biến cho penetration testing và security assessment |
| IDS | Intrusion Detection System, hệ thống phát hiện dấu hiệu xâm nhập |
| SIEM | Security Information and Event Management, nền tảng thu thập và tương quan log |

## Ghi chú chi tiết

### Linux kernel và distro

Linux kernel tự nó chưa phải trải nghiệm OS đầy đủ cho người dùng cuối. Một distro bổ sung shell, GNU utilities, package manager, desktop nếu cần, installer, driver và repository phần mềm. Vì vậy, hai distro cùng dùng Linux kernel nhưng có giao diện, cách quản lý package và mục tiêu sử dụng khác nhau.

### Giá trị của Linux trong SOC

Linux giúp analyst thao tác sâu với network, process, file và log. CLI cho phép lọc dữ liệu nhanh bằng pipeline, chạy script lặp lại và làm việc tốt qua SSH. Nhiều sensor, IDS và log collector chạy trên Linux vì nền tảng này ổn định và có thể cấu hình rất chi tiết.

### Security Onion và Kali Linux

Security Onion thường được dùng để thu thập packet, phân tích cảnh báo, chạy IDS, quản lý log và hỗ trợ điều tra sự kiện. Kali Linux tập trung vào công cụ kiểm thử như scanning, exploitation framework, password testing và wireless assessment. Một bên trả lời câu hỏi “chuyện gì đang xảy ra trong mạng”, bên kia thường trả lời “hệ thống có thể bị khai thác ra sao”.

## Lệnh / Ví dụ thực hành

```bash
uname -a
```

Lệnh này hiển thị thông tin kernel và hệ thống. Đây là bước kiểm tra nhanh khi bạn cần biết host đang chạy phiên bản Linux nào.

```bash
whoami
id
```

Các lệnh này giúp xác định người dùng hiện tại và quyền nhóm. Khi làm lab bảo mật, kiểm tra quyền trước khi chạy lệnh quản trị là thói quen tốt.

## Ví dụ liên quan

Trong một lab SOC, Security Onion có thể nhận packet từ SPAN port, tạo alert từ IDS và cho analyst xem sự kiện trong Sguil. Trong một lab kiểm thử hợp pháp, Kali Linux có thể chạy scanning để xác định service đang mở trước khi đánh giá rủi ro cấu hình.

## Cần ghi nhớ

Linux không phải một sản phẩm duy nhất. Hãy phân biệt kernel, distro, công cụ và mục đích sử dụng. Với bảo mật, cũng cần phân biệt công cụ phòng thủ và công cụ kiểm thử để tránh dùng sai bối cảnh.

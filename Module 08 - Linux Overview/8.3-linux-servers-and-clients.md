# 8.3 Linux Servers and Clients

## Tổng quan

Trong mạng, server cung cấp service còn client gửi yêu cầu sử dụng service đó. Mỗi service thường lắng nghe trên một port và dùng một protocol nhất định, ví dụ HTTP cho web hoặc FTP cho truyền file. Trên Linux, việc kiểm tra service, process và port đang mở là kỹ năng quan trọng để quản trị và bảo mật.

Các công cụ như Nmap và netstat giúp nhìn thấy bề mặt mạng của host. Port mở không tự động là nguy hiểm, nhưng mọi port mở đều cần có lý do, chủ sở hữu và cơ chế giám sát.

## Ý chính

* Server cung cấp tài nguyên hoặc service, client gửi request để sử dụng.
* Service thường gắn với protocol và port.
* Listening port là dấu hiệu một process đang chờ kết nối.
* Well known ports giúp client biết nên kết nối đến port nào cho service phổ biến.
* Nmap hỗ trợ scanning để xác định host, port và service.
* netstat giúp xem kết nối mạng và process liên quan trên host.

## Thuật ngữ quan trọng

| Thuật ngữ | Giải thích ngắn |
|---|---|
| Server | Hệ thống cung cấp service cho client |
| Client | Hệ thống hoặc ứng dụng gửi request đến server |
| Service | Chương trình cung cấp chức năng qua mạng hoặc cục bộ |
| Port | Số định danh endpoint giao tiếp của transport protocol |
| Listening | Trạng thái chờ kết nối đến trên một port |
| Protocol | Bộ quy tắc giao tiếp, ví dụ HTTP, FTP, SSH |
| Well known port | Port thường gắn với service phổ biến, ví dụ HTTP port 80 |
| Nmap | Công cụ network scanning và service discovery |
| netstat | Công cụ xem kết nối mạng, port và socket |
| Process | Chương trình đang chạy trong hệ thống |

## Ghi chú chi tiết

### Server, client và service

Một web browser là client khi gửi HTTP request đến web server. Một FTP client gửi yêu cầu đến FTP server để truyền file. Trên Linux, nhiều service chạy nền và bắt đầu cùng hệ thống, vì vậy cần biết service nào cần thiết và service nào nên tắt.

### Port và rủi ro

Port mở là điểm có thể tiếp xúc với mạng. Nếu service phía sau port bị cấu hình sai, lỗi thời hoặc có lỗ hổng, attacker có thể dùng port đó làm điểm vào. Vì vậy, quản trị viên cần kiểm kê port định kỳ và so sánh với baseline.

### Nmap và netstat

Nmap thường nhìn từ góc độ scanner, có thể quét localhost hoặc host khác trong phạm vi được phép. netstat nhìn từ bên trong host, cho biết socket, protocol, địa chỉ, port và đôi khi cả process. Hai góc nhìn này bổ sung cho nhau.

## Lệnh / Ví dụ thực hành

```bash
nmap localhost
sudo nmap -sU localhost
man nmap
```

Các lệnh này quét TCP mặc định trên local host, quét UDP với quyền cao hơn và đọc hướng dẫn của Nmap.

```bash
sudo ps -elf
sudo ps -ejH
```

Xem process dạng chi tiết và dạng cây để hiểu quan hệ parent, child giữa các process.

```bash
sudo /usr/sbin/nginx
```

Khởi chạy Nginx trong lab nếu package và đường dẫn tồn tại. Không chạy service mới trên hệ thống thật nếu chưa được phép.

```bash
netstat
sudo netstat -tunap
```

Xem kết nối mạng và danh sách TCP, UDP, numeric address, listening process.

## Ví dụ liên quan

Nếu `nmap localhost` cho thấy port 80 đang mở, analyst cần xác định đó có phải web server được phê duyệt hay không. Sau đó có thể dùng `sudo netstat -tunap` để xem process nào đang lắng nghe trên port đó.

## Cần ghi nhớ

Không đánh giá port mở chỉ bằng tên service. Hãy xác định process, phiên bản, mục đích nghiệp vụ, cấu hình và log đi kèm. Port không cần thiết nên được đóng hoặc chặn bằng firewall.

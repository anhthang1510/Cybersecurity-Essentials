# 13.4 - AAA Usage and Operation

## Tổng quan

AAA (Authentication, Authorization, Accounting) là khung cung cấp bảo mật truy cập có khả năng mở rộng. Bài này đi sâu vào cách AAA hoạt động trong mạng thực tế, sự khác biệt giữa AAA local và server-based, và hai giao thức AAA chính: RADIUS và TACACS+. Phần cuối đề cập AAA accounting logs — nguồn thông tin quan trọng cho kiểm toán và phân tích sự cố.

## Ý chính

**AAA Operation:**

Chính sách bảo mật mạng định nghĩa ai được phép kết nối và làm gì. AAA cung cấp khung để thực thi chính sách này ở quy mô lớn.

Ví dụ trực quan: thẻ tín dụng.
- Authentication: Ai sử dụng thẻ? (Danh tính chủ thẻ)
- Authorization: Được chi bao nhiêu? (Hạn mức tín dụng)
- Accounting: Đã chi tiêu vào đâu? (Sao kê giao dịch)

**Hai phương pháp triển khai AAA:**

| Phương pháp | Mô tả | Phù hợp với |
|-----------|------|------------|
| Local AAA | Xác thực dựa trên username/password lưu cục bộ trên thiết bị | Mạng nhỏ; đơn giản |
| Server-based AAA | Xác thực dựa trên server AAA trung tâm chứa tất cả user | Mạng trung bình đến lớn; khuyến nghị |

Server-based AAA có thể tích hợp Active Directory hoặc LDAP cho xác thực người dùng, đồng thời duy trì cơ sở dữ liệu authorization và accounting riêng.

**RADIUS vs TACACS+:**

| Tiêu chí | TACACS+ | RADIUS |
|---------|---------|--------|
| Tiêu chuẩn | Chủ yếu Cisco | Open/RFC chuẩn mở |
| Transport | TCP port 49 | UDP port 1812 và 1813 (hoặc 1645, 1646) |
| Chức năng | Tách biệt authentication, authorization, accounting | Kết hợp authentication và authorization; tách accounting |
| CHAP | Bidirectional challenge-response | Unidirectional từ server đến client |
| Mã hóa | Toàn bộ body gói tin | Chỉ mật khẩu trong access-request |
| Tùy chỉnh | Phân quyền lệnh router theo user/group | Không phân quyền lệnh router |
| Accounting | Hạn chế | Toàn diện |

Khi nào dùng TACACS+: môi trường Cisco toàn bộ; cần kiểm soát chi tiết lệnh IOS; yêu cầu mã hóa hoàn toàn.  
Khi nào dùng RADIUS: môi trường đa nhà sản xuất; xác thực VPN/Wi-Fi; cần accounting toàn diện.

**AAA Accounting Logs:**

Log accounting từ tất cả thiết bị được gửi về repository tập trung, đơn giản hóa kiểm toán hành vi người dùng.

Các loại thông tin accounting thu thập:

| Loại | Mô tả |
|------|------|
| Network accounting | Thông tin phiên PPP: gói tin và byte |
| Connection accounting | Kết nối outbound từ AAA client (SSH, etc.) |
| EXEC accounting | Phiên EXEC terminal: username, thời gian bắt đầu/kết thúc, IP |
| System accounting | Sự kiện system-level: reboot, bật/tắt accounting |
| Command accounting | Lệnh EXEC theo mức đặc quyền, thời gian và người dùng |

AAA accounting ghi lại bản ghi "start" khi bắt đầu phiên và "stop" khi kết thúc. Cisco còn hỗ trợ ghi bản ghi "stop" cho kết nối thất bại trong xác thực — hữu ích cho phát hiện tấn công.

Log AAA chứa: username, ngày/giờ, lệnh thực sự được thực thi. Dùng để: gỡ lỗi thiết bị, bằng chứng chống lại hành vi độc hại, compliance audit.

## Thuật ngữ quan trọng

| Thuật ngữ | Định nghĩa |
|-----------|-----------|
| AAA | Authentication, Authorization, Accounting — khung bảo mật truy cập mở rộng |
| Local AAA | Xác thực dựa trên dữ liệu lưu cục bộ trên thiết bị mạng |
| Server-based AAA | Xác thực thông qua server AAA tập trung |
| RADIUS | Remote Authentication Dial-In User Service — giao thức AAA chuẩn mở |
| TACACS+ | Terminal Access Controller Access-Control System Plus — giao thức AAA Cisco |
| LDAP | Lightweight Directory Access Protocol — giao thức truy cập thư mục |
| Active Directory | Dịch vụ thư mục Microsoft quản lý tài khoản và chính sách trong domain |
| Network accounting | Ghi log thông tin phiên mạng: bytes và packets |
| EXEC accounting | Ghi log phiên terminal người dùng |
| Command accounting | Ghi log từng lệnh được thực thi và người dùng |
| Start record | Bản ghi AAA đánh dấu bắt đầu phiên |
| Stop record | Bản ghi AAA đánh dấu kết thúc phiên |

## Ghi chú chi tiết

**Tại sao server-based AAA tốt hơn local AAA ở quy mô lớn?**

Với local AAA, mỗi router và switch có danh sách username/password riêng. Khi cần đổi mật khẩu của một admin, phải đăng nhập vào từng thiết bị và đổi thủ công. Khi admin nghỉ việc, phải nhớ xóa tài khoản trên tất cả thiết bị. Với server-based AAA, tất cả thiết bị đều query một AAA server tập trung. Thay đổi trên server áp dụng ngay lập tức cho toàn bộ mạng.

**TACACS+ và phân quyền lệnh:**

Đây là ưu điểm độc đáo của TACACS+. Ví dụ: Junior network admin có thể dùng lệnh `show` (xem cấu hình) nhưng không được dùng lệnh `configure terminal` (sửa cấu hình). Senior admin có thể dùng tất cả lệnh. RADIUS không hỗ trợ kiểm soát ở cấp độ lệnh này. Tính năng này rất quan trọng cho các tổ chức muốn kiểm soát chặt chẽ thay đổi cấu hình mạng.

**Giá trị của AAA logs trong điều tra:**

Kịch bản: server bị xâm phạm vào 2 giờ sáng thứ 7. AAA log cho thấy tài khoản "netadmin" đăng nhập vào router lúc 1:47 sáng từ IP không thường gặp, sau đó thực thi lệnh `ip route 0.0.0.0 0.0.0.0 [IP kẻ tấn công]`. Đây là bằng chứng trực tiếp, có timestamp, cho điều tra pháp lý. Nếu không có logging, sự kiện này sẽ biến mất khi router reboot.

**Compliance và AAA accounting:**

Nhiều quy định như PCI-DSS (thanh toán thẻ), HIPAA (y tế), SOX (tài chính) yêu cầu ghi log tất cả truy cập vào hệ thống chứa dữ liệu nhạy cảm và lưu giữ log trong thời gian nhất định (thường 1-7 năm). AAA accounting tự động hóa việc thu thập và tập trung hóa log này.

## Ví dụ thực tế

**Ví dụ cấu hình TACACS+:**

Trong môi trường Cisco, cấu hình router dùng TACACS+ server để xác thực:
- Router gửi thông tin đăng nhập đến TACACS+ server.
- Server kiểm tra trong cơ sở dữ liệu (có thể tích hợp AD).
- Nếu xác thực thành công, server gửi profile authorization (lệnh nào được phép).
- Router ghi log accounting về phiên làm việc.

**Ví dụ cấu hình RADIUS cho Wi-Fi doanh nghiệp:**

Nhân viên kết nối Wi-Fi với username công ty. Access point gửi credential đến RADIUS server. Server xác thực với Active Directory. Nếu thành công, gửi VLAN assignment phù hợp (nhân viên vào VLAN nội bộ; khách vào VLAN guest). RADIUS accounting ghi thời gian kết nối và lưu lượng sử dụng.

**So sánh cấu hình thực tế:**

Router R1 với local AAA: lưu tất cả user/password trong running-config — không mã hóa, phụ thuộc thiết bị.

Router R1 với TACACS+: chỉ cần IP và secret key của TACACS+ server — tất cả user được quản lý tập trung, mã hóa, audit đầy đủ.

## Cần ghi nhớ

- **TACACS+ mã hóa toàn bộ gói tin; RADIUS chỉ mã hóa mật khẩu**.
- **TACACS+ phân quyền theo lệnh; RADIUS không có tính năng này**.
- **Server-based AAA là lựa chọn cho mạng từ trung bình trở lên**.
- **AAA accounting logs là bằng chứng pháp lý và công cụ compliance**.
- **RADIUS là chuẩn mở; TACACS+ chủ yếu Cisco**.
- **Stop record cho kết nối thất bại** giúp phát hiện tấn công brute force hoặc tài khoản bị chiếm.

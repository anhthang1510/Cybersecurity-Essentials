# 19.2 - Security Technologies

## Tổng quan

Phần này giới thiệu các công nghệ bảo mật dùng để thu thập, phát hiện, ngăn chặn và tương quan sự kiện. Mỗi công nghệ có phạm vi quan sát khác nhau, nên cần kết hợp nhiều lớp để có cái nhìn đầy đủ.

## Ý chính

- Firewall kiểm soát traffic và tạo logs về allow/deny, NAT, session hoặc policy hits.
- IDS phát hiện hành vi đáng ngờ và sinh alert.
- IPS có thể chặn, reset hoặc drop traffic đáng ngờ.
- SIEM thu thập và correlation log từ nhiều nguồn.
- NetFlow/IPFIX cung cấp metadata về luồng traffic.
- Packet capture giúp phân tích sâu khi cần bằng chứng chi tiết.
- Proxy, WAF, EDR và NDR bổ sung góc nhìn theo ứng dụng, endpoint và mạng.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Firewall log | Log từ firewall về traffic được cho phép/chặn hoặc policy match. |
| IDS alert | Cảnh báo từ hệ thống phát hiện xâm nhập. |
| IPS action | Hành động ngăn chặn như drop, reset hoặc block. |
| SIEM correlation | Liên kết nhiều sự kiện để tạo ngữ cảnh bảo mật. |
| NetFlow | Flow telemetry phổ biến ghi metadata traffic. |
| IPFIX | Chuẩn xuất flow information mở rộng từ NetFlow. |
| PCAP | Packet capture file chứa packet để phân tích. |
| WAF | Web Application Firewall bảo vệ ứng dụng web. |
| EDR | Endpoint Detection and Response, giám sát và phản ứng trên endpoint. |
| NDR | Network Detection and Response, phân tích hành vi mạng. |

## Ghi chú chi tiết

**IDS và IPS:**

IDS chủ yếu phát hiện và cảnh báo, phù hợp để quan sát mà không ảnh hưởng trực tiếp tới traffic. IPS nằm inline hoặc có khả năng can thiệp, nên có thể giảm rủi ro nhanh hơn nhưng cũng cần tuning cẩn thận để tránh false positive làm gián đoạn dịch vụ.

**SIEM:**

SIEM không tự tạo dữ liệu bảo mật nếu nguồn log không đầy đủ. Giá trị của SIEM đến từ việc chuẩn hóa, lưu trữ, correlation và alerting. Một rule SIEM tốt thường kết hợp nhiều dấu hiệu thay vì chỉ dựa vào một log đơn lẻ.

**Flow vs packet:**

Flow data cho biết ai nói chuyện với ai, port nào, bao lâu và bao nhiêu bytes. Packet capture cho chi tiết sâu hơn nhưng tốn dung lượng và có thể chứa dữ liệu nhạy cảm. Flow phù hợp quan sát diện rộng; PCAP phù hợp điều tra sâu có mục tiêu.

**Firewall, proxy và WAF:**

Firewall kiểm soát network traffic theo policy. Proxy có thể ghi nhận user, URL và nội dung web ở mức ứng dụng. WAF tập trung bảo vệ web application khỏi các pattern như SQL injection hoặc cross-site scripting.

**EDR và NDR:**

EDR nhìn từ endpoint: process, file, registry, command line, user activity. NDR nhìn từ network: flow, protocol behavior và lateral movement. Hai nguồn này bổ sung cho nhau trong điều tra sự cố.

## Ví dụ thực tế

**Ví dụ 1: IDS alert cần ngữ cảnh**

IDS báo exploit attempt tới web server. Analyst kiểm tra WAF logs, web server logs và EDR trên server. Nếu request bị WAF chặn và server không có process lạ, mức độ ưu tiên có thể thấp hơn so với trường hợp exploit thành công.

**Ví dụ 2: NetFlow phát hiện exfiltration**

Một server nội bộ gửi lượng dữ liệu lớn tới IP lạ vào ban đêm. NetFlow cho thấy byte count tăng đột biến. Firewall logs và endpoint logs giúp xác định process nào tạo kết nối.

**Ví dụ 3: SIEM correlation**

SIEM liên kết đăng nhập VPN thành công từ quốc gia lạ, sau đó truy cập file server lớn bất thường và DNS query tới domain hiếm. Correlation giúp tạo alert có ngữ cảnh mạnh hơn từng log riêng lẻ.

## Cần ghi nhớ

- **IDS phát hiện, IPS có thể ngăn chặn**: hai công nghệ liên quan nhưng không giống nhau.
- **SIEM cần nguồn dữ liệu tốt**: garbage in, garbage out.
- **Flow phù hợp quan sát diện rộng**: không có payload chi tiết như PCAP.
- **PCAP cần dùng có mục tiêu**: dữ liệu lớn và có thể nhạy cảm.
- **Công nghệ bổ sung nhau**: firewall, IDS/IPS, SIEM, EDR, NDR không thay thế hoàn toàn nhau.

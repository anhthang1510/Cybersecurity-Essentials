# 19.3 - Technologies and Protocols Summary

## Tổng quan

Module 19 tổng kết cách protocol monitoring và security technologies hỗ trợ phát hiện, phân tích và phản ứng sự cố. Điểm quan trọng là hiểu mỗi nguồn dữ liệu trả lời loại câu hỏi nào và kết hợp chúng để tạo ngữ cảnh.

## Ý chính

| Chủ đề | Tóm tắt |
|---|---|
| Protocol monitoring | Quan sát DNS, HTTP/HTTPS, DHCP, authentication, email và remote access để tìm dấu hiệu bất thường. |
| Packet capture | Cung cấp chi tiết packet nhưng tốn dung lượng và có thể chứa dữ liệu nhạy cảm. |
| Flow data | Cung cấp metadata luồng traffic để quan sát diện rộng. |
| IDS/IPS | Phát hiện hoặc ngăn chặn traffic có dấu hiệu tấn công. |
| Firewall/proxy/WAF | Ghi nhận và kiểm soát traffic ở network hoặc application layer. |
| SIEM | Tập trung log, correlation và alerting. |
| EDR/NDR | Bổ sung góc nhìn endpoint và network behavior. |

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Protocol log | Log đã được phân tích theo giao thức cụ thể. |
| Alert triage | Quá trình đánh giá, ưu tiên và xử lý cảnh báo. |
| False positive | Cảnh báo không đại diện cho sự cố thật. |
| Correlation | Liên kết nhiều sự kiện để tạo ngữ cảnh. |
| Baseline | Mẫu hành vi bình thường dùng để phát hiện bất thường. |
| Telemetry | Dữ liệu đo lường và quan sát từ hệ thống. |

## Ghi chú chi tiết

Không có công nghệ nào nhìn thấy tất cả. Firewall biết policy allow/deny, DNS logs biết domain query, flow biết metadata kết nối, EDR biết process trên endpoint, còn SIEM giúp liên kết các dấu hiệu này. Analyst cần hiểu giới hạn của từng nguồn để tránh kết luận quá sớm.

## Mối liên hệ giữa các bài

```
Common protocols
    ↓
Protocol logs và metadata
    ↓
Firewall, IDS/IPS, proxy, NetFlow, EDR/NDR
    ↓
SIEM correlation
    ↓
Alert triage và incident response
```

## Ví dụ thực tế

Một host bị nghi nhiễm malware. DNS logs cho thấy domain lạ, NetFlow cho thấy kết nối định kỳ ra ngoài, EDR cho thấy process lạ chạy từ thư mục tạm, firewall logs cho thấy traffic được phép ra port 443. Khi kết hợp các nguồn, analyst có thể tự tin hơn so với chỉ nhìn một alert đơn lẻ.

## Cần ghi nhớ

- **Protocol monitoring tạo ngữ cảnh**: hiểu protocol giúp đọc log đúng hơn.
- **IDS/IPS không giống nhau**: phát hiện và ngăn chặn là hai vai trò khác nhau.
- **Flow không thay thế PCAP**: flow là metadata, PCAP là chi tiết packet.
- **SIEM phụ thuộc chất lượng log**: nguồn dữ liệu thiếu hoặc sai thời gian làm correlation kém.
- **Phân tích tốt cần nhiều nguồn**: endpoint, network, identity và application logs nên được kết hợp.

# 1.5 - Application Attacks

## Tổng quan

Phần này trình bày các cuộc tấn công vào ứng dụng web, database, email và browser. Application attacks thường khai thác lỗi code, xử lý input yếu, cấu hình sai, endpoint API hoặc hành vi người dùng khi mở email, link và file đính kèm.

## Ý chính

- Web applications ngày càng là mục tiêu phổ biến vì chúng kết nối người dùng với database và server.
- XSS cho phép attacker chèn script độc hại vào trang web và chạy trong trình duyệt nạn nhân.
- Injection attacks khai thác input không được lọc đúng cách, gồm XML, SQL, DLL và LDAP injection.
- Buffer overflow xảy ra khi dữ liệu vượt quá giới hạn bộ nhớ.
- Remote code execution cho phép chạy lệnh trên mục tiêu với quyền của ứng dụng/người dùng.
- CSRF, race condition, improper input handling, error handling, API attack, replay, directory traversal và resource exhaustion đều là application attacks quan trọng.
- Spam, phishing, vishing, pharming và whaling khai thác email/browser và niềm tin người dùng.
- Phòng thủ cần secure coding, input validation, patching, scanning attachment, anti-phishing awareness và cập nhật phần mềm.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| XSS | Cross-Site Scripting, chèn script độc hại vào web app để chạy trong browser của nạn nhân. |
| Injection attack | Đưa dữ liệu/lệnh độc hại vào input để thao túng ứng dụng hoặc backend. |
| SQL injection | Chèn SQL độc hại vào input để truy cập, sửa hoặc xóa dữ liệu. |
| XML injection | Thao túng xử lý XML để làm sai dữ liệu hoặc truy cập trái phép. |
| DLL injection | Ép ứng dụng gọi DLL độc hại trong tiến trình mục tiêu. |
| LDAP injection | Chèn truy vấn độc hại vào LDAP để lấy thông tin directory. |
| Buffer overflow | Ghi dữ liệu vượt giới hạn buffer, có thể gây crash hoặc thực thi mã. |
| RCE | Remote Code Execution, thực thi lệnh từ xa trên hệ thống mục tiêu. |
| Privilege escalation | Nâng quyền từ mức thấp lên mức cao hơn do bug, misconfiguration hoặc design flaw. |
| CSRF | Cross-Site Request Forgery, ép browser người dùng gửi lệnh trái phép đến web app đã tin cậy. |
| API attack | Lạm dụng API endpoint để truy cập dữ liệu hoặc thao túng hệ thống. |
| Directory traversal | Truy cập file ngoài thư mục web cho phép. |
| Spam | Email không mong muốn, có thể chứa link hoặc attachment độc hại. |
| Phishing | Lừa người dùng cung cấp thông tin bằng email, tin nhắn hoặc trang giả mạo. |
| Vishing | Voice phishing qua cuộc gọi hoặc voice technology. |
| Pharming | Chuyển hướng người dùng đến website giả dù họ nghĩ đang truy cập site thật. |

## Ghi chú chi tiết

### Web application attacks

Web app thường kết nối với database và server. Nếu code xử lý input kém, attacker có thể dùng form, URL, header hoặc API request để đưa dữ liệu độc hại vào hệ thống.

### Cross-Site Scripting

XSS xảy ra khi attacker chèn script độc hại vào web page. Khi nạn nhân mở trang, script chạy trong browser và có thể lấy cookie, session token hoặc thông tin nhạy cảm. Từ đó attacker có thể impersonate user.

### Code injection

Các loại injection quan trọng:

- XML injection: can thiệp cách ứng dụng xử lý XML query/data.
- SQL injection: chèn SQL statement vào input để đọc, sửa, xóa dữ liệu hoặc chiếm quyền database.
- DLL injection: lừa ứng dụng gọi DLL độc hại trong Windows process.
- LDAP injection: khai thác input validation yếu để truy vấn LDAP directory trái phép.

Điểm chung là input từ người dùng không được xem là đáng tin cậy. Tất cả input bên ngoài cần được validate, sanitize và xử lý theo nguyên tắc least privilege.

### Buffer overflow và RCE

Buffer overflow xảy ra khi dữ liệu được ghi vượt quá vùng nhớ cấp phát. Hậu quả có thể là crash, rò rỉ dữ liệu, privilege escalation hoặc attacker điều khiển luồng thực thi.

Remote code execution cho phép attacker chạy lệnh trên target. Nếu ứng dụng chạy với quyền cao, hậu quả càng nghiêm trọng. Privilege escalation có thể kết hợp với RCE để mở rộng quyền trong hệ thống.

### Other application attacks

- CSRF: browser của user đã đăng nhập bị lừa gửi request đến web app tin cậy.
- Race condition: hệ thống bị ép xử lý hai thao tác cùng lúc ngoài trình tự mong muốn.
- Improper input handling: input không được kiểm tra gây SQL injection, buffer overflow hoặc lỗi logic.
- Error handling attack: thông báo lỗi tiết lộ hostname, path, table, field hoặc thông tin hệ thống.
- API attack: attacker lạm dụng endpoint, token, rate limit hoặc authorization yếu.
- Replay attack: gửi lại transmission hợp lệ đã bị chặn bắt.
- Directory traversal: dùng đường dẫn như `../` để đọc file ngoài thư mục cho phép.
- Resource exhaustion: làm cạn tài nguyên phần cứng/phần mềm của server.

### Email và browser attacks

Spam là email không mong muốn. Một số spam chứa link, malware hoặc nội dung lừa đảo. Dấu hiệu đáng ngờ có thể gồm không có subject, yêu cầu cập nhật tài khoản, lỗi chính tả, link dài/khó hiểu, khác biệt nhỏ so với email thật hoặc yêu cầu mở attachment gấp.

Phishing là lừa người dùng cung cấp thông tin. Spear phishing nhắm đến cá nhân cụ thể bằng thông tin đã thu thập. Vishing dùng cuộc gọi, pharming chuyển hướng sang website giả, whaling nhắm vào lãnh đạo hoặc mục tiêu giá trị cao.

### Defending against application attacks

- Viết code chắc chắn và xem mọi input từ bên ngoài là hostile.
- Validate và sanitize input.
- Cập nhật OS, application, framework và library.
- Không bỏ qua update prompts.
- Dùng security testing, code review và dependency scanning.
- Không giả định email attachment an toàn, kể cả từ người quen.
- Scan attachment trước khi mở.
- Đào tạo người dùng về spam/phishing.
- Dùng antivirus, email filtering và báo cáo email đáng ngờ.

### Các attack khác

Module cũng nhắc đến:

- Physical attacks: cắm USB chứa malware, cáp/adapter độc hại, skimming thẻ.
- Adversarial AI attacks: làm bẩn dữ liệu đầu vào để mô hình machine learning dự đoán sai.
- Supply chain attacks: can thiệp vào phần mềm, component hoặc quy trình của bên thứ ba.
- Cloud-based attacks: khai thác dữ liệu, app, platform hoặc infrastructure trên cloud.

## Ví dụ thực tế

Một website login không lọc input đúng cách. Attacker thử SQL injection trong ô username và lấy được danh sách tài khoản. Sau đó attacker dùng credential để đăng nhập, tìm API endpoint thiếu kiểm tra quyền và tải dữ liệu khách hàng. Tình huống này kết hợp SQL injection, broken access control và API abuse.

## Cần ghi nhớ

- Ứng dụng là cầu nối đến dữ liệu, nên lỗi ứng dụng thường có tác động lớn.
- Input từ người dùng không bao giờ nên được tin tưởng mặc định.
- XSS đánh vào browser và session của user.
- Injection đánh vào backend hoặc logic xử lý dữ liệu.
- Email/browser attacks thường kết hợp deception và malware.
- Secure coding và patching là tuyến phòng thủ quan trọng.

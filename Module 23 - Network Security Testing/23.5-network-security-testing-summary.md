# 23.5 - Tóm Tắt Module (Network Security Testing Summary)

## Tổng quan

Bài tóm tắt này hệ thống lại toàn bộ kiến thức của Module 23, từ các loại security assessment, kỹ thuật kiểm thử, công cụ phổ biến, đến quy trình penetration testing chuyên nghiệp. Module này cung cấp nền tảng thực hành cho mọi security professional.

---

## Ý chính

- Security testing phải chủ động và định kỳ, không chỉ một lần
- Mọi testing phải có authorization rõ ràng bằng văn bản
- Công cụ (Nmap, Wireshark) chỉ mạnh khi người dùng hiểu kết quả
- Pentest có 5 giai đoạn, báo cáo là sản phẩm quan trọng nhất
- Wireshark chứng minh trực quan tầm quan trọng của mã hóa

## Ghi chú chi tiết

### Tổng kết: Security Assessments (Module 23.1)

**Các loại assessment:**

```
Vulnerability Scan -> Risk Assessment -> Security Audit -> Penetration Test
   (tự động)           (toàn diện)        (tuân thủ)         (tấn công thật)
```

**Credential vs Non-credential scan:**
- Credentialed: thấy bên trong hệ thống, nhiều chi tiết hơn
- Non-credentialed: góc nhìn attacker từ ngoài

**CVSS score:**

| 0-3.9 | 4.0-6.9 | 7.0-8.9 | 9.0-10.0 |
|-------|---------|---------|----------|
| Low | Medium | High | Critical |
| Theo kế hoạch | Trong tháng | Trong tuần | Ngay lập tức |

**Quy trình remediation:**
1. Triage và ưu tiên
2. Assign cho người chịu trách nhiệm
3. Track qua ticket system
4. Verify sau khi vá (re-scan)
5. Report lên management

### Tổng kết: Kỹ thuật kiểm thử (Module 23.2)

**Phân loại test:**

| Góc nhìn | Loại | Giải thích |
|----------|------|------------|
| Kiến thức | Black/White/Gray box | Tester biết bao nhiêu về hệ thống |
| Vị trí | Internal/External | Từ trong hay ngoài mạng |
| Tần suất | One-time/Periodic/Continuous | Một lần, định kỳ, liên tục |

**OpSec trong testing:**
- Minimize footprint
- Document everything
- Communicate trước
- Prepare rollback
- Protect test credentials

### Tổng kết: Công cụ (Module 23.3)

**Nmap states:**
- `open`: Service đang chạy
- `closed`: Không có service, port reachable
- `filtered`: Firewall chặn, không biết state

**Nmap command chính:**

```bash
nmap -sn [subnet]    # Host discovery (lab)
nmap [host]           # Basic scan
nmap -sV [host]       # Version detection
nmap -O [host]        # OS detection
nmap -F [host]        # Fast scan (top 100 port)
```

**Wireshark - Telnet vs SSH:**

| Giao thức | Port | Packet capture thấy gì? |
|-----------|------|------------------------|
| Telnet | 23 | Cleartext: username, password, lệnh |
| SSH | 22 | Encrypted: chỉ thấy metadata |

**SIEM chức năng:**
Log aggregation -> Normalization -> Correlation -> Alerting -> Forensics -> Reporting

### Tổng kết: Penetration Testing (Module 23.4)

**5 giai đoạn:**

```
1. Reconnaissance -> 2. Scanning -> 3. Gaining Access 
        -> 4. Maintaining Access -> 5. Reporting
```

**Điều kiện tiên quyết LUÔN LUÔN:**
- Authorization bằng văn bản từ người có thẩm quyền
- Scope document rõ ràng
- ROE ký kết trước khi bắt đầu

**Exercise types:**

| Type | Ai biết? | Mục đích |
|------|----------|---------|
| Red team | Blue team không biết | Test detection thực tế |
| Blue team | Biết có exercise | Cải thiện quy trình phòng thủ |
| Purple team | Cả hai phối hợp | Tối ưu hóa detection nhanh |
| White team | Biết tất cả | Trọng tài, giám sát |

## Ví dụ thực tế

**Kịch bản tổng hợp: Chương trình bảo mật toàn diện**

Tổ chức X muốn xây dựng chương trình kiểm thử bảo mật chuyên nghiệp:

*Cấp độ 1 - Continuous (tự động hàng ngày):*
- Vulnerability scanner chạy tự động
- SIEM monitoring real-time
- Alert khi phát hiện lỗ hổng Critical mới

*Cấp độ 2 - Periodic (hàng tháng):*
- Credentialed scan toàn bộ network
- Review và update firewall rules
- Kiểm tra compliance với policy

*Cấp độ 3 - Annual (hàng năm):*
- External penetration test bởi third party
- Red team exercise để test detection capability
- Review và update security program

*Áp dụng kết quả:*
- Mỗi finding -> ticket với priority và owner
- Vulnerability Critical: SLA 24-48h
- Monthly review meeting với management

## Cần ghi nhớ

**10 điểm quan trọng nhất của Module 23:**

1. Authorization trước tiên, luôn luôn, không có ngoại lệ
2. CVSS 9.0+ = Critical = xử lý trong 24-48h
3. Credentialed scan thấy nhiều hơn non-credentialed
4. Nmap `-sS` = SYN scan, nhanh hơn `-sT`, cần root
5. `filtered` trong Nmap = firewall đang chặn (không xác định open/closed)
6. Telnet = cleartext; SSH = encrypted; Wireshark chứng minh trực quan
7. 5 giai đoạn pentest: Recon -> Scan -> Access -> Maintain -> Report
8. Black box giống attacker nhất; White box có coverage cao nhất
9. Purple team = hiệu quả nhất để cải thiện detection nhanh
10. Báo cáo actionable là mục tiêu cuối cùng, không phải số lỗ hổng tìm được

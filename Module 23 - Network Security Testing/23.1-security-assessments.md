# 23.1 - Đánh Giá Bảo Mật (Security Assessments)

## Tổng quan

Security assessment là quy trình có hệ thống để **đánh giá trạng thái bảo mật** của tổ chức. Đây là bước đầu tiên trong mọi chương trình bảo mật chủ động, giúp xác định điểm mạnh, điểm yếu, và ưu tiên cải thiện.

---

## Ý chính

- Security assessment bao gồm nhiều loại: vulnerability scan, security audit, risk assessment
- Vulnerability scanner tự động tìm lỗ hổng đã biết trong hệ thống và ứng dụng
- Có nhiều loại scan: credentialed, non-credentialed, internal, external
- Security automation giúp tăng tần suất và độ phủ của assessment
- Kết quả assessment phải được ưu tiên và hành động cụ thể, không chỉ báo cáo

---

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|-----------|---------|
| Vulnerability assessment | Quá trình tìm và phân tích lỗ hổng bảo mật trong hệ thống |
| Vulnerability scanner | Công cụ tự động quét tìm lỗ hổng đã biết |
| Credentialed scan | Scan với tài khoản hợp lệ để xem kỹ hơn bên trong hệ thống |
| Non-credentialed scan | Scan không có tài khoản, nhìn từ góc độ attacker bên ngoài |
| CVE | Common Vulnerabilities and Exposures - danh sách lỗ hổng đã biết |
| CVSS | Common Vulnerability Scoring System - thang điểm mức độ nghiêm trọng |
| Risk assessment | Đánh giá rủi ro: xác suất và tác động của các mối đe dọa |
| Security audit | Kiểm tra tuân thủ chính sách, chuẩn mực, và yêu cầu pháp lý |
| Remediation | Khắc phục lỗ hổng sau khi được phát hiện |
| Security automation | Tự động hóa các tác vụ bảo mật lặp lại |

## Ghi chú chi tiết

### Các loại Security Assessment

**1. Vulnerability Assessment**

Mục đích: Tìm và phân loại lỗ hổng bảo mật.

Quy trình:
1. Xác định scope (hệ thống nào cần scan)
2. Chọn công cụ phù hợp
3. Thực hiện scan
4. Phân tích kết quả và loại bỏ false positive
5. Ưu tiên theo CVSS score
6. Báo cáo và khắc phục

**2. Risk Assessment**

Mục đích: Xác định rủi ro tổng thể, không chỉ lỗ hổng kỹ thuật.

```
Risk = Likelihood (xác suất xảy ra) x Impact (tác động)
```

Bao gồm cả rủi ro từ: lỗ hổng kỹ thuật, quy trình kém, con người, và môi trường.

**3. Security Audit**

Mục đích: Kiểm tra tuân thủ với policy, standard, hoặc regulation.

Khác vulnerability assessment ở chỗ: audit đánh giá *có tuân thủ quy trình không*, không chỉ *có lỗ hổng kỹ thuật không*.

### Vulnerability Scanner: Nguyên lý hoạt động

Vulnerability scanner hoạt động bằng cách:
1. **Discovery**: Tìm host đang hoạt động trong network
2. **Port scanning**: Xác định port và service đang mở
3. **Service detection**: Xác định phiên bản phần mềm
4. **Vulnerability matching**: So sánh với database CVE/plugin

**Các loại scan:**

| Loại scan | Đặc điểm | Khi nào dùng |
|-----------|----------|-------------|
| Credentialed | Có tài khoản, thấy cả bên trong | Internal assessment, kiểm tra patch |
| Non-credentialed | Không có tài khoản, thấy từ ngoài vào | External attacker perspective |
| Internal | Từ bên trong mạng | Phát hiện lateral movement risks |
| External | Từ internet, qua firewall | Kiểm tra attack surface công khai |
| Agentless | Không cần agent trên target | Network-based, ít xâm phạm |
| Agent-based | Cài agent trên target | Kiểm tra chi tiết hơn, offline host |

### CVSS Score và ưu tiên

CVSS (Common Vulnerability Scoring System) cho điểm 0-10:

| Điểm | Mức độ | Ưu tiên xử lý |
|------|--------|---------------|
| 9.0-10.0 | Critical | Ngay lập tức (24-48h) |
| 7.0-8.9 | High | Trong tuần |
| 4.0-6.9 | Medium | Trong tháng |
| 0.1-3.9 | Low | Theo kế hoạch |
| 0.0 | None | Không cần xử lý |

**Lưu ý:** CVSS score là điểm kỹ thuật, không phải rủi ro kinh doanh. Một lỗ hổng Medium trên hệ thống quan trọng có thể ưu tiên cao hơn Critical trên hệ thống không quan trọng.

### Security Automation

Security automation giúp tăng hiệu quả assessment:

**Lợi ích:**
- Tăng tần suất scan (có thể daily thay vì monthly)
- Giảm sai sót con người
- Phản ứng nhanh hơn với lỗ hổng mới

**Ví dụ automation:**
- Tự động scan khi deploy code mới (CI/CD pipeline)
- Tự động tạo ticket khi phát hiện lỗ hổng Critical
- Tự động cập nhật signature database

**Hạn chế của automation:**
- Cần con người phân tích kết quả và ra quyết định
- False positive vẫn cần được lọc thủ công
- Không thể thay thế creative thinking của pentester

### Áp dụng kết quả assessment

Kết quả assessment không có giá trị nếu không dẫn đến hành động:

**Quy trình remediation:**
1. **Triage**: Phân loại và ưu tiên theo business risk
2. **Assign**: Giao nhiệm vụ cho team/cá nhân phù hợp
3. **Track**: Theo dõi tiến trình (ticket system)
4. **Verify**: Xác nhận đã vá thành công (re-scan)
5. **Report**: Báo cáo lên management

## Lệnh / Ví dụ minh họa

> **Chú ý**: Tất cả ví dụ dưới đây chỉ dành cho môi trường lab hoặc hệ thống có sự cho phép hợp lệ.

### Kiểm tra kết nối cơ bản (lab)

```bash
# Ping để xác nhận host đang hoạt động trong lab
ping 192.168.1.1

# Traceroute để xem đường đi packet (Windows)
tracert 192.168.1.1

# Traceroute (Linux/Mac)
traceroute 192.168.1.1
```

### DNS lookup (thông tin công khai)

```bash
# Tra cứu DNS record (thông tin công khai)
nslookup example.com

# Tra cứu với server DNS cụ thể
nslookup example.com 8.8.8.8
```

### Netstat để xem port đang mở trên máy mình

```bash
# Xem port đang lắng nghe trên máy MÌNH (không cần quyền scan)
netstat -an | findstr LISTENING   # Windows
netstat -tlnp                      # Linux
```

## Ví dụ thực tế

**Scenario: Monthly vulnerability scan trong môi trường doanh nghiệp**

1. IT Security team schedule scan hàng tháng vào cuối tuần
2. Scan credentialed cho tất cả server production
3. Kết quả: 3 lỗ hổng Critical, 12 High, 45 Medium
4. Ưu tiên: 3 Critical liên quan đến patch chưa áp dụng trên web server
5. Remediation: Apply patch, re-scan trong 48h để xác nhận
6. Track: Mở ticket, assign cho system admin, deadline 24h

---

## Cần ghi nhớ

- Credentialed scan thấy nhiều hơn nhưng cần bảo vệ tài khoản scan
- CVSS score = mức độ kỹ thuật, không phải business risk
- False positive phải được lọc trước khi báo cáo
- Automation tốt cho tần suất, nhưng con người vẫn cần phân tích kết quả
- Kết quả assessment chỉ có giá trị khi được hành động

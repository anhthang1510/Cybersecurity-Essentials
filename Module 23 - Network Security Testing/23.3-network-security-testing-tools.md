# 23.3 - Công Cụ Kiểm Thử Bảo Mật Mạng (Network Security Testing Tools)

## Tổng quan

Công cụ kiểm thử bảo mật là phương tiện, không phải mục đích. Bài này trình bày các công cụ phổ biến nhất mà mọi security professional cần biết: **Nmap/Zenmap**, **SuperScan**, **SIEM**, và **packet analyzer (Wireshark)**. Trọng tâm là hiểu nguyên lý hoạt động và biết dùng đúng chỗ.

---

## Ý chính

- Nmap là công cụ scan mạng nền tảng, mọi security professional cần thành thạo
- Zenmap là GUI của Nmap, thân thiện hơn cho người mới
- SuperScan là công cụ scan port cho Windows, nhanh và đơn giản
- SIEM không chỉ để giám sát mà còn hỗ trợ forensics và compliance
- Wireshark cho phép phân tích sâu protocol và phát hiện dữ liệu không mã hóa
- Mọi công cụ này phải dùng trong phạm vi được phép

## Ghi chú chi tiết

### Nmap: Network Mapper

Nmap là công cụ scan mạng phổ biến và mạnh mẽ nhất. Được dùng bởi cả defender lẫn (authorized) penetration tester.

**Chức năng chính của Nmap:**
- Host discovery: Tìm host đang hoạt động trong mạng
- Port scanning: Xác định port nào đang mở/đóng/filtered
- Service detection: Xác định phần mềm và phiên bản
- OS detection: Nhận dạng hệ điều hành
- Scripting: Nmap Scripting Engine (NSE) để mở rộng chức năng

**Các loại scan Nmap:**

| Loại scan | Flag | Đặc điểm |
|-----------|------|----------|
| TCP Connect | `-sT` | Hoàn chỉnh TCP handshake, dễ bị phát hiện |
| SYN scan | `-sS` | "Half-open", nhanh, ít trace (cần root) |
| UDP scan | `-sU` | Chậm hơn, phát hiện UDP service |
| Version detect | `-sV` | Xác định phiên bản service |
| OS detect | `-O` | Nhận dạng OS (cần root) |
| Ping scan | `-sn` | Chỉ tìm host alive, không scan port |
| All ports | `-p-` | Scan toàn bộ 65535 port |

**Output format:**
- `-oN`: Normal output (text)
- `-oX`: XML output
- `-oG`: Grepable output
- `-oA`: Tất cả ba format

### Nmap trong lab (ví dụ authorized)

> **Chỉ thực hiện trên mạng lab của mình hoặc với sự cho phép bằng văn bản.**

```bash
# Scan cơ bản một host trong lab (ping + port scan)
nmap 192.168.1.100

# Scan range IP trong mạng lab
nmap 192.168.1.0/24

# Chỉ tìm host alive trong lab (không scan port)
nmap -sn 192.168.1.0/24

# Scan nhanh top 100 port phổ biến
nmap -F 192.168.1.100

# Xác định OS và phiên bản service (cần quyền)
nmap -O -sV 192.168.1.100

# Scan với output lưu file
nmap -oA /tmp/lab-scan 192.168.1.0/24
```

**Giải thích kết quả Nmap:**

```
PORT     STATE    SERVICE    VERSION
22/tcp   open     ssh        OpenSSH 8.4p1
80/tcp   open     http       Apache httpd 2.4.51
443/tcp  open     https      Apache httpd 2.4.51
3306/tcp filtered mysql
8080/tcp closed   http-alt
```

- **open**: Port đang lắng nghe, có service
- **closed**: Port không có service nhưng có thể kết nối được
- **filtered**: Firewall hoặc filter đang chặn, không biết state

### Zenmap: GUI cho Nmap

Zenmap là giao diện đồ họa của Nmap, phù hợp khi:
- Mới học Nmap, cần nhìn thấy command được tạo
- Cần network topology visualization
- Lưu và so sánh kết quả scan giữa các lần

**Profile scan có sẵn trong Zenmap:**
- Intense scan: `-T4 -A -v`
- Quick scan: `-T4 -F`
- Ping scan: `-sn`
- Regular scan: không flag đặc biệt

### SuperScan

SuperScan là công cụ scan port của Foundstone (nay là McAfee) dành cho Windows.

**Đặc điểm:**
- Giao diện đồ họa đơn giản cho Windows
- Scan port và hostname lookup
- Banner grabbing
- Tích hợp ping và traceroute
- Nhanh và nhẹ

**Khi nào dùng SuperScan thay Nmap?**
- Môi trường Windows-only, không cài Nmap
- Cần giao diện đơn giản cho non-technical staff
- Quick scan trong lab Windows

### SIEM: Security Information and Event Management

SIEM là hệ thống trung tâm thu thập, lưu trữ, và phân tích sự kiện bảo mật.

**Chức năng chính:**
1. **Log aggregation**: Thu thập log từ nhiều nguồn (firewall, IDS, server, endpoint)
2. **Normalization**: Chuẩn hóa format log khác nhau
3. **Correlation**: Tương quan sự kiện từ nhiều nguồn
4. **Alerting**: Tạo alert khi phát hiện pattern nguy hiểm
5. **Dashboards**: Hiển thị trực quan trạng thái bảo mật
6. **Forensics**: Tìm kiếm lịch sử sự kiện cho điều tra
7. **Compliance reporting**: Tạo báo cáo tuân thủ

**Các SIEM phổ biến:**

| SIEM | Đặc điểm |
|------|----------|
| Splunk | Enterprise, rất mạnh, license theo data volume |
| Elastic SIEM | Open source, linh hoạt, cần setup nhiều |
| IBM QRadar | Enterprise, tốt cho compliance |
| Microsoft Sentinel | Cloud-native, tích hợp Azure |
| Security Onion | Open source, tích hợp nhiều tool, tốt cho lab |

**SIEM trong kiểm thử bảo mật:**
- Kiểm tra xem activity test có được log không (detection gap assessment)
- So sánh baseline với trạng thái sau khi thay đổi cấu hình
- Xác nhận alert rules đang hoạt động đúng

### Packet Analyzer: Wireshark

Wireshark là công cụ phân tích gói tin mạnh nhất hiện có, mã nguồn mở.

**Chức năng:**
- Bắt gói tin real-time trên network interface
- Mở file pcap đã bắt trước
- Dissect (phân tích) hơn 2000 giao thức
- Filter và tìm kiếm packet
- Follow TCP/UDP stream để xem nội dung conversation
- Thống kê lưu lượng, endpoint, giao thức

**Capture filter vs Display filter:**
- **Capture filter**: Lọc khi bắt (dùng BPF syntax, ví dụ: `host 192.168.1.1`)
- **Display filter**: Lọc khi hiển thị kết quả đã bắt (dùng Wireshark syntax)

**Display filter ví dụ:**

```
# Chỉ hiển thị HTTP
http

# Traffic đến/từ một IP
ip.addr == 192.168.1.100

# TCP trên port 22 (SSH)
tcp.port == 22

# Chỉ SYN packet
tcp.flags.syn == 1 && tcp.flags.ack == 0

# Tìm cleartext credentials trong HTTP
http.request.method == "POST"
```

### So sánh Telnet và SSH qua Wireshark

Đây là một trong những demo kinh điển nhất để hiểu tầm quan trọng của mã hóa.

**Telnet (không mã hóa - port 23):**

Khi bắt gói tin Telnet bằng Wireshark và dùng "Follow TCP Stream":
- Thấy toàn bộ nội dung session dưới dạng cleartext
- Username và password hiện rõ ràng
- Mọi lệnh gõ đều thấy được
- Ví dụ trong stream: `login: admin` rồi `Password: pass123`

**SSH (mã hóa - port 22):**

Khi bắt gói tin SSH bằng Wireshark:
- Giao thức handshake thấy được (SSH-2.0, key exchange)
- Sau handshake: chỉ thấy encrypted payload
- Không đọc được username, password, hay lệnh
- Chỉ thấy: kích thước packet, thời gian, và IP

**Bài học quan trọng:**
- Không bao giờ dùng Telnet, rlogin, FTP trong môi trường production
- SSH, SFTP, HTTPS là thay thế mã hóa cần thiết
- Wireshark là công cụ tốt nhất để chứng minh điều này trực quan

## Lệnh / Ví dụ minh họa

> **Tất cả ví dụ dưới đây chỉ dành cho môi trường lab có sự cho phép. Không thực hiện trên mạng không thuộc quyền sở hữu.**

### Nmap scan trong lab

```bash
# Tìm host alive trong subnet lab (không scan port)
nmap -sn 192.168.56.0/24

# Scan top 1000 port trên host lab của mình
nmap 192.168.56.101

# Xem phiên bản service đang chạy
nmap -sV 192.168.56.101

# Scan nhanh và lưu kết quả
nmap -F -oN scan_result.txt 192.168.56.101
```

### tcpdump trong lab

```bash
# Bắt gói tin trên interface lab của mình (không phải mạng công ty)
# Cần quyền root/sudo
sudo tcpdump -i eth0 -n

# Bắt và hiển thị nội dung ASCII (để thấy cleartext)
sudo tcpdump -i eth0 -A -n port 23

# Lưu pcap để mở trong Wireshark
sudo tcpdump -i eth0 -w /tmp/lab-capture.pcap

# Bắt chỉ traffic đến/từ một host lab
sudo tcpdump -i eth0 -n host 192.168.56.101
```

### Wireshark filter examples

```
# Filter HTTP trong Wireshark display filter
http

# Filter theo IP
ip.addr == 192.168.56.101

# Filter SSH
tcp.port == 22

# Filter Telnet (để demo cleartext)
tcp.port == 23

# Follow stream: Click chuột phải vào packet -> Follow -> TCP Stream
```

## Ví dụ thực tế

**Demo: So sánh Telnet và SSH trong lab**

Setup: Máy A (Kali Linux lab) kết nối đến Máy B (Ubuntu lab với cả Telnet và SSH) trong network lab isolate.

1. Mở Wireshark trên Máy A, capture trên interface lab
2. Từ Máy A: `telnet 192.168.56.102` -> đăng nhập
3. Dừng capture, Follow TCP Stream -> **Thấy toàn bộ session cleartext**
4. Capture mới, từ Máy A: `ssh user@192.168.56.102` -> đăng nhập
5. Dừng capture, Follow TCP Stream -> **Chỉ thấy encrypted data**

*Kết luận*: Telnet không bao giờ an toàn; SSH là bắt buộc.

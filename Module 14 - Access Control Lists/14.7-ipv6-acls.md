# 14.7 - IPv6 ACLs

## Tổng quan

IPv6 ACL tương tự IPv4 ACL nhưng có một số điểm khác biệt quan trọng: không có Standard ACL; tất cả đều phải named; và phải đặc biệt chú ý đến NDP (Neighbor Discovery Protocol) vì IPv6 dùng NDP thay cho ARP. Nếu cấu hình deny ipv6 any any mà không permit NDP, thiết bị IPv6 không hoạt động được.

## Ý chính

**Tại sao IPv6 ACL quan trọng:**

Khi mạng chuyển đổi sang IPv6, các môi trường dual-stack (vừa IPv4 vừa IPv6) tạo lỗ hổng. Kẻ tấn công có thể:
- Dùng IPv4 để xâm nhập ban đầu.
- Gửi IPv6 Router Advertisement (RA) giả mạo để thiết bị dual-stack tự động cấu hình IPv6.
- Dùng kết nối IPv6 mới để di chuyển trong mạng (lateral movement).
- Dùng Teredo tunneling nhúng IPv6 trong UDP/IPv4 để vượt qua firewall chỉ hiểu IPv4.

**Sự khác biệt IPv6 ACL so với IPv4:**

| Đặc điểm | IPv4 ACL | IPv6 ACL |
|----------|---------|---------|
| Standard ACL | Có (chỉ source) | Không — tất cả IPv6 ACL đều là extended |
| Named only | Tùy chọn | Bắt buộc phải có tên |
| Implicit deny | `deny ip any any` | `deny ipv6 any any` |
| NDP consideration | Không cần | Cần permit NDP statements |
| Apply to interface | `ip access-group` | `ipv6 traffic-filter` |

**Cú pháp IPv6 ACL:**

```
Router(config)# ipv6 access-list <NAME>
Router(config-ipv6-acl)# {deny|permit} protocol {source-prefix/length|any|host src-addr} [op port] {dst-prefix/length|any|host dst-addr} [op port] [options]
```

Các option thêm: `dscp`, `fragments`, `log`, `log-input`, `sequence`, `time-range`

**Áp dụng IPv6 ACL:**

```
Router(config-if)# ipv6 traffic-filter <NAME> {in|out}
```

**NDP (Neighbor Discovery Protocol):**

IPv6 dùng NDP thay ARP để học địa chỉ Layer 2. NDP dùng ICMP với hai message quan trọng:
- **NA (Neighbor Advertisement)**: Tương tự ARP reply.
- **NS (Neighbor Solicitation)**: Tương tự ARP request.

Nếu `deny ipv6 any any` mà không permit NA và NS, các thiết bị IPv6 không thể học địa chỉ MAC của nhau và không giao tiếp được trong cùng mạng.

**Implicit statements trong IPv6 ACL:**

Mỗi IPv6 ACL tự động có:
1. `deny ipv6 any any` (implicit deny ở cuối — giống IPv4)
2. `permit icmp any any nd-na` (implicit permit NDP neighbor advertisement)
3. `permit icmp any any nd-ns` (implicit permit NDP neighbor solicitation)

Best practice: viết tường minh các permit NDP statements thay vì dựa vào implicit — để rõ ràng trong config.

## Thuật ngữ quan trọng

| Thuật ngữ | Định nghĩa |
|-----------|-----------|
| IPv6 ACL | ACL lọc traffic IPv6; luôn phải named; không có loại Standard |
| NDP | Neighbor Discovery Protocol — IPv6 dùng thay ARP để học địa chỉ MAC |
| ND-NA | Neighbor Advertisement — IPv6 tương tự ARP reply |
| ND-NS | Neighbor Solicitation — IPv6 tương tự ARP request |
| Teredo tunneling | Kỹ thuật nhúng IPv6 trong UDP/IPv4 để xuyên qua NAT |
| Dual-stack | Thiết bị có cả IPv4 và IPv6; tạo lỗ hổng nếu không bảo mật cả hai |
| ipv6 traffic-filter | Lệnh áp dụng IPv6 ACL lên interface |
| Rogue RA | Router Advertisement giả mạo — kẻ tấn công gửi để thiết bị tự cấu hình IPv6 |

## Ghi chú chi tiết

**Dual-stack và lỗ hổng bảo mật:**

Nhiều tổ chức tập trung bảo mật IPv4 nhưng quên rằng thiết bị hiện đại đều có IPv6 mặc định. Kẻ tấn công tận dụng: xâm nhập qua IPv4, kích hoạt IPv6 trên host bị nhiễm, dùng IPv6 để giao tiếp với hosts khác trong mạng — vì firewall IPv4 không kiểm tra traffic IPv6. Giải pháp: triển khai IPv6 ACL song song với IPv4 ACL, hoặc tắt IPv6 nếu không cần.

**Tại sao phải permit NDP explicit?**

Dù IPv6 ACL có implicit permit NDP, việc viết tường minh:
1. Làm cấu hình rõ ràng hơn — người đọc hiểu ngay tại sao có các permit đó.
2. Ngăn nhầm lẫn khi audit — không ai thắc mắc tại sao NDP hoạt động dù ACL có deny.
3. Best practice của Cisco: document implicit rules để bảo trì dễ hơn.

**IPv6 prefix notation:**

Trong IPv6 ACL, địa chỉ dùng format prefix/length thay vì wildcard mask:
- `2001:db8:1:1::/64` — subnet /64
- `any` — tương đương ::/0 (tất cả IPv6 addresses)
- `host 2001:db8::1` — một địa chỉ IPv6 cụ thể

## Ví dụ thực tế

**Ví dụ: Cho phép traffic từ subnet IPv6 nội bộ, block phần còn lại:**

```
Router(config)# ipv6 access-list LAN-ONLY
Router(config-ipv6-acl)# permit ipv6 2001:db8:1:1::/64 any
Router(config-ipv6-acl)# permit icmp any any nd-na
Router(config-ipv6-acl)# permit icmp any any nd-ns
Router(config-ipv6-acl)# deny ipv6 any any
Router(config-ipv6-acl)# exit
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ipv6 traffic-filter LAN-ONLY in

! Xác nhận:
Router# show ipv6 access-list
IPv6 access list LAN-ONLY
    permit ipv6 2001:DB8:1:1::/64 any sequence 10
    permit icmp any any nd-na sequence 20
    permit icmp any any nd-ns sequence 30
    deny ipv6 any any sequence 40
```

**Ví dụ: Chặn HTTP và HTTPS từ một subnet IPv6 (chống DoS):**

```
! Tình huống: subnet 2001:db8:1:11::/64 đang gây DoS đến server
Router(config)# ipv6 access-list BLOCK-HTTP
Router(config-ipv6-acl)# deny tcp 2001:db8:1:11::/64 host 2001:db8:1:30::30 eq 80
Router(config-ipv6-acl)# deny tcp 2001:db8:1:11::/64 host 2001:db8:1:30::30 eq 443
Router(config-ipv6-acl)# permit ipv6 any any
Router(config-ipv6-acl)# exit

! Áp dụng trên interface gần source (interface nhận traffic từ subnet nguồn):
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ipv6 traffic-filter BLOCK-HTTP in
```

**Ví dụ: Chặn ICMP ping để ngăn DDoS:**

```
Router(config)# ipv6 access-list BLOCK-ICMP
Router(config-ipv6-acl)# deny icmp any host 2001:db8:1:30::30 echo
Router(config-ipv6-acl)# permit ipv6 any any
Router(config-ipv6-acl)# exit

! Áp dụng gần server bị tấn công (outbound trên interface hướng đến server):
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ipv6 traffic-filter BLOCK-ICMP out
```

## Lệnh / Ví dụ cấu hình

```
! Tạo và xem IPv6 ACL:
Router(config)# ipv6 access-list MY-V6-ACL
Router# show ipv6 access-list
Router# show ipv6 interface GigabitEthernet0/0

! Áp dụng IPv6 ACL:
Router(config-if)# ipv6 traffic-filter MY-V6-ACL in

! Gỡ IPv6 ACL khỏi interface:
Router(config-if)# no ipv6 traffic-filter MY-V6-ACL in

! Xóa IPv6 ACL:
Router(config)# no ipv6 access-list MY-V6-ACL

! NDP permit statements:
Router(config-ipv6-acl)# permit icmp any any nd-na
Router(config-ipv6-acl)# permit icmp any any nd-ns
```

## Cần ghi nhớ

- **IPv6 ACL luôn phải có tên** — không có numbered IPv6 ACL.
- **Không có Standard IPv6 ACL** — tất cả đều extended (có thể lọc cả source và destination).
- **`ipv6 traffic-filter` thay vì `ip access-group`** cho IPv6.
- **Phải permit ND-NA và ND-NS** nếu có deny ipv6 any any.
- **Dual-stack là điểm yếu bảo mật** — phải bảo mật cả IPv4 và IPv6 song song.
- **Teredo tunneling có thể vượt qua firewall IPv4** — cần giám sát và lọc.

# 14.3 - Configure ACLs

## Tổng quan

Bài này đề cập cú pháp đầy đủ để tạo Standard và Extended ACL trên Cisco IOS. Standard ACL đơn giản hơn, chỉ cần source address. Extended ACL chi tiết hơn với nhiều tham số: protocol, source, destination, port. Ngoài ra, TCP established keyword cho phép lọc traffic stateful đơn giản.

## Ý chính

**Kế hoạch trước khi cấu hình:**

Trước khi gõ lệnh, cần:
1. Dùng text editor viết ra chính sách cần triển khai.
2. Thêm lệnh IOS cần thiết.
3. Thêm remark để ghi chú.
4. Copy-paste vào thiết bị.
5. Kiểm tra kỹ ACL sau khi áp dụng.

**Standard IPv4 ACL — Numbered:**

```
Router(config)# access-list <number> {deny|permit|remark} <source> [wildcard] [log]
```

| Tham số | Mô tả |
|--------|------|
| number | 1-99 hoặc 1300-1999 |
| deny | Từ chối nếu khớp |
| permit | Cho phép nếu khớp |
| remark | Ghi chú (tối đa 100 ký tự); không ảnh hưởng lọc |
| source | IP nguồn; dùng `any` cho tất cả; dùng `host IP` cho một host |
| wildcard | Tùy chọn; mặc định 0.0.0.0 nếu không ghi |
| log | Tùy chọn; tạo thông báo log khi ACE khớp |

**Standard IPv4 ACL — Named:**

```
Router(config)# ip access-list standard <NAME>
Router(config-std-nacl)# {deny|permit|remark} <source> [wildcard] [log]
```

ACL name: alphanumeric; không có dấu cách; phân biệt hoa thường; thường dùng HOA.

**Extended IPv4 ACL — Numbered:**

```
Router(config)# access-list <number> {deny|permit|remark} <protocol> <source> <src-wildcard> [operator port] <destination> <dst-wildcard> [operator port] [established] [log]
```

| Tham số | Mô tả |
|--------|------|
| number | 100-199 hoặc 2000-2699 |
| protocol | ip, tcp, udp, icmp, ospf, eigrp, ahp, esp, gre... |
| operator | lt, gt, eq, neq (so sánh port) |
| port | Số port hoặc tên (www=80, ftp=21, ssh=22, https=443) |
| established | Chỉ TCP; cho phép returning traffic (ACK hoặc RST bit đặt) |

**Extended IPv4 ACL — Named:**

```
Router(config)# ip access-list extended <NAME>
Router(config-ext-nacl)# {deny|permit|remark} <protocol> <src> <wild> [op port] <dst> <wild> [op port] [options]
```

**Áp dụng ACL lên interface:**

```
! Lên interface:
Router(config-if)# ip access-group {number|name} {in|out}

! Lên VTY lines:
Router(config-line)# ip access-class {number|name} {in|out}
```

**TCP Established:**

`established` keyword cho phép traffic phản hồi từ internet trở lại mạng nội bộ. Chỉ dùng với TCP. Packet khớp nếu có bit ACK hoặc RST trong TCP header — dấu hiệu của kết nối đang thiết lập, không phải kết nối mới từ bên ngoài.

Thiết kế thông thường:
- ACL outbound (từ LAN ra): cho phép HTTP/HTTPS ra ngoài.
- ACL inbound (từ internet vào): cho phép established TCP vào (response từ web server).

## Thuật ngữ quan trọng

| Thuật ngữ | Định nghĩa |
|-----------|-----------|
| remark | Ghi chú trong ACL; không ảnh hưởng đến packet filtering |
| protocol | Loại giao thức trong Extended ACL: ip, tcp, udp, icmp |
| operator | Toán tử so sánh port: lt, gt, eq, neq |
| eq | Equal — khớp đúng port số đó |
| lt | Less than — nhỏ hơn port số đó |
| gt | Greater than — lớn hơn port số đó |
| neq | Not equal — khác port số đó |
| established | Keyword TCP; khớp packet có ACK hoặc RST bit — kết nối đã thiết lập |
| log | Tạo thông báo syslog khi ACE khớp; cẩn thận — gây tải CPU |

## Ghi chú chi tiết

**Thứ tự xử lý Extended ACL:**

Với Standard ACL, Cisco IOS tự động sắp xếp lại theo thứ tự từ cụ thể nhất đến tổng quát nhất (phần bên dưới). Với Extended ACL, thứ tự nhập vào là thứ tự xử lý — Cisco không tự sắp xếp. Điều này có nghĩa là quản trị viên phải tự đảm bảo thứ tự đúng khi cấu hình Extended ACL.

**TCP Established và stateful filtering:**

`established` keyword cung cấp tính năng stateful firewall đơn giản nhưng không hoàn hảo như stateful firewall thực sự. Nó chỉ kiểm tra ACK/RST bit — kẻ tấn công có thể giả mạo gói tin với ACK bit set để vượt qua filter. Stateful firewall thực sự theo dõi toàn bộ trạng thái kết nối. Dù vậy, `established` vẫn hữu ích trong nhiều tình huống thực tế.

**Remark trong ACL:**

Remark làm ACL dễ đọc và dễ quản lý. Ví dụ:

```
Router(config)# ip access-list extended WEB-POLICY
Router(config-ext-nacl)# remark === Cho phép web browsing outbound ===
Router(config-ext-nacl)# permit tcp 192.168.0.0 0.0.255.255 any eq 80
Router(config-ext-nacl)# permit tcp 192.168.0.0 0.0.255.255 any eq 443
Router(config-ext-nacl)# remark === Block tất cả traffic còn lại ===
Router(config-ext-nacl)# deny ip any any log
```

**Log keyword và hiệu năng:**

`log` gây tải đáng kể lên CPU router vì mỗi packet khớp đều tạo syslog message. Chỉ dùng `log` khi đang troubleshoot hoặc router đang bị tấn công và cần xác định attacker. Không để `log` trong cấu hình production bình thường.

## Ví dụ thực tế

**Ví dụ Standard ACL — Cho phép một mạng, chặn host cụ thể:**

```
! Chặn host 10.1.1.100 từ mạng 10.1.1.0/24, cho phép phần còn lại
Router(config)# access-list 12 deny host 10.1.1.100
Router(config)# access-list 12 permit 10.1.1.0 0.0.0.255
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ip access-group 12 out
```

**Ví dụ Extended ACL — Cho phép HTTP/HTTPS, chặn FTP:**

```
Router(config)# access-list 101 permit tcp 172.16.0.0 0.0.255.255 any eq 80
Router(config)# access-list 101 permit tcp 172.16.0.0 0.0.255.255 any eq 443
Router(config)# access-list 101 deny tcp 172.16.0.0 0.0.255.255 any eq 21
Router(config)# access-list 101 permit ip any any
```

**Ví dụ TCP Established — Cho phép web browsing trong khi chặn kết nối khởi tạo từ ngoài:**

```
! ACL 110: cho phép internal users truy cập web
Router(config)# access-list 110 permit tcp 192.168.10.0 0.0.0.255 any eq 80
Router(config)# access-list 110 permit tcp 192.168.10.0 0.0.0.255 any eq 443

! ACL 120: cho phép response từ internet trở về
Router(config)# access-list 120 permit tcp any 192.168.10.0 0.0.0.255 established

Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip access-group 110 in     ! Outbound từ LAN
Router(config-if)# ip access-group 120 out    ! Inbound từ internet về LAN
```

**Ví dụ Named Extended ACL:**

```
Router(config)# ip access-list extended BLOCK-SSH-TELNET
Router(config-ext-nacl)# remark Chặn SSH và Telnet từ bên ngoài
Router(config-ext-nacl)# deny tcp any 10.0.0.0 0.0.0.255 eq 22
Router(config-ext-nacl)# deny tcp any 10.0.0.0 0.0.0.255 eq 23
Router(config-ext-nacl)# permit ip any any
Router(config-ext-nacl)# exit
Router(config)# interface Serial0/0/0
Router(config-if)# ip access-group BLOCK-SSH-TELNET in
```

## Lệnh / Ví dụ cấu hình

```
! Kiểm tra ACL sau khi cấu hình
Router# show access-lists
Router# show ip access-lists
Router# show running-config | include access-list
Router# show running-config | section access-list

! Gỡ ACL khỏi interface trước khi xóa
Router(config-if)# no ip access-group 101 in
Router(config)# no access-list 101

! Ports thông dụng:
! www (http) = 80
! 443 (https)
! 22 (ssh)
! 23 (telnet)
! 21 (ftp)
! 20 (ftp-data)
! 53 (domain/DNS)
! 25 (smtp)
! 110 (pop3)
! 143 (imap)
! 161 (snmp)
! 162 (snmptrap)
```

## Cần ghi nhớ

- **Cú pháp Standard**: `access-list <1-99> {permit|deny} <source> [wildcard]`
- **Cú pháp Extended**: thêm protocol, destination, port operators
- **`established`** cho phép returning TCP traffic mà không cần stateful firewall
- **Remark tối đa 100 ký tự** và không ảnh hưởng đến filtering
- **`log` chỉ dùng khi troubleshoot** — tốn nhiều CPU
- **Named ACL khuyến nghị** — tên mô tả mục đích; dễ chỉnh sửa sau

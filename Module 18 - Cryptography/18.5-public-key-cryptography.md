# 18.5 - Public Key Cryptography

## Tổng quan

Public key cryptography dùng cặp key liên quan toán học: public key có thể chia sẻ, private key phải giữ bí mật. Cơ chế này hỗ trợ encryption, key exchange, digital signature và hệ thống trust dựa trên certificates.

## Ý chính

- Public key có thể phân phối công khai, private key phải bảo vệ nghiêm ngặt.
- Dữ liệu mã hóa bằng public key thường chỉ private key tương ứng giải mã được.
- Dữ liệu ký bằng private key có thể được kiểm tra bằng public key.
- Public key cryptography thường kết hợp với symmetric encryption trong thực tế.
- Key exchange như Diffie-Hellman/ECDHE giúp hai bên tạo shared secret qua mạng không tin cậy.
- Trust vào public key thường cần certificate hoặc kênh xác minh đáng tin.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Public key | Key có thể chia sẻ để mã hóa hoặc kiểm tra chữ ký. |
| Private key | Key bí mật dùng để giải mã hoặc tạo chữ ký. |
| Key pair | Cặp public/private key có liên hệ toán học. |
| Key exchange | Cơ chế giúp hai bên tạo shared secret. |
| Digital signature | Chữ ký số tạo bằng private key. |
| RSA | Thuật toán public key phổ biến. |
| ECC | Họ thuật toán elliptic curve có key ngắn và hiệu quả. |
| ECDHE | Key exchange elliptic curve có forward secrecy. |

## Ghi chú chi tiết

**Mã hóa bằng public key:**

Nếu Alice muốn gửi dữ liệu bí mật cho Bob, Alice có thể mã hóa bằng public key của Bob. Chỉ Bob có private key tương ứng mới giải mã được. Tuy nhiên, trong hệ thống thực tế, cách này thường dùng để bảo vệ key phiên hơn là mã hóa dữ liệu lớn trực tiếp.

**Ký bằng private key:**

Nếu Bob ký dữ liệu bằng private key, người khác dùng public key của Bob để kiểm tra chữ ký. Điều này giúp xác nhận dữ liệu không bị sửa và chữ ký đến từ private key tương ứng.

**Vấn đề xác thực public key:**

Nếu attacker thay public key của Bob bằng public key của attacker, Alice có thể mã hóa nhầm cho attacker. Vì vậy, cần certificate, fingerprint verification hoặc trust store để xác minh public key thuộc đúng danh tính.

**Hybrid cryptography:**

TLS là ví dụ điển hình: public key cryptography hỗ trợ xác thực và tạo key phiên, sau đó symmetric encryption mã hóa dữ liệu chính vì nhanh hơn.

## Ví dụ thực tế

**Ví dụ 1: SSH key authentication**

User lưu public key trên server và giữ private key trên máy cá nhân. Khi đăng nhập, server kiểm tra user có private key tương ứng hay không mà không cần gửi password qua mạng.

**Ví dụ 2: TLS handshake**

Trình duyệt kiểm tra certificate của website, thực hiện key exchange để tạo session key, rồi dùng symmetric encryption để bảo vệ dữ liệu HTTP trong phiên.

## Cần ghi nhớ

- **Public key không cần bí mật**: nhưng phải xác thực đúng chủ sở hữu.
- **Private key phải được bảo vệ**: lộ private key làm mất confidentiality hoặc authenticity.
- **Public key crypto chậm hơn symmetric crypto**: thường dùng kết hợp.
- **Digital signature không giống encryption**: ký để xác thực và kiểm tra toàn vẹn.
- **Trust vào key cần cơ chế xác minh**: PKI giải quyết bài toán này ở quy mô lớn.

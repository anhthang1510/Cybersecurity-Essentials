# 18.8 - Cryptography Summary

## Tổng quan

Module 18 tổng kết các khái niệm cryptography quan trọng: confidentiality, obfuscation, integrity, authenticity, hashing, public key cryptography, certificates và PKI. Trọng tâm là hiểu đúng công dụng của từng cơ chế và tránh nhầm lẫn giữa chúng.

## Ý chính

| Chủ đề | Tóm tắt |
|---|---|
| Confidentiality | Encryption bảo vệ dữ liệu khỏi người không có quyền đọc. |
| Obscuring data | Encoding/obfuscation/masking không tương đương encryption. |
| Integrity | Hash và signature giúp phát hiện dữ liệu bị thay đổi. |
| Authenticity | HMAC, digital signature và certificates giúp xác thực nguồn. |
| Hashing | Hàm một chiều tạo digest, không dùng để giải mã. |
| Public key cryptography | Public/private key hỗ trợ encryption, key exchange và signatures. |
| PKI | CA và certificates tạo chain of trust cho public key. |
| Applications | TLS, VPN, password storage, software signing và secure updates. |

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Encryption | Mã hóa dữ liệu để bảo vệ confidentiality. |
| Hash | Digest một chiều để kiểm tra toàn vẹn. |
| HMAC | Xác thực thông điệp bằng hash và secret key. |
| Digital signature | Chữ ký số dùng private key để ký và public key để xác minh. |
| Certificate | Tài liệu số liên kết public key với danh tính. |
| CA | Bên phát hành và ký certificate. |
| PKI | Hạ tầng quản lý key, certificate và trust. |

## Ghi chú chi tiết

Một hệ thống bảo mật tốt thường kết hợp nhiều cơ chế cryptography. TLS dùng certificate để xác thực server, key exchange để tạo shared secret, symmetric encryption để bảo vệ dữ liệu và message authentication để phát hiện thay đổi. Nếu chỉ hiểu từng thuật toán riêng lẻ mà bỏ qua key management và trust model, hệ thống vẫn có thể bị tấn công.

## Mối liên hệ giữa các bài

```
Confidentiality
    ↓
Encryption và key management
    ↓
Hashing cho integrity
    ↓
HMAC và digital signature cho authenticity
    ↓
Public/private key
    ↓
Certificates, CA và PKI
    ↓
TLS, VPN, software signing và secure operations
```

## Ví dụ thực tế

Một website thương mại điện tử dùng TLS certificate để browser xác thực server, ECDHE để tạo session key, AES để mã hóa traffic, HMAC/AEAD để bảo vệ integrity và password hashing có salt để lưu mật khẩu người dùng. Đây là ví dụ nhiều cơ chế cryptography phối hợp trong một hệ thống thực tế.

## Cần ghi nhớ

- **Encryption khác hashing**: encryption có giải mã, hash là một chiều.
- **Confidentiality khác integrity**: bí mật và toàn vẹn là hai mục tiêu khác nhau.
- **Private key là tài sản cực kỳ nhạy cảm**: phải bảo vệ và rotate khi cần.
- **Certificate cần validation đầy đủ**: chain, domain, thời hạn, usage và revocation đều quan trọng.
- **Cryptography cần vận hành đúng**: key management, certificate renewal và algorithm updates quyết định hiệu quả lâu dài.

# 10.4 - Cybersecurity Principles, Practices, and Processes Summary

## Tổng quan

Bài 10.4 hệ thống lại toàn bộ kiến thức của Module 10 theo cấu trúc Cybersecurity Cube. Ba chiều của mô hình — mục tiêu CIA, trạng thái dữ liệu, biện pháp bảo vệ — tạo thành khung tư duy nền tảng mà mọi chuyên gia bảo mật cần nắm vững. Đây cũng là bài chuẩn bị để kiểm tra cuối module.

## Ý chính

**Cybersecurity Cube — Tóm tắt ba chiều:**

```
Dimension 1: CIA Goals
  Confidentiality → ngăn tiết lộ trái phép
  Integrity       → đảm bảo dữ liệu chính xác và không bị sửa đổi
  Availability    → đảm bảo truy cập khi cần

Dimension 2: Data States
  Data at rest    → đang lưu trữ (DAS, RAID, NAS, SAN, Cloud)
  Data in transit → đang truyền (sneaker net, wired, wireless)
  Data in process → đang xử lý (input, modification, output)

Dimension 3: Safeguards
  Technology      → phần mềm và phần cứng bảo vệ hệ thống
  Policy & Practices → bốn tầng Policy/Standard/Guideline/Procedure
  People          → đào tạo, awareness, văn hóa bảo mật
```

**CIA Triad — Kỹ thuật tương ứng:**

| Nguyên tắc | Kỹ thuật chính | Mối đe dọa |
|-----------|---------------|-----------|
| Confidentiality | Encryption, Tokenization, DRM, IRM, Anonymization | Unauthorized access, data leakage |
| Integrity | Hashing, HMAC, data validation, access controls | Tampering, malicious code, data corruption |
| Availability | Backup, redundancy, patching, monitoring, disaster recovery | DoS/DDoS, hardware failure, misconfiguration |

**Data States — Thách thức và giải pháp:**

| Trạng thái | Thách thức | Giải pháp |
|-----------|-----------|----------|
| At rest | Trộm thiết bị, truy cập trái phép, malware cục bộ | Mã hóa, phân quyền, kiểm soát vật lý |
| In transit | Capture, alteration, rogue device | VPN/TLS, HMAC, mutual authentication |
| In process | Input errors, malicious code, output misconfiguration | Validation, access control, kiểm tra output |

**Storage types — So sánh nhanh:**

| Loại | Quản lý | Rủi ro | Phù hợp |
|------|---------|--------|---------|
| DAS | Khó tập trung | Cao nhất per-device | Cá nhân |
| NAS | Tập trung qua mạng | Trung bình | Doanh nghiệp vừa |
| SAN | Tập trung, phức tạp | Cao nếu bị xâm phạm | Doanh nghiệp lớn |
| Cloud | Nhà cung cấp quản lý | Phụ thuộc provider | Mọi quy mô |
| RAID | Theo hệ thống | Fault tolerance tốt | Cần redundancy |

**Policy Framework — Bốn tầng:**

| Tầng | Bắt buộc | Ví dụ |
|------|----------|-------|
| Policy | Có | "Phải dùng mật khẩu mạnh" |
| Standard | Có | "Tối thiểu 8 ký tự, thay mỗi 30 ngày" |
| Guideline | Không bắt buộc | "Dùng cụm từ dễ nhớ làm mật khẩu" |
| Procedure | Có | "Bước 1: Ctrl+Alt+Del → Bước 2:..." |

**Security Policy — Sáu thành phần chính:**

1. Identification & Authentication Policy
2. Password Policy
3. Acceptable Use Policy (AUP)
4. Remote Access Policy
5. Network Maintenance Policy
6. Incident Handling Procedures

## Thuật ngữ quan trọng

| Thuật ngữ | Định nghĩa |
|-----------|-----------|
| CIA Triad | Confidentiality – Integrity – Availability |
| McCumber Cube | Mô hình 3D: 3 goals × 3 states × 3 safeguards |

| DRM | Digital Rights Management — bảo vệ nội dung số có bản quyền |
| IRM | Information Rights Management — kiểm soát quyền truy cập tài liệu nội bộ |
| Hashing | Tạo giá trị hash cố định để phát hiện sửa đổi dữ liệu |
| HMAC | Hash-based Message Authentication Code — hash + secret key |
| DAS | Direct-Attached Storage — lưu trữ gắn trực tiếp vào máy |
| NAS | Network-Attached Storage — lưu trữ qua mạng |
| SAN | Storage Area Network — mạng lưu trữ tốc độ cao |
| RAID | Redundant Array of Independent Disks |

| AUP | Acceptable Use Policy |
| Software firewall | Phần mềm kiểm soát kết nối trên host |

## Ghi chú chi tiết

**Tổng hợp các biện pháp theo từng CIA:**

*Confidentiality:*
- Technical: Encryption (symmetric AES, asymmetric RSA), Tokenization (giữ định dạng), DRM (nội dung số), IRM (tài liệu nội bộ)
- Administrative: Phân loại thông tin (personal/business/classified), data minimization, privacy policy

*Integrity:*
- Technical: MD5/SHA hashing, HMAC (hash + key), data validation, consistency checks
- Administrative: Access controls, change management, audit trails

*Availability:*
- Technical: RAID (fault tolerance), redundancy, load balancing, CDN
- Administrative: Backup policy, disaster recovery plan, patch management schedule, continuous monitoring

**Vòng đời bảo vệ dữ liệu:**

```
Thu thập → Lưu trữ → Xử lý → Truyền → Xóa
   ↑            ↑          ↑         ↑       ↑
  Input        At rest  In process  In transit  Disposal
 controls    security   controls    security   procedures
```

Mỗi điểm trong vòng đời đều cần biện pháp bảo vệ riêng.

## Ví dụ thực tế

**Tình huống tổng hợp — Công ty XYZ xây dựng hệ thống bảo mật:**

XYZ Inc. sản xuất và phân phối thiết bị bảo hộ y tế. Họ cần bảo vệ:
- Dữ liệu khách hàng (tên, địa chỉ, thông tin thanh toán) → **Confidentiality** → Mã hóa, AUP, phân quyền
- Giá sản phẩm và thông tin đặt hàng → **Integrity** → Hashing, audit log, procedural control
- Hệ thống đặt hàng online → **Availability** → Redundancy, backup, monitoring

Biện pháp:
- **Technology**: Hardware firewall, proxy server, vulnerability scanner
- **Policy**: Security policy với đầy đủ 6 thành phần, AUP cụ thể cho từng phòng ban
- **People**: Onboarding training, quarterly security workshop, KPI-linked compliance

## Cần ghi nhớ

- **CIA là bộ ba đồng đẳng** — không nguyên tắc nào quan trọng hơn nguyên tắc nào; tùy lĩnh vực mà ưu tiên có khác
- **Tokenization bảo toàn định dạng** — ưu điểm so với encryption khi tích hợp hệ thống cũ
- **HMAC bảo vệ cả toàn vẹn lẫn xác thực** — hashing thông thường chỉ bảo vệ toàn vẹn
- **Bảo vệ cả ba trạng thái** — at rest, in transit, in process đều là điểm tấn công tiềm năng
- **Con người là tuyến phòng thủ đầu tiên** — và thường là điểm yếu nhất nếu không được đào tạo
- **Policy Framework bốn tầng** — Policy định hướng, Standard quy định, Guideline gợi ý, Procedure hướng dẫn từng bước

# 17.3 - Cloud Infrastructure Security

## Tổng quan

Phần này tập trung vào bảo vệ cloud infrastructure: network, compute, storage, identity, management plane và cấu hình tài nguyên. Mục tiêu là giảm bề mặt tấn công và ngăn cấu hình sai dẫn đến lộ hệ thống.

## Ý chính

- Management plane/API cloud cần được bảo vệ bằng MFA, least privilege và logging.
- Network segmentation dùng VPC/VNet, subnet, route table, security group và firewall cloud-native.
- Không nên public tài nguyên nếu không cần thiết.
- Infrastructure as Code giúp chuẩn hóa cấu hình nhưng cũng cần review bảo mật.
- CSPM/continuous assessment giúp phát hiện misconfiguration.
- Tagging và inventory giúp quản lý tài nguyên cloud đang thay đổi nhanh.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Management plane | Lớp API/console dùng để tạo, sửa, xóa và quản lý tài nguyên cloud. |
| VPC/VNet | Mạng ảo riêng trong cloud. |
| Subnet | Phân đoạn mạng trong VPC/VNet. |
| Security group | Firewall logic gắn với workload/tài nguyên. |
| Route table | Bảng điều khiển hướng đi của traffic trong mạng cloud. |
| Bastion host | Host trung gian được hardening để quản trị tài nguyên private. |
| CSPM | Công cụ đánh giá posture và phát hiện cấu hình cloud rủi ro. |

## Ghi chú chi tiết

Bảo mật cloud infrastructure bắt đầu từ management plane. Nếu tài khoản admin cloud bị chiếm quyền, attacker có thể tạo VM, mở firewall, đọc storage, tạo access key hoặc tắt logging. Vì vậy cần MFA, role-based access, least privilege, break-glass account được kiểm soát và log mọi thay đổi quan trọng.

Ở tầng network, nên tách subnet public/private. Tài nguyên public như load balancer hoặc reverse proxy có thể nằm ở public subnet; database và application server nhạy cảm nên ở private subnet. Security group chỉ mở port cần thiết, từ nguồn cụ thể.

Infrastructure as Code giúp triển khai nhất quán nhưng cũng có rủi ro nếu template chứa rule quá rộng hoặc secret. Do đó cần review, scanning và quy trình approval trước khi áp dụng vào production.

## Ví dụ thực tế

Một hệ thống web cloud có load balancer public nhận HTTPS, application servers trong private subnet, database không có public IP, admin truy cập qua bastion hoặc VPN, security group chỉ cho app server kết nối database port cần thiết. Cloud audit log ghi lại mọi thay đổi security group và IAM policy.

## Cần ghi nhớ

- Bảo vệ management plane là ưu tiên cao.
- Public exposure phải là ngoại lệ có lý do, không phải mặc định.
- Security group nên cụ thể theo source, destination và port.
- IaC cần security review giống như code ứng dụng.
- Inventory và logging giúp phát hiện tài nguyên “bị quên”.

# 17.0 - Introduction

## Tổng quan

Module 17 mở đầu chủ đề Cloud Security. Nội dung giúp người học hiểu cloud computing thay đổi cách triển khai hệ thống và cũng thay đổi cách bảo vệ infrastructure, application, data và workloads.

## 17.0.1 Why Should I Take this Module?

Cloud hiện diện trong hầu hết môi trường doanh nghiệp: email SaaS, storage, web app, database, virtual machines, container và dịch vụ AI/data. Nếu không hiểu cloud security, tổ chức dễ gặp lỗi như public storage bucket, IAM quá rộng, thiếu logging, không mã hóa dữ liệu hoặc bảo vệ VM như máy vật lý thông thường mà bỏ qua đặc thù cloud.

## 17.0.2 What Will I Learn in this Module?

Bạn sẽ học virtualization, cloud computing, các mô hình dịch vụ cloud, shared responsibility model và các miền bảo mật chính trong cloud. Module cũng tập trung vào cách bảo vệ cloud infrastructure, application, data và virtual machines.

## Mục tiêu học tập

- Giải thích virtualization và vai trò của hypervisor/VM.
- Phân biệt IaaS, PaaS và SaaS về trách nhiệm bảo mật.
- Nhận diện các miền cloud security quan trọng.
- Mô tả biện pháp bảo vệ infrastructure, application và data trên cloud.
- Hiểu cách hardening, patching, monitoring và backup virtual machines.

## Cần ghi nhớ

- Cloud không tự động an toàn; cấu hình sai là rủi ro lớn.
- Shared responsibility model thay đổi theo IaaS/PaaS/SaaS.
- Identity thường là perimeter mới trong cloud.
- Dữ liệu cần được bảo vệ ở rest, in transit và in use khi phù hợp.
- VM trên cloud vẫn cần hardening, patching, EDR/anti-malware, firewall và logging.

## Câu hỏi ôn tập nhanh

1. Vì sao cloud security quan trọng với doanh nghiệp hiện đại?
2. Cloud computing khác virtualization ở điểm nào?
3. Shared responsibility model có ý nghĩa gì?
4. IAM ảnh hưởng đến cloud security như thế nào?
5. VM trên cloud cần được bảo vệ bằng những lớp nào?

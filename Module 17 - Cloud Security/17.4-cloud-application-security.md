# 17.4 - Cloud Application Security

## Tổng quan

Phần này nói về bảo mật ứng dụng chạy trên cloud. Cloud application security kết hợp secure coding, API security, dependency management, CI/CD security, secrets management và runtime monitoring.

## Ý chính

- Ứng dụng cloud thường dùng API, microservices, container, managed database và third-party services.
- DevSecOps đưa kiểm tra bảo mật vào pipeline phát triển.
- Secrets như API key, token và password phải được quản lý bằng secret manager, không hard-code trong source code.
- API cần authentication, authorization, rate limiting, input validation và logging.
- Dependency và container image cần được scan lỗ hổng.
- WAF và runtime protection giúp giảm rủi ro tấn công web phổ biến.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| DevSecOps | Tích hợp bảo mật vào quy trình phát triển và vận hành. |
| CI/CD | Pipeline tự động build, test và deploy ứng dụng. |
| Secret manager | Dịch vụ lưu trữ và cấp phát secret an toàn. |
| API security | Bảo vệ endpoint API khỏi truy cập trái phép và abuse. |
| WAF | Web Application Firewall, lọc tấn công web như injection hoặc XSS. |
| SAST | Static Application Security Testing, kiểm tra code tĩnh. |
| DAST | Dynamic Application Security Testing, kiểm tra ứng dụng đang chạy. |

## Ghi chú chi tiết

Ứng dụng cloud thường thay đổi nhanh và phụ thuộc nhiều vào dịch vụ bên ngoài. Nếu pipeline deploy tự động nhưng không có kiểm tra bảo mật, lỗi có thể được đưa vào production rất nhanh. Vì vậy DevSecOps khuyến khích kiểm tra code, dependency, image, secret và cấu hình trước khi deploy.

Secrets là vấn đề lớn trong cloud application security. API key hoặc credential bị commit lên Git có thể cho attacker truy cập tài nguyên cloud. Secret manager giúp lưu secret tập trung, xoay vòng, kiểm soát quyền truy cập và audit.

API cần kiểm soát chặt vì chúng là cửa vào chính của ứng dụng. Authentication xác định ai gọi API, authorization xác định họ được làm gì, rate limiting giảm abuse, còn input validation giảm injection và lỗi logic.

## Ví dụ thực tế

Một nhóm phát triển triển khai ứng dụng container lên cloud. Pipeline CI/CD scan dependency, scan container image, kiểm tra secret trong repository, chạy test bảo mật cơ bản và chỉ deploy nếu policy pass. Ứng dụng lấy database password từ secret manager, không lưu trong file cấu hình.

## Cần ghi nhớ

- Cloud application security phải bắt đầu từ SDLC, không chờ đến production.
- Không hard-code secret trong code, image hoặc pipeline log.
- API cần authentication và authorization rõ ràng.
- Dependency lỗi thời là nguồn rủi ro phổ biến.
- WAF hữu ích nhưng không thay thế secure coding.

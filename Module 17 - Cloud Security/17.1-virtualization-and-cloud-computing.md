# 17.1 - Virtualization and Cloud Computing

## Tổng quan

Phần này giải thích virtualization là nền tảng kỹ thuật quan trọng giúp cloud computing hoạt động linh hoạt. Virtualization tạo tài nguyên ảo, còn cloud computing cung cấp tài nguyên đó theo mô hình dịch vụ, tự phục vụ và mở rộng linh hoạt.

## Ý chính

- Virtualization cho phép nhiều VM chạy trên một host vật lý.
- Hypervisor quản lý CPU, memory, storage và network ảo cho VM.
- Cloud computing cung cấp tài nguyên IT theo nhu cầu qua network.
- Cloud có các đặc tính như on-demand self-service, broad network access, resource pooling, rapid elasticity và measured service.
- Virtualization không đồng nghĩa với cloud; cloud thường dùng virtualization nhưng còn có automation, orchestration và service model.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Host | Máy vật lý chạy hypervisor và cung cấp tài nguyên cho VM. |
| Guest | Hệ điều hành hoặc workload chạy bên trong VM. |
| Hypervisor Type 1 | Hypervisor chạy trực tiếp trên phần cứng vật lý. |
| Hypervisor Type 2 | Hypervisor chạy trên hệ điều hành host. |
| VM image | Mẫu/ảnh dùng để tạo VM. |
| Snapshot | Bản ghi trạng thái VM tại một thời điểm, dùng cho rollback hoặc backup ngắn hạn. |
| Orchestration | Tự động phối hợp tạo, cấu hình và quản lý tài nguyên cloud. |

## Ghi chú chi tiết

Virtualization trừu tượng hóa tài nguyên vật lý thành tài nguyên logic. Một server vật lý có thể chạy nhiều VM, mỗi VM có hệ điều hành, ứng dụng và cấu hình riêng. Điều này giúp tận dụng phần cứng tốt hơn và triển khai hệ thống nhanh hơn.

Cloud computing mở rộng ý tưởng đó thành dịch vụ. Người dùng có thể tự tạo VM, database, storage hoặc application platform qua portal/API. Tài nguyên có thể tăng giảm theo nhu cầu và được tính phí theo mức sử dụng.

Điểm cần nhớ là cloud không chỉ là “máy ảo ở nơi khác”. Cloud có automation, API, IAM, logging, template, tagging, policy và mô hình trách nhiệm riêng. Vì vậy bảo mật cloud phải kiểm soát cả cấu hình tài nguyên, quyền truy cập và vòng đời workload.

## Ví dụ thực tế

Một công ty có thể dùng virtualization trong data center nội bộ để chạy 10 VM trên một server vật lý. Khi chuyển sang cloud IaaS, công ty tạo VM qua portal, dùng security group để lọc traffic, dùng IAM để quản lý quyền, dùng image chuẩn để triển khai nhanh và bật logging để giám sát thay đổi cấu hình.

## Cần ghi nhớ

- Virtualization là công nghệ nền; cloud computing là mô hình cung cấp dịch vụ.
- Hypervisor là thành phần quan trọng cần được bảo vệ.
- VM image và snapshot có thể chứa dữ liệu nhạy cảm.
- Cloud tài nguyên thay đổi nhanh nên cần automation và policy.
- Không nên nhầm “đưa lên cloud” với “đã an toàn”.

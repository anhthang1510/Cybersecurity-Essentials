# 17.6 - Protecting VMs

## Tổng quan

Phần này trình bày cách bảo vệ virtual machines trong cloud. VM vẫn cần các biện pháp quen thuộc như hardening, patching, endpoint protection và firewall, đồng thời cần kiểm soát đặc thù cloud như image, metadata service, IAM role, snapshot và security group.

## Ý chính

- VM image nên được harden trước khi dùng để triển khai hàng loạt.
- OS và application trong VM cần patch thường xuyên.
- Security group/host firewall chỉ mở port cần thiết.
- EDR/anti-malware, logging và monitoring giúp phát hiện compromise.
- Snapshot và image có thể chứa dữ liệu nhạy cảm nên cần kiểm soát quyền.
- VM metadata service và attached IAM role cần được bảo vệ khỏi abuse.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Golden image | VM image chuẩn đã được hardening và cấu hình baseline. |
| Hardening | Giảm bề mặt tấn công bằng cách tắt dịch vụ thừa, cấu hình an toàn và áp baseline. |
| Patch management | Quản lý cập nhật bảo mật cho OS và application. |
| EDR | Endpoint Detection and Response, phát hiện và phản ứng trên endpoint/workload. |
| Metadata service | Dịch vụ nội bộ cung cấp thông tin cấu hình/credential tạm thời cho VM. |
| Snapshot sprawl | Tình trạng nhiều snapshot tồn tại không quản lý, có thể chứa dữ liệu cũ/nhạy cảm. |

## Ghi chú chi tiết

VM trong cloud có nhiều điểm giống server vật lý: cần tài khoản an toàn, patch, log, firewall, anti-malware và cấu hình bảo mật. Tuy nhiên, cloud VM còn có đặc điểm như được tạo/xóa nhanh, dùng image để nhân bản, có metadata service và có thể được gán IAM role.

Golden image giúp chuẩn hóa baseline. Thay vì mỗi VM cấu hình thủ công, tổ chức tạo image đã harden: tắt dịch vụ không cần, bật logging, cài EDR, cấu hình host firewall và áp policy mật khẩu/SSH phù hợp. Image cần được cập nhật định kỳ để tránh triển khai VM mới từ bản lỗi thời.

Snapshot không nên được xem là backup dài hạn mặc định. Snapshot có thể chứa dữ liệu nhạy cảm và credential cũ. Cần quản lý quyền truy cập, vòng đời, mã hóa và xóa snapshot không cần thiết.

## Ví dụ thực tế

Một công ty triển khai VM Linux cho ứng dụng web. VM chỉ mở SSH từ bastion và HTTPS từ load balancer, dùng IAM role quyền tối thiểu, bật EDR, gửi syslog về SIEM, tự động patch theo lịch bảo trì và snapshot được mã hóa với retention 14 ngày. Image chuẩn được rebuild hằng tháng để tích hợp bản vá mới.

## Cần ghi nhớ

- VM cloud vẫn cần hardening như server truyền thống.
- Security group và host firewall nên cùng theo nguyên tắc least privilege.
- Golden image lỗi thời có thể nhân bản lỗ hổng ra nhiều VM.
- Snapshot/image phải được bảo vệ như dữ liệu nhạy cảm.
- Metadata service/IAM role bị lạm dụng có thể dẫn đến truy cập tài nguyên cloud khác.

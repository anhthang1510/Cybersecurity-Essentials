# 17.7 - Cloud Security Summary

## Tổng quan

Module 17 nhấn mạnh rằng cloud security là sự kết hợp giữa công nghệ, cấu hình, quy trình và trách nhiệm chia sẻ. Cloud giúp triển khai nhanh, nhưng nếu thiếu kiểm soát, lỗi cũng có thể lan rộng nhanh.

## Ý chính

| Chủ đề | Tóm tắt |
|---|---|
| Virtualization | Cho phép chạy nhiều VM/workload trên hạ tầng vật lý chung. |
| Cloud computing | Cung cấp tài nguyên IT theo nhu cầu với khả năng mở rộng linh hoạt. |
| Shared responsibility | Provider và customer cùng chịu trách nhiệm tùy mô hình dịch vụ. |
| Infrastructure security | Bảo vệ management plane, network, IAM, compute và storage. |
| Application security | Tích hợp bảo mật vào SDLC, API, secret và CI/CD. |
| Data security | Phân loại, mã hóa, kiểm soát truy cập, backup và DLP. |
| VM protection | Hardening, patching, EDR, logging, image và snapshot management. |

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Cloud workload | Tài nguyên chạy ứng dụng như VM, container, function hoặc service. |
| Least privilege | Chỉ cấp quyền tối thiểu cần thiết. |
| Misconfiguration | Cấu hình sai gây rủi ro, ví dụ public storage hoặc firewall quá rộng. |
| Automation | Tự động hóa triển khai và kiểm soát để giảm lỗi thủ công. |
| Continuous monitoring | Giám sát liên tục log, cấu hình và hành vi. |

## Ghi chú chi tiết

Cloud security bắt đầu từ hiểu đúng trách nhiệm. Provider thường bảo vệ hạ tầng cloud nền tảng, nhưng customer vẫn phải bảo vệ identity, data, application, cấu hình network và workload tùy mô hình dịch vụ. Trong IaaS, customer chịu trách nhiệm nhiều hơn so với SaaS.

Các rủi ro cloud phổ biến gồm IAM quá rộng, public exposure, thiếu mã hóa, thiếu log, secret bị lộ, image lỗi thời và backup không kiểm tra. Vì cloud có API và automation mạnh, một credential bị lộ có thể gây thiệt hại nhanh nếu không có least privilege và monitoring.

Một chiến lược tốt nên kết hợp: IAM mạnh, MFA, segmentation, encryption, secure SDLC, workload hardening, logging tập trung, policy-as-code, CSPM và incident response plan.

## Ví dụ thực tế

Một tổ chức triển khai ứng dụng thương mại điện tử trên cloud. Họ dùng private subnet cho database, WAF trước ứng dụng web, secret manager cho credential, KMS cho encryption, CI/CD có scanning, VM image chuẩn đã harden, IAM role quyền tối thiểu và SIEM để thu thập log. Cách tiếp cận này tạo nhiều lớp kiểm soát thay vì phụ thuộc vào một firewall duy nhất.

## Cần ghi nhớ

- Cloud security là shared responsibility.
- Identity, data và automation là trọng tâm lớn trong cloud.
- Misconfiguration là rủi ro phổ biến và có thể phòng tránh bằng kiểm tra liên tục.
- VM, application và data đều cần lớp bảo vệ riêng.
- Logging và incident response phải được chuẩn bị trước sự cố.

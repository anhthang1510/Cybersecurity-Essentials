# 17.2 - The Domains of Cloud Security

## Tổng quan

Phần này trình bày các miền chính cần bảo vệ trong cloud. Cloud security không chỉ là firewall; nó bao gồm identity, network, workload, application, data, monitoring, compliance và incident response.

## Ý chính

- Shared responsibility model xác định provider và customer chịu trách nhiệm phần nào.
- Identity and Access Management là lớp kiểm soát cốt lõi trong cloud.
- Network security vẫn quan trọng nhưng phải kết hợp security groups, segmentation và private connectivity.
- Workload security bảo vệ VM, container, serverless và service runtime.
- Data security tập trung vào phân loại, mã hóa, backup, DLP và lifecycle.
- Monitoring/logging giúp phát hiện cấu hình sai và hành vi bất thường.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Shared responsibility | Chia trách nhiệm bảo mật giữa provider và customer. |
| IAM policy | Chính sách cấp quyền cho user, role, service hoặc workload. |
| Security group | Firewall logic kiểm soát traffic đến/đi khỏi tài nguyên cloud. |
| Network ACL | Lớp lọc traffic ở mức subnet hoặc network boundary tùy nền tảng. |
| Logging | Ghi nhận sự kiện, truy cập và thay đổi cấu hình. |
| Compliance | Tuân thủ quy định, tiêu chuẩn và yêu cầu kiểm toán. |
| Incident response | Quy trình phát hiện, phân tích, cô lập và khôi phục khi có sự cố. |

## Ghi chú chi tiết

Trong cloud, trách nhiệm bảo mật phụ thuộc vào mô hình dịch vụ. Với IaaS, customer thường phải bảo vệ OS, application, data, IAM và cấu hình network. Với PaaS, provider quản lý nhiều phần runtime hơn, nhưng customer vẫn chịu trách nhiệm code, data, access và cấu hình. Với SaaS, provider quản lý ứng dụng nền tảng, còn customer tập trung vào user, data, permission và policy.

Các miền cloud security liên kết với nhau. IAM quá rộng có thể làm lộ data dù network được cấu hình tốt. Logging thiếu có thể khiến incident response không đủ bằng chứng. VM không patch có thể bị khai thác dù storage đã mã hóa.

Vì vậy, cloud security nên được thiết kế theo defense-in-depth: identity mạnh, network segmentation, workload hardening, data protection, continuous monitoring và automation để phát hiện cấu hình sai.

## Ví dụ thực tế

Một ứng dụng web trên cloud dùng IAM role để truy cập database, security group chỉ mở HTTPS, database không public internet, dữ liệu được mã hóa, log truy cập được gửi về SIEM và backup được kiểm tra định kỳ. Nếu một API key bị lộ, quyền tối thiểu và logging giúp giảm thiệt hại và điều tra nhanh hơn.

## Cần ghi nhớ

- Shared responsibility thay đổi theo IaaS/PaaS/SaaS.
- IAM sai là một trong các rủi ro cloud lớn nhất.
- Network security cần kết hợp identity và workload security.
- Logging phải bật trước khi có sự cố.
- Compliance không chỉ là giấy tờ; nó cần control kỹ thuật và bằng chứng.

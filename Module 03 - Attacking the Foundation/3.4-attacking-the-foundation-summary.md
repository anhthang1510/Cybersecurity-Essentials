# 3.4 - Attacking the Foundation Summary

## Tổng quan

Phần tổng kết nhắc lại các điểm quan trọng của Module 3: packet/segment headers là nền tảng để hiểu tấn công mạng. IP, ICMP, TCP và UDP đều có chức năng hợp pháp nhưng cũng có thể bị attacker lợi dụng nếu thiếu kiểm soát và giám sát.

## Ý chính

- IP là Layer 3 connectionless protocol, không tự xác minh source IP.
- IPv4 header có nhiều trường hơn IPv6 header; IPv6 header gọn hơn và có extension headers.
- ICMP dùng cho chẩn đoán mạng nhưng cũng có thể dùng cho reconnaissance, scanning và flood.
- IP vulnerabilities gồm ICMP attacks, DoS/DDoS, address spoofing, MiTM và session hijacking.
- Amplification và reflection dùng source spoofing và hệ thống trung gian để overwhelm nạn nhân.
- MAC spoofing trong LAN có thể làm switch cập nhật CAM table sai.
- TCP cung cấp reliable delivery, flow control và stateful communication.
- TCP attacks gồm port scanning, SYN flood, TCP reset và TCP session hijacking.
- UDP nhẹ, connectionless và phù hợp request/reply đơn giản nhưng dễ bị flood.
- UDP flood có thể làm cạn bandwidth và tạo nhiều ICMP port unreachable responses.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Layer 3 | Network Layer, nơi IP hoạt động. |
| Layer 4 | Transport Layer, nơi TCP và UDP hoạt động. |
| IPv4 header | Header chứa các trường điều khiển packet IPv4. |
| IPv6 header | Header cố định 40 bytes, đơn giản hơn IPv4. |
| ICMP attack | Lạm dụng ICMP để do thám, scan hoặc gây DoS. |
| Spoofing | Giả mạo địa chỉ hoặc danh tính nguồn. |
| MiTM | Man-in-the-middle, attacker đứng giữa hai bên giao tiếp. |
| Session hijacking | Chiếm phiên truyền thông đã xác thực. |
| TCP SYN flood | Tấn công làm cạn tài nguyên bằng nhiều SYN chưa hoàn tất handshake. |
| UDP flood | Flood UDP packets để tạo DoS hoặc cạn bandwidth. |
| Resource exhaustion | Làm cạn tài nguyên hệ thống hoặc mạng. |

## Ghi chú chi tiết

### Tóm tắt IP PDU Details

IP chịu trách nhiệm chuyển packet qua mạng nhưng không đảm bảo reliability và không xác thực nguồn. IPv4 có header tối thiểu 20 bytes với các trường như TTL, Protocol, Header Checksum và fragmentation fields. IPv6 có header 40 bytes, dùng Hop Limit thay TTL, Next Header thay Protocol và extension headers cho thông tin tùy chọn.

Hiểu header giúp analyst phân tích Wireshark, IDS alerts và firewall logs.

### Tóm tắt IP Vulnerabilities

IP vulnerabilities chủ yếu đến từ việc source address có thể bị giả mạo và một số protocol hỗ trợ như ICMP có thể bị lạm dụng.

ICMP giúp báo lỗi và kiểm tra kết nối, nhưng attacker có thể dùng để:

- Tìm host đang hoạt động.
- Map topology.
- Fingerprint OS.
- Kiểm tra firewall.
- Tạo flood hoặc DDoS.

Amplification/reflection attacks dùng hệ thống trung gian để gửi traffic về nạn nhân. Spoofing có thể là IP spoofing hoặc MAC spoofing. Với MAC spoofing, switch có thể học sai CAM table và chuyển traffic đến attacker.

### Tóm tắt TCP and UDP Vulnerabilities

TCP giữ trạng thái kết nối nên có thể bị tấn công vào state. SYN flood tạo nhiều half-open connections. TCP reset giả có thể ngắt kết nối. Session hijacking cố gắng chiếm phiên đã xác thực bằng IP spoofing và sequence prediction.

UDP không có handshake và ít overhead. Điều này phù hợp với DNS, DHCP, SNMP, VoIP hoặc streaming, nhưng cũng khiến UDP dễ bị flood. Nếu attacker gửi nhiều UDP packets đến port đóng, server có thể phản hồi ICMP port unreachable hàng loạt, làm cạn bandwidth.

### Ý nghĩa của sơ đồ và bảng

- Sơ đồ IPv4 header cho thấy packet có nhiều trường điều khiển, trong đó fragmentation, TTL, Protocol và source/destination address rất quan trọng khi phân tích tấn công.
- Sơ đồ IPv6 header cho thấy header đơn giản hơn nhưng địa chỉ dài hơn và hỗ trợ extension headers.
- Sơ đồ ICMP flood minh họa request/reply hợp pháp có thể bị biến thành flood.
- Sơ đồ spoofed MAC/CAM table cho thấy switch tin vào source MAC học được từ frame, nên có thể bị lừa.
- Sơ đồ TCP three-way handshake cho thấy vì sao TCP cần giữ trạng thái và tại sao SYN flood hiệu quả.
- Sơ đồ UDP header cho thấy UDP rất nhỏ gọn, ít overhead nhưng thiếu cơ chế reliability mặc định.

## Ví dụ thực tế

Một SOC nhận cảnh báo firewall về số lượng lớn ICMP Echo Request và UDP packets từ nhiều source khác nhau. Wireshark cho thấy nhiều source IP có vẻ ngẫu nhiên, TTL giống nhau và server trả nhiều ICMP unreachable. Analyst nghi ngờ kết hợp spoofing, ICMP probing và UDP flood, sau đó áp dụng rate limiting, ACL ở edge và điều tra các log liên quan.

## Cần ghi nhớ

- TCP/IP vulnerabilities thường xuất phát từ thiết kế protocol nền tảng, không chỉ từ lỗi ứng dụng.
- Header fields là manh mối quan trọng khi điều tra attack.
- Spoofing là nền tảng của nhiều attack khác như reflection, session hijacking hoặc DoS.
- TCP attacks thường khai thác handshake, flags và sequence numbers.
- UDP attacks thường khai thác connectionless behavior và thiếu state.

# 3.2 - IP Vulnerabilities

## Tổng quan

Phần này giải thích các điểm yếu và cách lạm dụng IP/ICMP trong tấn công mạng. Vì IP không xác thực nguồn packet và ICMP được dùng để chẩn đoán lỗi mạng, attacker có thể dùng chúng để reconnaissance, scanning, spoofing, DoS/DDoS, amplification, reflection, MiTM và session hijacking.

## Ý chính

- IP-related attacks gồm ICMP attacks, DoS, DDoS, address spoofing, MiTM và session hijacking.
- ICMP được thiết kế để báo lỗi và chẩn đoán mạng nhưng có thể bị dùng để scan và map mạng.
- ICMP Echo Request/Reply có thể phục vụ host discovery hoặc flood attack.
- ICMP redirects có thể bị lợi dụng để buộc host gửi traffic qua thiết bị compromise.
- Amplification và reflection dùng hệ thống trung gian để tạo lưu lượng lớn về nạn nhân.
- Smurf attack là ví dụ dùng ICMP Echo Request giả source là nạn nhân để nhiều host reply về nạn nhân.
- IP address spoofing dùng source IP giả để che giấu attacker hoặc giả dạng host hợp lệ.
- MAC spoofing trong internal network có thể làm switch cập nhật CAM table sai.
- ACL, firewall, IDS và log analysis giúp phát hiện hoặc giảm rủi ro ICMP/IP attacks.

## Ghi chú chi tiết

### Các kiểu IP-related attacks

Module liệt kê các nhóm tấn công thường nhắm vào IP:

- ICMP attacks: dùng ICMP để tìm subnet, host, firewall state hoặc tạo flood.
- DoS attacks: ngăn người dùng hợp lệ truy cập dịch vụ.
- DDoS attacks: nhiều máy cùng tấn công mục tiêu.
- Address spoofing attacks: giả source IP để che giấu danh tính hoặc giả dạng người hợp lệ.
- Man-in-the-middle: attacker đứng giữa source và destination để nghe lén hoặc chỉnh packet.
- Session hijacking: attacker dùng vị trí trong mạng hoặc MiTM để chiếm phiên.

### ICMP attacks

ICMP được tạo ra để gửi diagnostic messages và báo lỗi khi route, host hoặc port không khả dụng. Ping dùng ICMP Echo Request để kiểm tra kết nối.

Attacker dùng ICMP để:

- Map network topology.
- Tìm host đang hoạt động.
- Thực hiện OS fingerprinting.
- Kiểm tra firewall state.
- Tạo ICMP flood hoặc hỗ trợ DDoS.

Các ICMP message đáng chú ý:

| ICMP message | Cách attacker có thể lạm dụng |
|---|---|
| Echo request/reply | Host verification hoặc DoS. |
| Unreachable | Reconnaissance và scanning. |
| Mask reply | Map internal IP network. |
| Redirects | Dụ host gửi traffic qua thiết bị compromise để tạo MiTM. |
| Router discovery | Chèn route giả vào routing table của host mục tiêu. |

### Ý nghĩa sơ đồ ICMP flood

Sơ đồ ICMP flood cho thấy attacker gửi ICMP Echo Request có source giả. Victim phản hồi bằng Echo Reply nhưng không nhận được phản hồi hợp lệ hoặc bị cuốn vào lượng request lớn. Ý nghĩa chính là ICMP vốn hợp pháp nhưng nếu bị gửi với số lượng lớn hoặc source spoofed, nó có thể gây cạn tài nguyên.

### Phòng thủ ICMP

Mạng nên có ICMP ACL filtering chặt ở network edge để giảm probing từ Internet. Tuy nhiên, không nên chặn ICMP một cách mù quáng vì ICMP cũng cần cho troubleshooting và một số hoạt động mạng. Security analyst cần xem packet capture, logs, firewall alerts và IDS alerts để phát hiện bất thường.

### Amplification và reflection

Amplification làm traffic phản hồi lớn hơn request ban đầu. Reflection dùng hệ thống trung gian phản hồi về nạn nhân bằng cách giả source IP là nạn nhân.

Smurf attack minh họa: attacker gửi ICMP Echo Request với source IP là victim; nhiều host phản hồi Echo Reply về victim, làm victim bị overwhelm.

Các biến thể hiện đại có thể dùng DNS reflection/amplification hoặc NTP amplification. Resource exhaustion attacks cũng nhằm làm cạn CPU, memory, bandwidth hoặc state table.

### Address spoofing

IP spoofing xảy ra khi attacker tạo packet với source IP giả. Mục tiêu có thể là ẩn danh, giả dạng host hợp lệ, vượt qua policy hoặc tạo DoS/reflection.

- Non-blind spoofing: attacker nhìn thấy traffic giữa host và target. Điều này giúp dự đoán sequence number, kiểm tra firewall state hoặc hijack session.
- Blind spoofing: attacker không thấy traffic phản hồi. Dạng này thường dùng trong DoS.

### MAC spoofing và CAM table

MAC spoofing xảy ra trong mạng nội bộ khi attacker đổi MAC address của máy mình giống MAC của host hợp lệ. Sơ đồ trong bài cho thấy server có MAC AABBCC trên port 1, attacker đổi MAC thành AABBCC trên port 2. Khi switch nhận frame từ attacker, nó cập nhật CAM table và ánh xạ AABBCC sang port 2. Kết quả là frame gửi đến server có thể bị chuyển đến attacker.

Điểm cần nhớ: switch học MAC dựa trên source MAC trong frame. Nếu không có biện pháp như port security, attacker trong LAN có thể làm bảng học MAC sai.

### Application/service spoofing

Một ví dụ khác là rogue DHCP server. Nếu attacker cắm DHCP server trái phép, client có thể nhận default gateway hoặc DNS giả, tạo điều kiện cho MiTM hoặc traffic redirection.

## Ví dụ thực tế

Trong một mạng văn phòng, attacker cắm laptop vào cổng mạng trống và đổi MAC address giống server nội bộ. Switch cập nhật CAM table, làm traffic đến server bị chuyển sang laptop attacker. Nếu attacker kết hợp thêm packet forwarding, họ có thể tạo MiTM và ghi lại thông tin phiên.

## Cần ghi nhớ

- ICMP vừa hữu ích cho troubleshooting vừa có thể bị lạm dụng.
- Source IP trong packet có thể bị giả mạo.
- Reflection/amplification làm nạn nhân nhận traffic từ bên thứ ba.
- MAC spoofing cần quyền truy cập nội bộ nhưng có thể rất nguy hiểm.
- ACL, port security, DHCP snooping, IDS và log analysis là các tuyến phòng thủ quan trọng.

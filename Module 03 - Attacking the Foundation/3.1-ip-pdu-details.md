# 3.1 - IP PDU Details

## Tổng quan

Phần này giải thích cấu trúc IPv4 và IPv6 packet header. Đây là nền tảng để hiểu cách packet di chuyển qua mạng và vì sao một số trường trong header có thể bị attacker lợi dụng để spoofing, fragmentation abuse, reconnaissance hoặc DoS.

## Ý chính

- IP hoạt động ở Layer 3 và là connectionless protocol.
- IP chịu trách nhiệm chuyển packet từ source host đến destination host qua nhiều mạng liên kết.
- IP không tự theo dõi luồng packet và không tự xác minh source IP address.
- IPv4 header có nhiều trường như Version, IHL, DS, Total Length, Identification, Flags, Fragment Offset, TTL, Protocol, Header Checksum, Source IP và Destination IP.
- IPv6 header gọn hơn, có 8 trường chính và độ dài cố định 40 bytes.
- IPv6 dùng Hop Limit thay cho TTL và Next Header thay cho Protocol.
- IPv6 hỗ trợ extension headers để chứa thông tin tùy chọn như fragmentation, security hoặc mobility.
- Routers không fragment routed IPv6 packets như IPv4.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| PDU | Protocol Data Unit, đơn vị dữ liệu của một tầng giao thức. |
| Packet | PDU ở Network Layer, gồm IP header và payload. |
| Connectionless | Không thiết lập phiên trước khi gửi packet và không tự theo dõi luồng truyền. |
| IPv4 | Phiên bản IP dùng địa chỉ 32-bit. |
| IPv6 | Phiên bản IP dùng địa chỉ 128-bit và header đơn giản hơn. |
| IHL | Internet Header Length, độ dài IPv4 header. |
| DS/ DiffServ | Trường ưu tiên hoặc phân loại dịch vụ trong IPv4. |
| DSCP | Differentiated Services Code Point, phần của DS dùng cho QoS. |
| ECN | Explicit Congestion Notification, báo hiệu congestion. |
| Fragment Offset | Trường hỗ trợ tái lắp ráp fragment trong IPv4. |
| TTL | Time-to-Live, giảm mỗi khi qua router để tránh packet lặp vô hạn. |
| Protocol | Trường IPv4 cho biết payload thuộc ICMP, TCP, UDP hoặc protocol khác. |
| Header Checksum | Giá trị kiểm tra lỗi trong IPv4 header. |
| Next Header | Trường IPv6 cho biết header/payload tiếp theo. |
| Hop Limit | Trường IPv6 tương đương TTL. |
| Extension Header | Header tùy chọn trong IPv6 đặt giữa IPv6 header và payload. |

## Ghi chú chi tiết

### IP là connectionless

IP được thiết kế để đưa packet từ nguồn đến đích qua hệ thống mạng liên kết. Nó không đảm bảo packet đến nơi, không tự sắp xếp lại thứ tự, không quản lý flow và không xác nhận nguồn thật của source IP address. Những chức năng như reliability và flow control nếu cần thường được TCP ở Layer 4 xử lý.

Vì IP không xác thực source IP, attacker có thể tạo packet với source IP giả. Đây là cơ sở của nhiều spoofing, reflection hoặc DoS attacks.

### IPv4 packet header

Sơ đồ IPv4 header trong bài thể hiện header tối thiểu 20 bytes, chia theo các hàng 4 bytes. Ý nghĩa chính của các trường:

| Trường IPv4 | Ý nghĩa |
|---|---|
| Version | Giá trị 0100 cho biết packet là IPv4. |
| Internet Header Length | Cho biết độ dài IP header, tối thiểu 20 bytes. |
| Differentiated Services | Dùng cho ưu tiên/QoS, gồm DSCP và ECN. |
| Total Length | Tổng độ dài IP packet gồm header và data, tối đa theo trường 2 bytes là 65,535 bytes. |
| Identification, Flags, Fragment Offset | Hỗ trợ chia nhỏ và tái lắp ráp packet khi cần fragmentation. |
| TTL | Giới hạn vòng đời packet; router giảm 1 mỗi hop và hủy packet khi về 0. |
| Protocol | Cho biết payload là ICMP, TCP, UDP hoặc protocol khác. |
| Header Checksum | Kiểm tra lỗi trong IPv4 header khi truyền. |
| Source IPv4 Address | Địa chỉ nguồn 32-bit, thường là unicast. |
| Destination IPv4 Address | Địa chỉ đích 32-bit. |
| Options and Padding | Tùy chọn độ dài biến đổi, padding để đủ bội số 32-bit. |

TTL đặc biệt quan trọng trong phân tích mạng. Nếu TTL giảm về 0, router hủy packet và gửi ICMP Time Exceeded về source IP. Công cụ như traceroute lợi dụng hành vi này để xác định đường đi.

### IPv6 packet header

IPv6 header có độ dài cố định 40 bytes và ít trường hơn IPv4. Sơ đồ trong bài cho thấy 8 trường chính:

| Trường IPv6 | Ý nghĩa |
|---|---|
| Version | Giá trị 0110 cho biết packet là IPv6. |
| Traffic Class | Tương đương DS field trong IPv4. |
| Flow Label | Gợi ý các packet cùng flow cần được xử lý tương tự bởi router. |
| Payload Length | Độ dài phần payload của IPv6 packet. |
| Next Header | Tương đương Protocol field của IPv4, cho biết header hoặc upper-layer protocol tiếp theo. |
| Hop Limit | Tương đương TTL, giảm mỗi router và hủy packet khi về 0. |
| Source IPv6 Address | Địa chỉ nguồn 128-bit. |
| Destination IPv6 Address | Địa chỉ đích 128-bit. |

### IPv6 extension headers

IPv6 có thể dùng extension headers để chứa thông tin tùy chọn như fragmentation, security hoặc mobility. Extension headers nằm giữa IPv6 header và payload. Không giống IPv4, routers không fragment routed IPv6 packets; fragmentation nếu cần được xử lý theo cơ chế của IPv6 endpoints.

### Ý nghĩa với phân tích bảo mật

Khi đọc packet capture, security analyst có thể dùng header fields để trả lời:

- Packet đến từ đâu và đi đâu?
- Payload là ICMP, TCP hay UDP?
- TTL/Hop Limit có bất thường không?
- Có dấu hiệu fragmentation lạ không?
- Source IP có đáng tin hay có khả năng spoofing?
- IPv6 có extension header đáng chú ý không?

## Ví dụ thực tế

Trong Wireshark, analyst thấy nhiều packet ICMP Echo Request có cùng destination nhưng source IP thay đổi liên tục và TTL giống nhau bất thường. Vì IP không xác thực source address, đây có thể là dấu hiệu spoofing hoặc flood traffic được tạo tự động.

## Cần ghi nhớ

- IPv4 header tối thiểu 20 bytes; IPv6 header cố định 40 bytes.
- IPv6 header ít trường hơn nhưng địa chỉ dài hơn.
- TTL/Hop Limit giúp tránh packet tồn tại vô hạn.
- Protocol/Next Header giúp xác định payload tiếp theo.
- Source IP trong packet không phải lúc nào cũng đáng tin.

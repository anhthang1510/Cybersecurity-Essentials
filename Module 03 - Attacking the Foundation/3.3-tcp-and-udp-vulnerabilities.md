# 3.3 - TCP and UDP Vulnerabilities

## Tổng quan

Phần này giải thích cách TCP và UDP ở Layer 4 có thể bị khai thác. TCP cung cấp reliability, flow control và stateful communication nhưng có thể bị SYN flood, TCP reset hoặc session hijacking. UDP có overhead thấp và connectionless nhưng có thể bị flood hoặc bị chỉnh sửa dữ liệu nếu không có bảo vệ bổ sung.

## Ý chính

- TCP segment nằm sau IP header và có nhiều trường như source port, destination port, sequence number, acknowledgment number, flags, window và checksum.
- TCP dùng three-way handshake để thiết lập kết nối: SYN, SYN-ACK, ACK.
- TCP cung cấp reliable delivery, flow control và stateful communication.
- Port scanning giúp attacker tìm dịch vụ đang mở.
- SYN flood khai thác handshake bằng cách tạo nhiều half-open connections.
- TCP reset attack gửi RST giả để ngắt kết nối.
- TCP session hijacking cố chiếm phiên đã xác thực bằng spoofing và sequence prediction.
- UDP header nhỏ hơn TCP, chỉ có source port, destination port, length và checksum.
- UDP không cung cấp reliability, sequencing hoặc flow control ở transport layer.
- UDP flood có thể làm cạn bandwidth và buộc server tạo nhiều ICMP port unreachable messages.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| TCP | Transport Control Protocol, giao thức connection-oriented và reliable. |
| UDP | User Datagram Protocol, giao thức connectionless, ít overhead. |
| Segment | PDU thường dùng để nói về dữ liệu Layer 4. |
| Datagram | Thuật ngữ chính xác cho đơn vị dữ liệu UDP. |
| Source port | Cổng nguồn của TCP/UDP segment. |
| Destination port | Cổng đích của TCP/UDP segment. |
| Sequence number | Số thứ tự TCP dùng để theo dõi dữ liệu. |
| Acknowledgment number | Số xác nhận TCP cho biết dữ liệu đã nhận. |
| Control bits | Các TCP flags như SYN, ACK, FIN, RST. |
| Three-way handshake | Quy trình SYN, SYN-ACK, ACK để mở TCP connection. |
| Half-open connection | Kết nối TCP chưa hoàn tất handshake. |

| TCP reset | Gói TCP có RST bit để ngắt kết nối. |
| Session hijacking | Chiếm hoặc giả mạo phiên đã xác thực. |
| UDP flood | Flood UDP packets để tạo DoS hoặc cạn bandwidth. |

## Ghi chú chi tiết

### TCP segment header

Sơ đồ TCP header trong bài cho thấy header tối thiểu 20 bytes, gồm:

- Source Port và Destination Port: xác định ứng dụng hoặc service.
- Sequence Number: theo dõi thứ tự dữ liệu.
- Acknowledgment Number: xác nhận dữ liệu đã nhận.
- Header Length: độ dài TCP header.
- Reserved và Control Bits: chứa các flags như SYN, ACK, FIN, RST.
- Window: hỗ trợ flow control.
- Checksum: kiểm tra lỗi TCP segment.
- Urgent, Options và Application Layer Data.

Security analyst cần hiểu các trường này để đọc packet capture và phân biệt traffic bình thường với scan, flood hoặc reset bất thường.

### TCP services

TCP cung cấp:

- Reliable delivery: dùng acknowledgment và retransmission nếu không nhận ACK đúng thời gian.
- Flow control: điều chỉnh lượng dữ liệu truyền để bên nhận không bị quá tải.
- Stateful communication: kết nối có trạng thái, bắt đầu bằng three-way handshake.

Three-way handshake:

1. Client gửi SYN.
2. Server trả SYN-ACK.
3. Client gửi ACK và phiên được thiết lập.

Sơ đồ trong bài cho thấy sequence và acknowledgment numbers tăng theo quá trình bắt tay. Ý nghĩa chính là TCP phải giữ state trong lúc chờ handshake hoàn tất.

### TCP attacks

#### Port scanning

Attacker scan ports để biết mục tiêu đang chạy dịch vụ nào. Ví dụ port 80/443 cho web, 22 cho SSH, 53 cho DNS. Port mở giúp attacker chọn kỹ thuật khai thác tiếp theo.

#### TCP SYN flood

SYN flood khai thác three-way handshake. Attacker gửi nhiều SYN với source IP ngẫu nhiên hoặc spoofed. Server trả SYN-ACK và chờ ACK cuối cùng, nhưng ACK không đến. Khi quá nhiều half-open connections tồn tại, server cạn tài nguyên và người dùng hợp lệ không kết nối được.

#### TCP reset attack

TCP connection bình thường đóng bằng FIN/ACK exchange. Nhưng nếu host nhận TCP RST, nó hiểu rằng kết nối cần bị ngắt ngay. Attacker có thể gửi packet giả chứa RST đến một hoặc cả hai endpoint để phá phiên.

#### TCP session hijacking

Session hijacking khó hơn nhưng nguy hiểm. Attacker cần giả IP của một host, dự đoán sequence number tiếp theo và gửi ACK phù hợp. Nếu thành công, attacker có thể gửi dữ liệu vào phiên đã xác thực, dù có thể không nhận được dữ liệu trả về.

### UDP segment header và operation

UDP thường được dùng bởi DNS, DHCP, TFTP, NFS, SNMP, media streaming và VoIP. UDP connectionless, ít overhead hơn TCP vì không có handshake, không có retransmission phức tạp, sequencing hoặc flow control ở transport layer.

UDP header chỉ 8 bytes:

| Trường UDP | Ý nghĩa |
|---|---|
| Source Port | Cổng nguồn. |
| Destination Port | Cổng đích. |
| Length | Độ dài UDP datagram. |
| Checksum | Kiểm tra lỗi, tùy ngữ cảnh sử dụng. |

UDP không đồng nghĩa với ứng dụng “không đáng tin”. Nó chỉ có nghĩa transport layer không cung cấp reliability; ứng dụng có thể tự xử lý nếu cần.

### UDP attacks

UDP không mã hóa mặc định. Nếu không có bảo vệ ở tầng khác, traffic có thể bị đọc hoặc chỉnh sửa. Checksum có thể bị tính lại nếu attacker sửa payload.

UDP flood phổ biến hơn. Attacker dùng công cụ để gửi hàng loạt UDP packets đến nhiều ports. Server kiểm tra port, thấy nhiều port đóng và trả ICMP port unreachable. Lượng traffic này tiêu tốn bandwidth và tài nguyên, tạo hiệu ứng tương tự DoS.

## Ví dụ thực tế

Một web server bị SYN flood: firewall log cho thấy hàng ngàn SYN đến từ nhiều source IP khác nhau, nhưng số ACK hoàn tất handshake rất ít. Server có nhiều half-open connections và người dùng thật không truy cập được website. Đây là dấu hiệu điển hình của TCP SYN flood.

## Cần ghi nhớ

- TCP đáng tin cậy vì giữ state, nhưng state cũng có thể bị làm cạn.
- SYN flood nhắm vào quá trình thiết lập kết nối TCP.
- RST có thể ngắt TCP connection đột ngột.
- UDP nhanh vì ít overhead nhưng không tự cung cấp reliability hoặc encryption.
- UDP flood thường gây cạn bandwidth và tạo nhiều ICMP unreachable responses.

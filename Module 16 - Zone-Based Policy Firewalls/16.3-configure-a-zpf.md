# 16.3 - Configure a ZPF

## Tổng quan

Phần này trình bày quy trình cấu hình ZPF cơ bản. Cấu hình nên đi theo thứ tự: tạo zone, gán interface, phân loại traffic, tạo policy, tạo zone pair và áp service policy.

## Ý chính

- Cấu hình ZPF cần có đủ zone, zone membership, class map, policy map và zone pair.
- Class map xác định traffic nào được policy xử lý.
- Policy map xác định hành động đối với từng class.
- Zone pair xác định hướng traffic cần kiểm soát.
- Service policy là bước gắn policy map vào zone pair để thực thi.
- Sau khi cấu hình, cần kiểm tra bằng lệnh show và test traffic thực tế.

## Thuật ngữ quan trọng

| Thuật ngữ | Ý nghĩa |
|---|---|
| Class map | Dùng để match traffic theo protocol, ACL hoặc tiêu chí khác. |
| Policy map | Dùng để áp hành động cho traffic đã match. |
| Zone pair | Liên kết source zone và destination zone theo một hướng. |
| Service policy | Gắn policy vào zone pair. |
| Named ACL | ACL có tên, thường dùng để match traffic trong class map. |
| Verification | Bước kiểm tra cấu hình và traffic sau khi triển khai. |

## Ghi chú chi tiết

Quy trình cấu hình ZPF nên được làm có kế hoạch. Trước khi gõ lệnh, hãy vẽ topology và xác định các luồng traffic cần cho phép. Ví dụ: user cần DNS/HTTP/HTTPS ra internet; internet chỉ được HTTPS vào web server DMZ; admin chỉ được SSH từ management network.

Sau khi xác định luồng, tạo zone tương ứng với các vùng mạng. Tiếp theo gán interface vào zone đúng. Đây là lỗi rất thường gặp: đã tạo zone và policy nhưng quên `zone-member security`, làm traffic không đi qua logic ZPF như mong muốn.

Class map nên match traffic càng cụ thể càng tốt. Nếu dùng ACL, ACL nên mô tả đúng source, destination và port. Policy map nên có class mặc định để drop/log traffic không khớp.

Cuối cùng, tạo zone pair theo đúng hướng và áp service policy. Nếu muốn kiểm soát chiều ngược lại như `OUTSIDE` vào `DMZ`, phải tạo zone pair riêng với source/destination ngược tương ứng.

## Lệnh / Ví dụ cấu hình

Ví dụ học tập: cho phép user trong `INSIDE` truy cập DNS, HTTP và HTTPS ra `OUTSIDE`; traffic khác bị chặn và ghi log.

```text
! 1. Tạo zone
Router(config)# zone security INSIDE
Router(config)# zone security OUTSIDE

! 2. Gán interface vào zone
Router(config)# interface GigabitEthernet0/0
Router(config-if)# description Link to Internet
Router(config-if)# zone-member security OUTSIDE
Router(config-if)# exit

Router(config)# interface GigabitEthernet0/1
Router(config-if)# description User LAN
Router(config-if)# zone-member security INSIDE
Router(config-if)# exit

! 3. Tạo class map để nhận diện traffic cần cho phép
Router(config)# class-map type inspect match-any INSIDE-ALLOWED
Router(config-cmap)# match protocol dns
Router(config-cmap)# match protocol http
Router(config-cmap)# match protocol https
Router(config-cmap)# exit

! 4. Tạo policy map
Router(config)# policy-map type inspect PM-INSIDE-OUTSIDE
Router(config-pmap)# class type inspect INSIDE-ALLOWED
Router(config-pmap-c)# inspect
Router(config-pmap-c)# exit
Router(config-pmap)# class class-default
Router(config-pmap-c)# drop log
Router(config-pmap-c)# exit
Router(config-pmap)# exit

! 5. Tạo zone pair và áp policy
Router(config)# zone-pair security ZP-INSIDE-OUTSIDE source INSIDE destination OUTSIDE
Router(config-sec-zone-pair)# service-policy type inspect PM-INSIDE-OUTSIDE
Router(config-sec-zone-pair)# exit

! 6. Kiểm tra
Router# show zone security
Router# show zone-pair security
Router# show policy-map type inspect zone-pair
```

## Ví dụ thực tế

Một chi nhánh có router kết nối internet. Người dùng cần truy cập web và DNS, nhưng không được dùng dịch vụ lạ ra ngoài. ZPF tạo zone `INSIDE` và `OUTSIDE`, match DNS/HTTP/HTTPS bằng class map, dùng `inspect` để cho phép return traffic và drop/log mọi traffic khác. Khi cần thêm dịch vụ mới, quản trị viên cập nhật class map/policy thay vì mở rộng rule quá mức.

## Cần ghi nhớ

- Cấu hình ZPF thiếu một bước sẽ làm policy không có hiệu lực.
- Gán interface vào đúng zone là bước bắt buộc.
- Zone pair phải đúng hướng traffic khởi tạo.
- `class class-default` nên drop/log để tránh traffic ngoài ý muốn.
- Luôn dùng lệnh `show` và test thực tế sau khi cấu hình.
